Appendix A: Add Databases from Two Different Servers

You can configure a database to have multiple physical connections by mapping a Document to more than one table if the Document has one key and all the database tables that it is being mapped to have the same key column (with exactly the same name and attribute in each database table).

To configure your application to use two physical connections, perform the following tasks:

  • Add the database connection
  • Add the physical connection
  • Add the logical connection
  • Map the document with the logical connections

Add the database connection

To create a new database connection in Velocity Studio, follow these steps:

  1. Click Database > Connect from the Velocity Studio menu bar to open the Database Login dialog. For more information on connecting your project to the database, see Connect to the Database.

  2. Provide the connection information Connection Properties dialog and click OK.

 

Add the physical connection

Configure your application to add the physical connection. For example, shibly_5246. For more information, see Physical Connections.

Add the logical connection

Configure your application to add the logical connections, Order and Reference. For more information, see Logical Connections.

Map the document with the logical connections

As in the example, map the database Document with the Order and Reference logical connections to associate the connections with a database schema. This step generates the table name and column names for all Document variables that are not explicitly marked as do not map so that the connections are mapped to the database. For more information on mapping the document with logical connections, see Mapping Tab: Database Mapping.

  1. Click Database > Map Documents to DB from the menu bar.

  2. Click the DB Map tab and map the document with the logical connection, Order.

  3. Map the document with the logical connection, Reference.
