Physical Connection

Clicking Databases > Physical Connection displays the physical connections to the database, which are responsible for defining database connection parameters to be used by logical connections.

Your physical connections must be specified to run your metadata. Specifically, the names of the physical connections must match the database names used in your metadata. You can view the database connection list used by your metadata by clicking Database > Connect from Velocity Studio's menu bar. The drop-down list in the Database Login dialog shows the database connection list.

Notes:

  • Physical connections are only editable at the cluster level. They are read-only at the node level.
  • When setting up the application under normal circumstances as described in the Installation Guide, you have one physical database connection. However, in advanced application development and optimization (for example, using an Oracle Real Application Clusters (RAC)), you can have the flexibility of multiple physical connections to separate database operations based on functionality for the associated logical connections.
  • The physical connections table does not show all parameters of a physical connection, which are different based on the Driver Type selection. After creating a physical connection, you must select it, and then click the Edit button to complete all parameters.
  • When making changes to a CIL connection, it is recommended that you save them and then restart Velocity Studio. Due to the CIL client library caching previously used URLs, any new URL is not available until after you restart Velocity Studio to make the configuration active.

You can perform the following tasks from the Physical Connection page:

  • Add a physical connection
  • Edit a physical connection
  • Delete a physical connection

Note: It is recommended to restart JBoss if you make changes to configuration variables.
