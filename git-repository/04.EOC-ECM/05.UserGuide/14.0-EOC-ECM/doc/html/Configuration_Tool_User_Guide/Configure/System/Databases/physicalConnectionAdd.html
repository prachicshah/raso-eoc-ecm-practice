Add a Physical Connection

To add a physical connection, complete these steps:

  1. From the Physical Connection page, click the Add button.



  2. The New Physical Connection dialog appears.



    3. On the Connection Properties page, the following fields are available:
    Field Driver Type Description
    Driver Type Any driver type Specify the JDBC driver to use for the physical connection by clicking the field's drop-down menu and selecting from one of the following options:
    • Oracle thin
    • Any JDBC

    Note: Use the Any JDBC option to connect to any non-Oracle database or to an Oracle RAC database.
    Connection Name Any driver type Enter a name to identify your physical connection.
    Host Oracle thin Enter the host name of your Oracle database (for example, localhost).
    Port Oracle thin Enter your Oracle database's port number. By default, the port number is 1521.
    Connection Type Oracle thin Click this field's drop-down menu and select one of the following options:
    • Service ID (SID)
    • Service Name
    Service ID (SID) Oracle thin Enter your Oracle database's service ID. This field appears when you have specified Service ID (SID) in the Connection Type field.
    Service Name Oracle thin Enter your Oracle database's service name. This field appears when you have specified Service Name in the Connection Type field.
    User Any driver type Enter the database username.
    Password Any driver type Enter the database password.
    Maximum Connections Oracle thin Specify the maximum number of connections in the connection pool for each node. By default, this field is set to 25.
    Inactivity Timeout (sec) Any driver type Specify the number of seconds of inactivity before the connection closes. By default, this field is set to 300.
    Fast Connection Failover Any driver type Select this checkbox to enable Oracle RAC fast connection failover. For more information on Oracle RAC settings, consult the Oracle help documentation.
    Validate Connection Any driver type Select this checkbox to validate the connection.
    Driver Class Name Any JDBC Class name of Driver (for example, oracle.jdbc.driver.OracleDriver).
    URL Any JDBC The valid JDBC format for the Oracle RAC database is <user>@<class>;<url>

    Driver URL (for example, cw@oracle.jdbc.driver.OracleDriver; jdbc:oracle:thin:@(DESCRIPTION=(LOAD_BALANCE=on)(ADDRESS=(PROTOCOL=TCP)(HOST=rac1.conceptwave.com)(PORT=1521))
    (ADDRESS=(PROTOCOL=TCP)(HOST=rac2.conceptwave.com)(PORT=1521))(CONNECT_DATA=(SERVICE_NAME=orcl))).

  3. When you have finished, click the Apply button. A message appears, indicating that you have successfully updated your Oracle thin connection.



  4. To test your connection settings, click the Test button.



    5. If your connection settings are properly set up, a confirmation message appears.

  5. The Advanced Configuration page allows you to specify property values when establishing a database connection, such as an SSL-encrypted JDBC connection. To add a property, click the Add button, and specify both the property's Name and its Value.



  6. To remove a property from the Advanced Configuration page, do the following:
    1. Select the checkbox next to the property Name that you want to delete and click the Delete button.



    2. A confirmation dialog appears. Click the Yes button to continue. The page is refreshed and your property has been successfully removed.
  7. On the Connection Properties page, click the Apply button, and then click the Close button to return to the Physical Connection page.
  8. Click the Save button to save your configuration.

Note: If you are using Oracle 11g in a RAC environment and want to use the SCAN option, ensure that you have set the following properties:

  • Define your Driver Type as Oracle thin.
  • Ensure that you enter your SCAN name in the Host field.
  • For the Connection Type field, ensure that you select Service Name from the drop-down menu.

runActivate Vuno@racnode-cluster-scan:1521/racdb.conceptwave.com cwf.war Test_MA active
