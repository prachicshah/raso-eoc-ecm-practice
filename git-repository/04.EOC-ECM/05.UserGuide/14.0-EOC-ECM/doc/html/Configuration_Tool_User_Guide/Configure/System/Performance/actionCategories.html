Action Categories

Action categories provide additional server-side control to prevent server overload. User actions can be grouped into action categories such that server-intensive actions can be queued to restrict the number of actions to be performed simultaneously. The Configuration application contains both definitions and parameters for action categories.

On the client side (that is, the product), the action category is assigned to a User action method. At runtime, user actions assigned to an action category are queued based on the action categories parameters that follow. Once the action enters the queue, the client displays an In Process dialog that allows you to cancel the action. The client then polls the server every set time (that is, the Polling interval parameter) to update the action's status.

In Process and Cancel
While the user action is in queue, the client displays an In Process dialog that also contains a Cancel button. You have the ability to cancel the action as long as the action remains in queue. Once the action has begun its processing, you cannot cancel the action. The server communicates with the client for the action's status to disable the Cancel button from the In Process dialog. However, there may be an instance when the action has starting processing and the client is not yet aware to disable the Cancel button. In this case, regardless whether you click the Cancel button, the action cannot be cancelled.

Clicking Performance > Action Categories displays a list of action categories.

You can perform the following tasks from the Action Categories page:

  • Add an action category
  • Delete an action category

Add an Action Category

To add an action category, complete these steps:

  1. Click the Add button to add an empty row to the table.



  2. Complete the following fields:

    Field Description
    Name This field denotes the action category's name. This name must be unique within the user interface.
    Threads This field indicates the number of allowed actions within the action category that can be run simultaneously for the deployed instance.
    Wait time (sec) This field specifies the number of seconds consumed before the client displays the In Process dialog for the action.
    Polling Interval (sec) This field indicates the number of milliseconds that the client waits before polling the server for an update for the action.

  3. When you have finished making your changes, click the Save button to save your configuration.

Delete an Action Category

To delete an action category, complete these steps:

  1. Select the checkbox next to the action category that you want, and then click the Delete button.



  2. A confirmation message appears. Click the Yes button to confirm.
  3. Your selected action category has been successfully removed from the list.
