Binding

Clicking Services > Binding displays the bindings that link service ports, service providers, and interfaces together. This tab includes a binding-operation tree on the left. Clicking a binding shows its details pane on the right. Similarly, clicking an operation shows its details pane on the right.

The left pane shows a list of nodes (external services). You can specify how the node label appears by clicking the Show Node Label drop-down field. See the Service's Show the node label page, which works the same way for the Binding page.

You can perform the following tasks from the Binding page:

  • View binding details
  • View binding operation details
  • Add a new port

View Binding Details

You can view a binding's details by selecting one from the left pane to view its details in the right pane.

The following fields appear when viewing a binding's details:

Field Description
Provider Type This field denotes the provider type (for example, SOAP).
Interface This field contains the provider type's interface name.
Port

This field lists the provider's ports. If a port is associated with a binding, it appears in bolded text that is a hyperlink. Clicking the hyperlink switches your view to the Services tab and shows you the specific port's details.

You can also create a new port by clicking the New Port button, which launches the New Port dialog.
Encoding This field specifies how message data is formatted when performing a request. This field is optional.
Style

This field allows you to specify whether messages are RPC-oriented (data passed as parameters) or document-oriented (data passed as a XML document). This attribute can have one of the following values:

  • Document
  • RPC

The default value is Document.

The choice of style affects WSDL generation and formation of the SOAP message's BODY element in both SOAP requests and responses:

  • Use Document when operation messages are defined as elements and you want full control in the metadata on constructing the SOAP message's BODY contents.
  • Use RPC when operation messages are defined as types.

View Binding Operation Details

You can view a binding operation's details by selecting one from the left pane to view its details in the right pane.

The following checkbox field appears when viewing the binding operation's details:

Field Description
Log Messages

By default, this field is unchecked. If you select this checkbox, the messages that are exchanged with this operation are automatically logged in the runtime database.

The following fields appear under the Input, Output, and Fault sections, which may differ depending on the service provider that you are using. This example shows field for the SOAP service provider:

Field Description
Parts This field accepts either a single string value or multiple values separated with by semicolons (;). If specified, each value in the string is considered a child tag name of the input data structure. As a result, it is extracted from the input data structure and as part of the SOAP body element in the SOAP request. If you leave this field, the entire input data structure is used in the SOAP body.
Header This field specifies what information to set as the header of the SOAP request. This field accepts a string value containing the header name and part, separated by a semicolon (for example, hname;hpart).
  • If hname equals the data structure name used as the input message, the input data structure is taken as the base object for the header. Otherwise, the passed-in header parameter (that is, when calling invokeInterface()) is used.
  • The hpart value works similarly to the Parts setting. The product attempts to find the child node in the base object whose name equals hpart and this child node is set in the SOAP header.
Use

This field specifies the message's encoding rules. This field can have the value of literal (mainly for document-style messages).
Namespace This field contains the name to use for encoding. This field is optional.
Timeout

This field denotes the length of time before the operation times out. This field's value is in seconds.

The following field appears in the Ping section:

Field Description
Ping Message

This field allows you to specify your ping message in XML format.

Add a New Port

New ports can be created dynamically from the Services and Binding tabs. The newly created ports in the System Configuration application can share the port and binding settings from another port by using the port reference. You do not have to set the JMS or MQ connectivity, and queue settings multiple times. From the Binding tab, click the node in which you want to add a port, and then click the New Port button.

Refer to Add a new port for more detailed information. You can view the details of new port on the binding tab. For example, CatalogBasketCondition is a dynamic port associated with the CatalogBasketSOAP port and SOAP as the service provider.
