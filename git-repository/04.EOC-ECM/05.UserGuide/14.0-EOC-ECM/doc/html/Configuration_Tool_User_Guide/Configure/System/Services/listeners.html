Listeners

Clicking Services > Listeners displays the metadata-defined listeners.

The following fields are available on this page:

Field Description
Namespace This field denotes the namespace to which the listener belongs.
External Service This field has the external service name to which the listener belongs.
Port

This field represents the listener's port name.
Interface Name This field contains the listener's interface name.
Binding Type This field denotes the listener's binding type.
Threads

This field contains the number of threads that the listener has. The default value is 1. This field can also take a value of 0, which disables the listener.

Note: For SOAP and HTTP requests, the AVM listener thread setting only supports the values: 0 for off, and 1 for on. As listener threads are managed by the web container, to configure the number of threads available for SOAP and HTTP requests, refer to Web container documentation.
Velocity Studio's embedded web container creates and pools threads as needed to handle HTTP and SOAP requests and these listener thread settings cannot be configured.

 

Edit the Number of Listener Threads

To edit the number of threads allocated to a listener, complete these steps:

  1. From the Listeners page, double-click the listener row that you want.



  2. The Threads field is the only one that can be changed. Enter the number of threads in this field, and then click the Save button.

Notes:

  • The Threads field can be set at both cluster and node levels. If you have different hardware dimensioning for each node in the cluster, you may want to specify a different thread count at the node level for each node that suits the dimensioning.
  • When you change the number of threads for either a listener or a queue, this value is also updated under the same port in the Services tab or the same queue under the Queues tab.
  • At runtime, listeners are managed through JMX.
