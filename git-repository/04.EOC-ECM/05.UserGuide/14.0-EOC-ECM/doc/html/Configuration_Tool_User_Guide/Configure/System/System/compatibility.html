Compatibility

Clicking System > Compatibility displays settings that support document and store procedure modes from product version 5.x.

By default, the settings are enabled. To disable them, deselect the checkbox next to the setting that you want. When you have finished making your changes, click the Save button to save your configuration.

The following settings are available:

Setting Description
5.x User Profile Document Mode

By default, support for 5.x mode is available where the cwf_up.userProfile document is returned when accessing relevant UserProfile methods.

Note: If this setting is not selected, a data sturcture is returned instead of the cwf_up.userProfile document.
5.x Worklist Document Mode By default, support for 5.x mode is available where the cwl_worklist.worklist document, which is aliased from cwf_pm.BaseWorklist, is returned wherever the worklist document is expected.
5.x Worklist Stored Procedure Mode

By default, support for 5.x mode is available where worklist behaviours rely on stored procedures in the database. Selecting this checkbox allows you to extend or customize a stored procedure, indicating that the worklist calls the specified stored procedure. Otherwise, when this setting is not selected, the worklist uses the newly implemented JavaScript function. This feature supports backward compatibility.

The following stored procedures are available and are implemented in JavaScript:

  • COMPLETEALERT
  • COMPLETEWLTASK
  • COMPLETEWLTASKWITHPART
  • CREATEALERTARCHIVE
  • CREATETASKARCHIVE
  • INSERTPARTICIPANTSMESSAGE
  • STARTWORKONTASK

Example: Use the 5.x Worklist Stored Procedure Mode Setting

This example determines whether the result is the same by calling either the STARTWORKONTASK stored procedure or the JavaScript.

  1. Create two worklist tasks.
  2. In the System Configuration application, under System > Compatibility, select the 5.x Worklist Stored Procedure Mode setting.
  3. Call worklist.startWork on task 1.
  4. Uncheck the 5.x Worklist Stored Procedure Mode setting.
  5. Call worklist.startWork on task 2.
  6. Verify that the start date has been set in for both tasks.
