Config Variables

Clicking System > Config Variables displays a list of configuration variables. Configuration variables are system-specific variables that you can define to adjust system settings.

You can perform the following tasks on the Config Variables page:

  • Add a new configuration variable
  • Edit a configuration variable
  • Delete a configuration variable
  • Encrypt a configuration variable

Note: It is recommended to restart JBoss if you make changes to configuration variables.

Add a Configuration Variable

To add a new configuration variable, follow these steps:

  1. Click the Add button to add an empty row to the table.



  2. A dialog opens. Enter the Name of your configuration variable, its Value, and select whether you want to Encrypt the value in the fields provided.



  3. When you have finished creating your configuration variable, click the Save button to save your configuration.
  4. The dialog closes and your new configuration variable appears in the list.

Notes:

  • The Name field is mandatory, while the Value field can be left blank.
  • At runtime, values set on this page may be retrieved for specific keys using the Global._getConfigVariable("MsgDir") JavaScript function.
  • See the list of configuration variables that you can add to your configuration.

Edit a Configuration Variable

To edit a configuration variable, do the following:

  1. Double-click the field within the table cell that you want and make your changes.
  2. When you have finished editing your configuration variable, click the Save button to save your configuration.

Delete a Configuration Variable

To delete a configuration variable, complete these steps:

  1. Select the checkbox next to the configuration variable that you want, and then click the Delete button.



  2. A confirmation message appears. Click the Yes button to confirm.
  3. Your selected configuration variable has been successfully removed from the list.

Encrypt a Configuration Variable

To encrypt a configuration variable, complete the following steps:

  1. Select the checkbox next to the configuration variable that you want, and then click the Encrypt button.



    Note: Alternatively, you can click the configuration variable that you want and drag it to the Encrypted Variables section.
  2. A confirmation message appears. Click the Yes button to confirm.
  3. Your selected configuration variable has been successfully encrypted and appears in the Encrypted Variables section.

Notes:

  • A decryption button is not available due to security reasons. This feature is unavailable to prevent encrypted values from being disclosed.
  • The getConfigPassword and getConfigVariable methods are available, which support this functionality. Refer to the JavaScript documentation for details.

List of Configuration Variables

The following are a list of available configuration variables:

Variable Description
CW_NLS_COMP

This configuration variable supports Oracle linguistic sorting (for example, sorting customer names that are written in both uppercase and lowercase letters, and require being sorted in a list on the UI). An example of Oracle linguistic sorting is as follows:

SELECT * FROM test ORDER BY NLSSORT(name, 'NLS_SORT=german');

This variable denotes the original value of Oracle database collation behaviour. Valid values for this configuration variable are as follows:

  • BINARY
  • LINGUISTIC
  • ANSI
CW_NLS_SORT This configuration variable supports Oracle linguistic sorting (for example, sorting customer names that are written in both uppercase and lowercase letters, and require being sorted in a list on the UI). This variable denotes the original value of Oracle database collating sequence for ORDER BY queries.
cwPreloadCatalogRules When this configuration variable is set to true, Catalog definitons and scripts are loaded at startup.
ignoreNonExistingItemAttributes When this property is set to true, it returns a null value if you access a non-existing item attribute on the basket item. Otherwise, if this property is set to false or is not specified, an exception occurs when you access a non-existing item attribute.
itemStatusToReadFromDB

The value of this configuration variable defines what states the objects must be in to be read into the catalog cache. Values can be a single string (for example, ACT) or several strings (one per line, such as ACT DEF, pressing the Enter key after ACT, so that ACT and DEF are on two separate lines). If this variable is not set, its default value is ACT.

Note: This variable takes effect after Velocity Studio is restarted.
KEYSTORE_PATH, KEYSTORE_PASSWORD Keystore-related variable names must be configured when using encryption.
noReadOnlyIcons A not null value disables the read only icons (they will be displayed as regular icons).
peUnconsumedMessagedPeriod The PE shows diagnostic information as output that is related to the participant message cache when messages have not been processed for a specific period for a given PE instance. The peUnconsumedMessagedPeriod configuration parameter controls the period in seconds. If this parameter is not specified or set to being less than 1 second, no analysis occurs. If it is set to n and n seconds passes since the last consumed message, the PE produces information showing the participant cache's state. If the peConsumerId user configuration parameter is set to a specific consumer, additional information related to that consumer becomes output, which appears in either the console log or in cwprocesseventlog at the error level.
phasedValidation

Setting this configuration variable to true indicates that mandatory fields are first checked for validity (that is, where Nullable = false on a data level or Optional = false on a data type or variable level). If all mandatory fields are populated, it verifies the variable value data type, such as length, enumeration code, and so on.

If all variables pass data type verification, it then runs custom validation scripts for variables and document-level validation rules.

Notes:

  • The validation runs all variable and document rule scripts even when there are errors for mandatory fields or at the data type level.
  • The default value for this configuration variable is false.
screenreader Set this configuration variable to true to enable screen reader mode. You can also set this mode at the application user interface level.
