General

Clicking System > General displays a list of system settings.

The following system settings are available when you have selected the cluster node:

System Setting Description
Log Queries Longer Than (Seconds)

This property specifies the query length in seconds. The default value is 0 seconds. This field can range in value from 0 to 250.
Partitioning Interval (Months)

This property sets the database partitioning interval in months, if applicable. A setting of 0 indicates that this feature is not in use.
Script Optimization Level This property specifies whether to have AVM optimize scripts (Optimized) or not (None). Turn optimization off when debugging.
Leaf Encryption Key (Hex)

This property defines the encryption key used by the system. This key must be at least 48 hexadecimal characters. It is recommended to generate a cryptographically strong key, as opposed to using random hexadecimal characters. If this field is left blank, the system's default key is used for encryption.

Note: The leaf encryption key is defined as low as the system level.
Enable Audit This checkbox property allows you to enable or disable auditing. There is an Audit trail flag in the General tab of the Document in the Velocity Studio that triggers the system to record a history log for the Document. When the Audit flag is unchecked in the Configuration application, no log is recorded for any Document.
HTTP API Login Required

If this property's checkbox is selected, login is required for any API access through HTTP, which is relevant to any HTTP listener created in the application.
Multiple Application Deployment Select this property's checkbox to indicate that the system has been configured for multiple application deployment.
Last Updated This property contains the last date and time that the configuration node was updated.
Last Updated By This property denotes the user ID of the last person who updated the configuration node.


Selecting a non-cluster node displays a different set of fields:

The following system settings are available when you have selected any non-cluster node:

System Setting Description
Active Configuration

Select this property's checkbox to indicate that the configuration can be used by a runtime instance. Otherwise, leaving this property unchecked denotes that this configuration cannot be used at runtime.
Last Updated This property contains the last date and time that the configuration node was updated.
Last Updated By This property denotes the user ID of the last person who updated the configuration node.
