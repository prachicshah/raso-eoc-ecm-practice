Logging

Clicking System > Logging displays a list of system logging levels, which specifies the level of logging of various components.

Logging works with the Enable Action Logging parameter found by clicking UI > General from the menu bar.

You can perform the following tasks on the Logging page:

  • Add a new logging variable
  • Edit a logging variable
  • Delete a logging variable

Note: It is recommended to restart JBoss if you make changes to configuration variables.

Add a Logging Variable

To add a new logging variable, follow these steps:

  1. Click the Add button to add an empty row to the table.



  2. Enter the Name of your loggiing variable. Proceed to set the logging level for both the Console Level and Database Level fields by clicking their respective drop-down menu and selecting a value from the list.



    The Console Level and Database Level fields enable additional runtime logging information for diagnostic needs. The following choices are available for these fields:

    Choice Description
    OFF This setting indicates that logging is off.
    ERROR This setting allows for error-level logging.
    WARNING This setting denotes warning-level logging.
    INFO This setting represents information-level logging.
    DEBUG This setting indicates debug-level logging.
    TRACE This setting allows for trace-level logging.
    ALL This setting indicates that everything is logged.

  3. When you have finished creating your logging variable, click the Save button to save your configuration.

Notes:

  • The Javascript interpreter log level can be set to DEBUG to allow output from the JavaScriptGlobal._println and Global._logDebug functions.
  • The User Interface log level needs to be set to DEBUG to generate product GUIDs for all generated user interface objects.
  • The com.conceptwave.statistics log level can be set to DEBUG (or TRACE to get statistics for all resource server requests) to enable the Statistics log. There are server-side statistics and client-side statistics. For server-side, you should find the statistics in the log, akin to the following: 
    SERVER statistics [ user: upadmin, action: userAction[Button1cwOnFormFieldAfter], extract: 0, invoke: 78, generate: 16, full_cycle: 94] (ms)
    where invoke is the time to invoke the action script from metadata and full_cycle is the time to process the request on the server. For client-side, statistics are collected by browser, akin to the following: 
    CLIENT statistics [ user: upadmin, action: ShowTree, generate: 3, response: 95, update: 4, full_cycle: 103] (ms)

    where generate is the time to collect field data and generate request to the server, response is the time to get response from the server, and update is the time to update received data to the page. Browser-side statistics is gathered with delay -- at the next server request. If you set the database log level for the statistics, it increases the server response time significantly. Instead, logging statistics on the file log is recommended.

  • To enable tracing to show what rules were processed for various actions, add com.conceptwave.runtimemetadata.CwfMetadataRTRule and set its log level to either TRACE or ALL.

Edit a Logging Variable

To edit a logging variable, do the following:

  1. Double-click the field within the table cell that you want and make your changes.
  2. When you have finished editing your logging variable, click the Save button to save your configuration.

Delete a Logging Variable

To delete a logging variable, complete these steps:

  1. Select the checkbox next to the logging variable that you want, and then click the Delete button.



  2. A confirmation message appears. Click the Yes button to confirm.
  3. Your selected logging variable has been successfully removed from the list.
