General

Clicking UI > General displays a list of settings that strictly pertains to the user interface.

The following user interface settings are available:

UI Setting Description
Iframe Name

This property allows you to specify the iframe ID. The default value for this property is cwIFrame. This property must be a single word, with no spaces. Setting this property allows the product to resize the iframe based on the current content that is being displayed.
Hover Delay

This property specifies the amount of time to wait for the hover form to show in the table or tree element when you have the mouse over the table or tree. By default, hover delay is set to 2000 milliseconds unless otherwise specified in the configuration.

The current allowed range for a table element is from 300 to 4999 milliseconds, and from 500 to 4999 milliseconds for a tree element. The reason for having the lower bound is that if the hover delay is too small, the hover action may block either the click or double-click action when you attempt to click a row in either the tree or table.
HTTP Port The framework uses this property to redirect the client Web browser when redirection is required to or from HTTPS.
SSL Port The framework uses this property to redirect the client Web browser when redirection is required to or from HTTPS.
Session Timeout This property sets the client browser session timeout. Its default value is 30 minutes.
Session Timeout Warning

This setting allows the application to notify the user that the session will be automatically invalidated. The default value is 0, meaning that no timeout warning is issued. This parameter takes a minimum value of 15 seconds and a maximum value is the current Session timeout in seconds.

Refer to Session timeout warning for details on customizing the popup in your metadata.
Worklist Poll Time This property sets the client browser worklist polling frequency. Default value is 5 minutes.

Note: Decreasing this value will incur additional framework loading.
Maximum Query Duration (Seconds) This property specifies the maximum duration in seconds and limits the duration specified in finder objects within the Velocity Studio. The default value is 90 seconds. This field can range in value from 2 to 250.
Maximum Attachment Size (KB) This property denotes the maximum attachment size allowed when uploading a file. The default value for this setting is 102400 KB (100 MB). A condition in the system checks the file size before the file is uploaded to the database.
Fire Initial Tab Selected

Set this variable to true if the tabSelected method defined on the tabset element should be called when it is first shown on the screen.
Enable Auto Move Focus By default, this property's checkbox is selected. When you have entered the maximum characters allowed in the field, the next character entered moves the focus to the next editable field. To disable this functionality, deselect this property's checkbox. Alternatively, you may set the Auto-focus to Next Field property on the element itself.
Enable Request Parameters

This configuration variable allows you to enable URL request parameters by selecting the checkbox. For example, if the given URL is http://localhost:8080/cwf?startPage=news, you can retrieve the parameter using the getRequestParam("startPage") method. This configuration variable is disabled by default.

Parameters are kept in a map. Each new request, if it has parameters, replaces parameters of the same name with new values. Request parameters are cumulative, until cleared using the clearRequestParams() method.

Notify application that request parameters have changed
When you have changed request parameters within the Web browser address bar, the application needs to be notified that the request parameters have changed. To be notified of page loading, you can either add a new script or the onAppLoading user action method of the application UI. This method is called before the application page loads in the Web browser, including these cases:

  • When selecting the application (after both the onInit and appSelected methods)
  • When moving from one application URL to another
  • When entering new URL parameters in the address bar or pressing the Enter key in the address bar (in this case, the page is also being reloaded)
  • When refreshing your Web browser (pressing the F5 key)

Notes:

  • Since the onAppLoading method can be invoked many times, you may be introducing a performance issue. As a result, it is recommended that you do not use heavy logic and perform only necessary tasks that require updated URL parameters.
  • To get all parameter names, use the getRequestParamNames(CwfContext uc) method.

URL parameters
The following are recognized URL parameters:

URL parameter Description
app This parameter allows you to to open a Web browser, select an application, and trigger an init script (for example, http://localhost:8080/cwf/?app=app:mainApp&orderId=225000).
sessionId The sessionId="xx" as a URL parameter allows the product to create and maintain a separate pseudo session with its own independent user context and UI context (for example, http://localhost:8080/cwf/TestApp?userID=1234&productID=CCS_CALL_CARDS&lineID=123&action=print&pi=CC&mode=advanced&loginID=TESTUSER&sessionId=print).
Enable Action Logging When an action is invoked, the associated logging action is also invoked when this parameter is selected. The default value for this parameter is false (that is, the checkbox has not been selected).
Different Domain This property must be set to false if the application is to run in an iframe. The domain used for the iframe must be the same as the application; otherwise, dialog centering does not work. iframes that are not in the same domain are unsupported. Enable Partial Update

This property's checkbox is selected by default, meaning that at runtime, if there are dialogs in the runtime user interface, the server and clients only exchange data in the foremost dialog. This communication reduces server processing time, client-server communication time, and client data parsing time.

If you deselect this property's checkbox, server and client communication involves data for the entire page. As a result, communication requires more time and can be time-consuming. This checkbox is helpful when working with metadata and attempting to debug while making changes to it.
Allow to Deselect Radio Button The default value is false, meaning that a radio button works as expected and cannot be deselected. If this setting is set to true, deselection is allowed.
Scroll on Update

This property controls the scrolling behaviour after a table update and takes the following values:

  • Default
    The table scrolls to the first selected record. If no record is selected, the table scrolls to the top of the records.
  • Restore position
    The table scrolls to the position before the update.
  • No scrolling
    The table always scrolls to the top after an update.

When you define the Scrolling on Update property at the table level, it overrides this property that you have defined in the Configuration application.
