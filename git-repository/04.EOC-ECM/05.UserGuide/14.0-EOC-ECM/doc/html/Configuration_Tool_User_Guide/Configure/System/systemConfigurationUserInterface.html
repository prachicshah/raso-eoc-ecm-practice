System Configuration User Interface

The Configuration application consists of a node menu on the left and a details menu on the right.

Node Menu

The node menu displays all nodes by Node ID that are configured under the cluster, which is typically named CLUSTER. You have the option of naming your cluster node in Velocity Studio, but it cannot be renamed after its creation.

In Velocity Studio, you can create multiple clusters. When you connect to your database and start runtime, a dialog appears, prompting to you specify the application that you want to launch. Click the New button to create a new application by completing all field, and then click the OK button. The newly created cluster's name automatically is the application name. Once created, you can neither remove, nor rename the cluster.

Select a node in the node menu to edit the configuration of that particular node in the main area on the right. You can also select the cluster to edit the configuration at the cluster level.

You can also add or remove a node to or from the cluster from the node menu.

Cluster Node

When you select the cluster node from the left menu, all icons in the details menu and their associated tabbed pages are visible.

When you make changes to your configuration at the cluster level, the following buttons appear in the node menu:

Button Icon Description
Save

Click this button to save your configuration.

Note: Clickiing this button with the cluster (that is, root node) selected overwrites any child value if the child value has the same value as the root node.
Save All

Click this button to save your configuration, including child nodes. This button appears when the selected node has children.

Notes:

  • Clicking this button with the cluster (that is, root node) selected overwrites any child value even if the child has a different value.
  • Clicking this button with the cluster or root node value being empty causes the child to delete its original value, then look up the parent list until it finds a non-empty value. As a result, you cannot propagate a null value to its children unless the topmost default value is null.

PE and PE_UI Nodes

When you select a PE (process engine) or PE_UI node, or any child node from the left menu, the following icons do not appear in the details menu:

  • Databases
  • UI

UI nodes

When you select a UI node or any child node from the left menu, the following icons do not appear in the details menu:

  • Databases
  • UI
  • Processes

Additionally, only these tabbed pages are available:

  • Under the System icon, only the General and Config Variables tabs appear
  • Under the Processes icon, only the General, PE Processes, and PE Queues tabs are visible
  • Under the Services icon, only the Listeners tab displays
  • Under the Catalog icon, only the Catalog tab appears
  • Under the Performance icon, only the AVM Cache and Catalog Cache tabs are visible

Second- and Third-Level Nodes

When you make changes to a second-level node and select it from the left menu, the following buttons appear:

Button Icon Description
Save Click this button to save your configuration.
Add Click this button to add a new PE node to your configuration.
Save All Click this button to save your configuration, including child nodes. This button appears when the selected node has children.

When you select a third-level node from the left menu, the following buttons appear:

Button Icon Description
Save Click this button to save your configuration.
Copy

Click this button to copy the selected node under the same parent node. A dialog then appears, prompting you to enter a node name in the field provided. Click the Save button to complete copying the node.

Note: Changing the value of a config variable belonging to the source node (that is, the node used to create a copy of it) does not change the value of the same config variable in the copy node. In other words, nodes that contain the same config variable can have different values and are independent of one another.
Add Click this button to add a new PE node to your configuration.
Delete Click this button to delete the selected node from your configuration. Only an inactive node-level configuration (that is, the Active configuration field is unchecked) can be deleted.
Save All Click this button to save your configuration, including child nodes. This button appears when the selected node has children.
Rename Click this button to rename the selected node. A dialog appears, which allows you to change the node ID. Click the Save button to complete renaming your node.

Details Menu

The details pane displays the configuration of a node or the cluster node selected from the left node menu. The following catagories are available, depending on the node type, by clicking their corresponding icon:

  • System
  • Databases
  • Processes
  • UI
  • Services
  • Catalog
  • Performance

Within each category, the configuration is further divided into horizontal tabs. When a property is changed, the affected tabbed page appears with an asterisk (*) to denote that the change has not been saved. You must click the Save button to persist your changed configuration into the database before selecting another node in the node menu or closing the System Configuration application.

The configuration properties may differ between node-level configuration and cluster-level configuration. These differences are explained on each tabbed page in this guide.

Note: For most properties, they are editable at both the cluster and node levels. In this case, changing these properties at the cluster level overwrites the existing node-level values. You can also change these properties at the node level, to specify different values among nodes.
