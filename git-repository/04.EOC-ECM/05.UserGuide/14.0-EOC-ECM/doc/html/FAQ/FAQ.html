With the usage of JavaScripting, what is done if an error occurs in the application that results in non-working environment?

JavaScript errors are captured by the application. The end-users action will fail but the user is permitted to proceed to perform other actions, for example, if an error occurs while working with a web site that is JavaScript based, the website does not crash. The fail safe for the end user in the event of long-running queries, dropped connections, etc. is a simple refresh command (F5 on MS IE).

From a support perspective, the errors are recorded in the event log with a stack trace for diagnosis purposes (and the user is provided the event log number to report). The Order Care platform provides a debugging facility that includes break point and stepping functionality to diagnose errors.

Refer to Diagnostic and Support Guide for more information.

 

Is there a quick way to create a database user?

Instead of creating a user manually, you can use the following script to create database users. Use any database program, such as Oracle SQL Developer.

              DROP USER cw CASCADE; 
              
              CREATE USER cw  IDENTIFIED BY cw;
              
              GRANT CONNECT TO cw  WITH ADMIN OPTION;
              GRANT ALTER USER TO cw ;
              GRANT CREATE JOB TO cw;
              GRANT CREATE PROCEDURE TO cw ;
              GRANT CREATE SEQUENCE TO cw ;
              GRANT CREATE SYNONYM TO cw ;
              GRANT CREATE TABLE TO cw ;
              GRANT CREATE TRIGGER TO cw ;
              GRANT CREATE TYPE TO cw ;
              GRANT CREATE USER TO cw ;
              GRANT CREATE VIEW TO cw ;
              GRANT CREATE CLUSTER TO cw ;
              GRANT CREATE INDEXTYPE TO cw ;
              GRANT CREATE OPERATOR TO cw ;
              GRANT CREATE SESSION TO cw ;
              GRANT DROP USER TO cw ;
              
              GRANT UNLIMITED TABLESPACE TO cw ;
              
              alter user cw quota unlimited on users;

Here cw is used as username and password. You can replace the cw with a username or password of your choice.

 

Why the run button is not active?

Sometimes there are no errors in a Project yet you are not able to run it. Right-click the root metadata (for example, Test Metadata) and click the Refresh option. This action will reload the entire project and will resolve any non-error conflicts.

How to add custom error message codes in Velocity Studio?

To add a new error or message code, follow these steps:

  1. Click the Resources tab in the left pane of Velocity Studio.
  2. Select the Application Resources in the left pane and click the Add button ().
  3. The New Value dialogue appears; enter information in the ID, Message, and Description fields.
  4. Click the Ok button.

The new error or message code has been added to the project. Custom created error codes are relevant to project they are created in. To use custom error codes in other projects copy the Resources folder from current metadata folder and paste in the metadata folder of another project.

For more information refer to Navigation Pane - Resource.

 

How to export the error list from Velocity Studio?

You can export the list of errors as an Excel file, follow these steps:

  1. From the menu bar of Velocity Studio, navigate to File > Export > Translation.

  2. Select a preferred language from the Select language to export dialog and click the OK button.
  3. The Select Directory dialog appears. Specify a directory to save the excel file. Do not specify the name of the file; the name is generated by the system.

 

Is there any impact on user Interface (UI), if the fields defined in Product Catalog, and non-product data elements (for example, fields (data elements) which are used to capture reporting or tags for external systems) defined in the Velocity Studio are moved in UI at development time and later the product definition is changed?  Does these changes retain in Product catalog if more products are added? Is there any need to rearrange fields in UI?

By default, Catalog defined attributes presented on the UI are presented as a list of attributes that need to be completed. The addition of a new attribute will extend the list of attributes or characteristics that need to be configured. It will not affect the positioning or placement of non-catalog data elements.

If a custom form is developed to tailor the presentation of these configurable attributes to speed order entry, or to include integration to a 3rd party system (such as number management), this custom form must be manually adjusted to include the new attribute(s) defined in the catalog.

If changes are made to data elements within the metadata, these changes need to be reflected in the UI forms as defined within Velocity Studio. Each catalog attribute is assigned an internal 32 character GUID. It is identified internally by this name and as such can be moved anywhere on a form without impact. Subsequent changes to the catalog will not affect the assigned ID (unless it is deleted in the catalog, then re-added).

 

How to capture data elements for each product and get them into the Product Catalog in a time effective way?

Catalog manager provides the functionality to gather data elements in no time. It includes a bulk load facility that permits to import data elements from a provided MS Excel template. The template is broken into one sheet per catalog entity, with the columns being the attributes.

 

Does two to three months time frame is sufficient to generate the UI using Order Care and Catalog Manager?

With Order Care and Catalog Manager, two to three months is more than sufficient time to generate the UI for all product data elements. The longest lead time will be in discovering or gathering the required elements to enter into the system.  Defining the elements and designing the forms is not a large endeavor.

Order Analytics (OA) module can be used for real-time operational reporting (statistics and measurements) to address all aspects of the end-to-end order fulfillment lifecycle. How to show or block different fields in an application based on user profile (for example, external user sees less fields than internal user)?

To allow certain functional tasks to be performed only by specific users with certain skills, Order Care allows you to assign user privileges to user participants.  User privileges specify allowable actions in the system. User privileges are defined and assigned to the user in the System Administration application under User Profile Manager menu.
Privileges are used in the following manners:

  • To define a privilege that must be held in order to access an order care application
  • To define a privilege that must be held in order to view or perform a specific action
  • To identify the orders, order pages, forms, and fields that are visible and editable, or both
  • To identify membership within a participant or workflow actor
  • To define a privilege that must be held for external systems to communicate to Order Care through an API

The relationship between users and the privileges are defined using Groups so that:

  • A group holds privileges
  • A user may be a member of one or more groups

The user holds the sum of all privileges held by all the groups to which he or she belongs.

 

Where you can find the download object and how to specify a user interface that can be used to look up the result table?

In the Velocity Studio, the toXML(), toXLS(), and toCSV() functions are available to create a download object, and to invoke download data at runtime. The following are the steps to access and edit these functions:

  1. Create a finder user interface that extends ui_common.baseFinder (Finder UI).
  2. Click the Methods tab of finder UI, and click the Show base methods () button.
  3. The built in functions toXML(), toXLS(), and toCSV() can be found on this tab as JavaScripts.
  4. Right-click one function, for example, toXML() and select Override toXML() option. You can edit the Download.resultUI and download.resultForm options in the script to specify which user interface to use to look up the result table if result form of finder does not have a table element.

 

How to resolve the errors of a rule set that has invalid rule items?

If any variable of rule set source object is deleted or changed (for example, data structure variable was changed from data type to element), the rule set becomes invalid.
For rule sets, error cleanup is performed automatically within the Velocity Studio. To resolve the errors select the rule set and do save.

 

How to set custom application level HTTP status codes in the REST HTTP listeners which limits the REST communication between client and server?

  1. Create a new project in Velocity Studio.
  2. Create a namespace (for example, testNS) and create two common data structures (for example, dsInput and dsOutput) under it.
    1. Create new data types (for example, message of type String) under dsInput data structure
    2. Create two data types (for example, httpMethod and customURL of type String) under dsOutput data structure.
  3. Create an external HTTP listener service in Velocity Studio with scripts associated with it (ServerScript and TestClient). You create one operation for each Httpmethod (for example, OpDelete, OpHead, OpGet, OpPut, and OpPost).

    4. You can use the following as an example for the TestClient script.

    var testCase="TT_26535";
var testDescr="HTTP Listener - to be able to set the HTTP Response code in the header";
var passed = false;
try{
    Global.logDebug("Client Script sending DELETE request");
    var aresponse = Global.invokeInterface("TT_26535.ClientInterface","OpDelete");
    Global.logDebug("Client Script received method: " + aresponse[0].httpMethod);
    Global.logDebug("Client Script received custom URL: " + aresponse[0].customURL);
    Global.logDebug("Client Script sending GET request");
    var aresponse = Global.invokeInterface("TT_26535.ClientInterface","OpGet");
    Global.logDebug("Client Script received method: " + aresponse[0].httpMethod);
    Global.logDebug("Client Script received custom URL: " + aresponse[0].customURL);
    Global.logDebug("Client Script sending HEAD request");
    var aresponse = Global.invokeInterface("TT_26535.ClientInterface","OpHead");
    Global.logDebug("Client Script received method: " + aresponse[0].httpMethod);
    Global.logDebug("Client Script received custom URL: " + aresponse[0].customURL);
    Global.logDebug("Client Script sending PUT request");
    var aresponse = Global.invokeInterface("TT_26535.ClientInterface","OpPut");
    Global.logDebug("Client Script received method: " + aresponse[0].httpMethod);
    Global.logDebug("Client Script received custom URL: " + aresponse[0].customURL);
    Global.logDebug("Client Script sending POST request");
    var aresponse = Global.invokeInterface("TT_26535.ClientInterface","OpPost");
    Global.logDebug("Client Script received method: " + aresponse[0].httpMethod);
    Global.logDebug("Client Script received custom URL: " + aresponse[0].customURL);
}
catch(ex){
     APIS.Helper.logging(testCase,"An exception was thrown in the client script: "+ex,false);
}

    You can use the following as an example for the ServerScript. 

    var result = new testHTTP.dsOutput();
    var method = UserProfile.getHttpMethod();
    result.httpMethod = method; // http method as string in the result message
    // Global.logDebug("Server reveived HTTP method " + method);
    // script to be invoked with a custom URL configured in the Binding/Location section of the configuration
    var adir = UserProfile.getRequestPath();
    var dir = "";
    for(var i=0;i < adir.length; i++){
        dir +=  adir[i];
        dir += " ";
    }
    result.customURL = dir;
    // Global.logDebug("Server reveived custom URL path with " + adir.length + " elements");
    // Global.logDebug("Path element received: " + result.customURL);
    if(method == "HEAD"){
        // when the method is HEAD, the server is not expected to return any contents
        // therefore we cannot verigy that the call has worked by checking returned data in the client
        // so, I'm using this reposnse code as a way of indicating to the client that the server has
        // received a HEAD message
        UserProfile.setHttpResponseCode(301);
    }
    return result;
  4. Under ClientBindings, associate the appropriate HttpMethod for each operation. For example, select the DELETE HttpMethod for the delete operation.

  5. Create an application user interface (for example, httpUI). Create a method for each operation and initialize the method with custom status codes. For example, testHttp_Delete method corresponds to the Delete operation.

  6. Override the Menu Page of the user interface and add menu items for each operation. Associate the method created in the step 5 to the Click Method property of the menu item.
  7. At runtime, a success message is displayed when you select the Delete menu.

    8. Note:

    The node in the System Configuration application has to be configured. Client port's location must be http://localhost:8080/cwf/api/testHTTP-ServerInterface/. Make sure Client and Server ports are active. In the Bindings tab, the OpDelete, OpHead, OpGet, OpPut, and OpPost operation's location must be set to the name of the server operation, RequestResponse + a custom URL. Make sure the listener's thread number is set to 1.

    For the GET, HEAD, PUT, DELETE, and POST, the server will return the name of the HTTP method invoked by the client and the client will verify the result. In the result message, the server will also send a string obtained from the concatenation of the path elements configured as custom URL in the bindings tab of the node configuration. The client will verify that this string is not empty. The purpose is to ensure that the server was able to access the custom URL path.
