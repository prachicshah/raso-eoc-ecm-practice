Glossary (A to M)

The following table represents ths glossary terms from letter A to M:

Term Definition
Account Account consists of account data and can have one or more instances of account contacts and contact media.
Account number A system-generated number that is linked to an account.
Account contact An account contact is identical in structure to a customer contact and contains contact information about parties associated with an account.
Activity A component that performs a specific function within the process, such as invoking a service or another process. Activities can be as simple as sending or receiving a message or as complex as coordinating the execution of other processes and activities.
Adapter Licensed software that supports the interfacing to a specific protocol or vendor systems.  Generally industry vendor specific adaptors are provided.
Alert A specific task type which notifies a user or system of some event in the process. The alert may take the form of a task which requires acknowledgement by a user; it can also be an e-mail or SMS message which (require no acknowledgement).

AM

Accounting Management is referred to as billing management.

App

Application Layer – client software or system that interface to Business Management Layer API or interface.
Application An application is defined as metadata configured to requirements using our products. Software that operates on the AVM runtime and possibly in conjunction with Modules.
Application Program Interface (API) A formalized set of software calls and routines that can be referenced by an application program to access supporting network services.
It is an interface between the application software and the application platform, across which all services are provided.
Application Virtual Machine (AVM) or Platform  AVM is the run-time engine for all metadata. The AVM provides common services to the metadata based modules including a UI server, workflow engine, integration framework, rules engine, user authentication and authorization, and data persistence services.  The AVM is the runtime framework of all of our applications.
Assigned or Unassigned Describes if a task has been assigned to a user, or not.
Association Types Association types define all characteristics of an item, for example, content, structure, business rules, business name, image, and specification to name a few. Association types can be created within the Catalog to define items with any level of details and any complexity of rules.
Atomic Activity An elementary unit of work that cannot be further decomposed. Atomic activities can be used to start other processes, perform calculations, or perform operations.
Attribute Type Attribute Type defines a basic element or characteristic of any component or any product.
Availability rules These rules filter products based on geographical location, customer and account profile information (for example, account category). Attributes in the catalog define if the product is visible to CSRs during the ordering process.

BML

Business Management Layer performs functions related to business aspects, analyzes trends and quality issues.
Catalog Hierarchies Catalog hierarchies are directory structures for organizing products and offers.

CE Router

Customer Edge (CE) routers are the routers located at the customer premises.  These routers interface with the PE routers and use some form of routing exchange, either static or dynamic, including such protocols as RIP, IGRP, EIGRP, OSPF, IS-IS and BGP.  The interconnection between a CE and a PE router can happen over any Layer 2 link.
Channel This term denotes the channel in terms for frequency for example, wireless channel, TV channel. This term also refers to user interface delivery channel, such as Web, mobile, TV, IVR, etc.
Charge Types Charge types are the formulas and price tables within the Catalog.

CM

Configuration Management is responsible for:

  1. Gathering and store configurations from network devices
  2. Simplifying the configuration of device
  3. Track changes that are made to the configuration
  4. Configure (provision) circuits or paths through the network
  5. Plan for future expansion and scaling
Communication Service Provider (CSP) A CSP is defined as a service provider that transports information electronically. The term refers to the companies that provides telecom (landline and wireless), Internet, cable, satellite, and managed services. Communication providers can even support end-to-end communications services that businesses and carriers need. The service provider is  also known as operator, telecom operator, carrier, MSO, ect.
Configuration Environment The design-time environment which is used to configure any solution or applications (such as Catalog ).
Contexts Contexts are variables that are configurable in the catalog to define user specific environmental variables (such as location or market segment).
CPO Convergent Postpaid.
CPR Convergent Prepaid.
CRM Customer Relationship Management.
CSR Customer Service Representative.
Customer Information System (CIS) A CIS manages account, consumption, and billing data. It provides a single solution for tracking all interactions with your customers.
Customer Lifecycle Management (CLM) CLM denotes to the sum of experiences a CSP has over the duration of its relationship with its customer.
Data Reference Type Data reference type is a data type that enables a metadata object to reference a document or an order, as well as defining valid choices to select as the reference through a Finder.
DB  Database
DBA Database administrator
DBoR Database of Record
Decision Tree Decision Tree is a new metadata object that encapsulates complex logic graphically using BPML (Business Process Modeling Language) model, as well as executing at runtime. It is similar to business process metadata, but decision trees may be executed, with only instantaneous life-span, and in any AVM rather than enclosed in dedicated Process Engine. Decision tree is ideal to modeling complex decision logic, facilitating a divide-and-conquer approach with benefits of increased readability and decreased maintenance effort.
Derivation Chain Chain of derived or extended metadata objects in project. For example, md2 extends md1, md3 extends md2, etc. Going up the derivation chain means going from an extended element to its base element.
Developed Software Developed software means metadata development or configuration or, other software developed by our company or partners specifically for customer in connection with an implementation of the licensed software. For greater certainty, developed software does not include any licensed software.
Documentation Documentation means all documentation (including, without limitation, applicable Software terms, administration guides, user guides, training guides, and other instructional materials) related to licensed materials, which may be licensed or otherwise provided to customer under the terms of this agreement, as such documentation is amended and updated from time to time.
DPL Deployed
DSL Digital Subscriber Line

EML

Element Management Layer contains functions for the handling of individual network elements. This includes alarm management, handling of information, backup, logging, and maintenance of hardware.
Enterprise Content Delivery Service (ECDS) ECDS stands for the technology to support mobile streaming service
eTOM Enhanced Telecom Operations Map
Exception Handling Defines activities that respond to an unexpected event. An exception event could be a time-out or an exception activity.

FM

Fault Management is responsible for recognizing, isolating, correcting, and logging faults that occur in the network.
Forms Forms define the presentation fragments of a Web page. Forms are always contained within a UI and typically define the presentation for metadata object included in the UI .
Global Scripts The scripts defined in script metadata object. They may have user defined parameters (including no parameters at all).
Google Web Toolkit (GWT) GWT is a development toolkit for building and optimizing complex browser-based applications. Refer to the GWT website for more information.
HSI High Speed Internet
HSIA This term stands for High Speed Internet Access (broadband - used for HIS and/or IPTV)
ICCID The external identifier for a SIM, printed on the card. See also IMSI.
ID Identifier
IDE Interactive Development Environment
IMEI International Mobile Equipment Identification. The ID for a handset or other mobile device.
IMSI
  1. International mobile Subscriber Identity
  2. The internal identifier that identifies a SIM in the network.  See also ICCID.
IPTV Internet Protocol Television
JConsole JConsole is a JMX-compliant monitoring tool, which is provided as part of Sun's Java SDK.
JDK Java Development Kit
JMX Java™ Management Extensions (JMX) is a Java technology that supplies tools for managing and monitoring applications, system objects, devices, and more. 
Licensed Software

Licensed Software means the machine-readable executable code version of the applicable specific software program(s) identified in the sale contract, and for which licenses are granted under an Agreement, including, without limitation:

  1. Any modifications or additions to such Licensed Software (including enhancements, updates, upgrades, modifications, maintenance releases, bug-fixes or workarounds, interfaces and templates) supplied to Customer pursuant to an Agreement.

  2. All open source software, freeware or other third party software or materials incorporated or embedded in the Licensed Software.
Lifecycle Management Nomenclature that represents Customer, Product or Order lifecycle methodologies.  Primarily defined by the industry.  
Line of Business (LOB) Line of Business, is an internal division to the CSP which may or may not be aligned with the external Market Segments. It has been referred to the services the CSP offers, such as Voice (Local, Long Distance), Mobile, Data, Internet, Public Phones.
Market Segment

Market Segments, for CSPS, are usually categorized as Residential, SOHO, Commercial, and Wholesale.

  • Residential – Telecom service providers deliver a range of residential services, from traditional voice services to competitive video and IPTV services to new connected home capabilities (including home security, energy management, networked health care services, and more).
  • Commercial – Business (Small, Medium, and Enterprise)
  • Wholesale - MVNE or reseller
MCN Missed Call Notification, a VAS service with its own device
MIB Management Information Base
Milestone The ‘Expected Completion Time’ for a Process Activity, or a Sequence of Process Activities
MMS Multimedia Messaging Service
MMSC Multimedia Messaging Service Centre, a server for MMS. 
Modules  Licensed Software representing specific out-of-box functionality.
MoIP Mobile communications over Internet Protocol.
MSISDN 1. Mobile Station International Subscriber Directory Number
2. Mobile Subscriber Integrated Services Digital Network Number. The mobile telephone number. 

MTU-s

A switch that has bridging capabilities but does not have routing capabilities.  This switch represents the access layer of the H-VPLS architecture.  The MTU-s device establishes Pseudo wires to one or two PE-rs routers through which VPLS traffic is forwarded.
