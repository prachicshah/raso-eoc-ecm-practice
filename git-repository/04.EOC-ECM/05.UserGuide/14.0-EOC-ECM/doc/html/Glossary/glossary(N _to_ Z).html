Glossary (N to Z)

The following table represents the glossary terms from letter N to Z:

Term Definition
Navigation Tree Navigation Tree is the metadata object type for presentation purposes that can model nodes of a tree-structure, which can be presented and navigated by the user in a Tree Form Element by default.
Navigation Bar Navigation Bar is one of the metadata objects available under the top-level Presentation item. It displays a series of steps contained within an procedure.
Network The CSP network to deliver services to customers, such as Cable, Copper, Ethernet, Satellite, Wireless, Fiber, etc.
NGCP Next Generation Control Point

NML

Network Management Layer performs functions for distribution of network resources, configuration, control, and supervision of network.
Object Scripts The scripts defined at metadata object level such as Documents, Orders, Data Structures, etc.

OCh

One-click hosting, describes web services that allow internet users to easily upload one or more files from their hard drives(or from a remote location) onto the one-click host's server free of charge.

ODUk

Optical Data Unit

OMS

Optical Multiplex Section

ONE

Optical Network Elements

Operation A specific type of Process Activity which requires interaction with an Interface. Operations are either Synchronous or Asynchronous.

OPUk

Optical Payload Unit

Order The central data object.
Order Instance In the database each Order is represented as an Order Instance Document. The Order Instance Document can be extended and overridden by the user's application, if necessary (in the Library tab, Order Entry Product namespace).
Order Lifecycle Management (OLM) Order Lifecycle Management
OSS Operational Support System
OTA Over the air update (for example, of handset software)

OTN

Optical Transport Network

OTUk

Optical Transport Unit refers to a set of Optical Network Elements (ONE) connected by optical fiber links, able to provide functionality of transport, multiplexing, switching, management, supervision, and survivability of optical channels carrying client signals.
Page A Page is a top-level Form that can be requested and served to a Web browser.
Party A party, as an entity of the customer, is either an Organization or an Individual. A party may be known by one or more names and have multiple forms of identification, birth certificate, driver's license, etc. Most importantly, a party may assume multiple roles. The role may be a customer, account manager, sales representative, customer contact, etc. 
PCR Project Change Request
PCS Policy Control Server. PCS policy modifies data service (such as Boost on Demand for a specific time period).
PDE Plug-in Development Environment
PE Process Engine
PoS Point of Service
POTS Plain Old Telephone Service
Privilege A user right which is assigned to a user and that specifies allowable actions in the system.
Process The running entity which orchestrates tasks and operations in sequence or in parallel.
Process Activity The granular components of a Process (switch, case sequence, etc.) which guide the direction of the process.
Process Priority Describes if the Process has been flagged as Regular, High, or Low priority.
Process State or Status This term refers to the state of the process. For example, Executing, In Error, Suspended, Terminated, Cancelled, etc. 
Process Order The Order instance that a process is working on. Each process instance can be associated with only one Order instance.
Process Revision An archived, read-only version of a process metadata object that was then current process metadata before changes were made to it and revisioned. Among other purposes, it provides the accurate snapshot of the process definition for in-flight process instances (that is, process instances that have started but not completed) to continue execution upon process metadata change. The terminology may also refer to the process revision number of the process metadata object, which keeps track of the numbering in the latest revision.
Product Products offered to CSP customers, along with offers, bundles, etc.
Product Lifecycle Management (PLM) PLM defines the process of managing the product catalog and the items within catalog from cradle to grave. This process includes the following sub-processes: conceptualize, design, develop and test, deploy, activate, operate, and retire.
Professional Services Professional Services denotes to the work or tasks to be performed related to development of Developed Software, installation, implementation, configuration consulting, training and other professional services, pursuant to the terms and conditions of an Agreement or applicable Statement of Work.
Projects Projects are used to control changes to the catalog definition and to release the changes to a live production environment in a controlled manner.
PSTN Public Switched Telephone Network
QA Quality Assurance
RBT Ring Back Tone, a VAS service with its own device
RTE Real-time Environment
SaaS Software as a Service
Service
  • Any service provided by a telecommunication provider, such as wireless, TV, voice, VAS, SMS, etc.

  • A collection of functionality described by one or more contracts. The service represents the unit of manageability.

  • Services are developed by a Service Provider for sale within Products. The same service may be included in multiple products, packaged differently, with different pricing, etc.

  • A telecommunication service is a set of independent functions that are an integral part of one or more business processes. This functional set consists of the hardware and software components as well as the underlying communications medium. The Customer sees all of these components as an amalgamated unit.

  • Result generated by activities at the interface between the supplier and the customer and by supplier internal activities, to meet customer needs; the supplier or the customer may be represented at the interface by personnel or equipment.
Shared session A browser session that occurs if you open a second browser window by selecting File > New from your current browser session, or if you use shortcuts to access the Application URL that do not require you to log in to each instance separately. If you have multiple windows open, but you logged in only once, then you are sharing sessions.
SIM This term refers to Subscriber Identity Module. The small smart card required in a GSM mobile phone or other GSM device.
SMS Short Message Service.
SMSC Short Message Service Centre, a server for SMS. Adding SMS requires updating the SMSC and HLR.
SOAP Simple Object Access Protocol - a protocol for XML over HTTP
Solution An end-to-end solution tailored for a given industry or market segment.  It can support one or more Products, Modules, or Applications
SOW Statement Of Work
SP Service Pack
SQL Structured Query Language
Subscriber

In a public switched telecommunications network, the ultimate user, such as, customer of a communications service.

  1. Subscribers include individuals, activities, organizations, etc.

  2. Subscribers use end instruments, such as telephones, modems, facsimile machines, computers, and remote terminals, which are connected to a central office.

  3. Subscribers are usually subject to tariff.

  4. Subscribers do not include communications systems operating personnel except for their personal terminals.
System Scripts or System Methods The object scripts that perform predefined methods such as document load scrip". These scripts are called automatically by the AVM at predefined points of the application execution. They have predefined fixed number of parameters.
TAM Telecom Application Map
Tax Model An ordered set of tax types and discounts. Allows the CSP to set up various tax situations within a catalog.
TCO Total cost of ownership
Template A Template is a unique and proven metadata configuration defined in a Namespace within the Library tab of Configuration Environment Tool that can be re-used for a customer application.
Metadata templates are used in the customer application development process to shorten the development cycle, improve quality of the application and provide a robust metadata foundation for customer extensions and customization.
Templates can be built by the company, OEM partners (such as Telcordia, HP, IBM), or customers. Templates can be developed for recurring business applications, product and service types, and interfaces.
Third Party Software Third Party Software means the third party software in conjunction with which the software operates, which customer is required to obtain and install prior to installation of the Licensed Software.
TMF TeleManagement Forum. Now known as TM Forum.
UAT User Acceptance Test or User Acceptance Testing
UC Use Case
UI User Interface
User ID The User ID of an Application end-user.
User Scripts or User Methods The Object Scripts that are not predefined system scripts. They may have user defined parameters (including no parameters at all). 
USSD This term refers to Unstructured Supplementary Service Data. A subscriber can use USSD or IVVR to submit orders from a handset.
VAS Value Added Service
Velocity Studio (VS) Velocity Studio (VS) is a design time component used to configure a new or modify an existing solution.  Velocity Studio is used to model the workflow tasks/activities, configure CSP-specific business rules, establish system interfaces and generate the GUI screens required by participants in the order fulfillment process.
VMS Voice mail service means a subset of VVMS with different COS (Class of Service).
VoIP Voice over Internet Protocol
Work Item A Unit of work (Task), assigned to be done by a User or by an External System.
Workgroup Entity defining a group of Users assigned a fixed set of Application Privileges.
WSDL Web Services Description Language.  XML-based description for web services. 
XML Extensible Markup Language
