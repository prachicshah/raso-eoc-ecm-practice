Hardware Sizing Recommendations

Several methodologies can be used to size a system. Ericsson uses a semi-empirical approach using the data from performance test results, analytical calculations based on input parameters requested from the operator (such as, number of services created or modified, working hours per day, concurrent customer care users,and so on.), and the processing capabilities of the hardware platform. Due to the flexible nature of the solution, all parameters influencing the performance cannot be considered. Therefore, Ericsson uses the experience from other similar solution systems in production to come to an appropriate sizing. So, the sizing is also based on an average product usage. There might always be additional parameters which can influence the performance in an unforeseen way. Therefore the sizing might be revised during the implementation stage and also when the influence of design changes becomes measurable.You must check if the assumptions made by Ericsson are correct and if there are additional conditions and unusual practices which might influence the performance of the overall system.

Ericsson provides some hardware recommendation guidelines required to meet the proposed solution. The factors affecting the proposed hardware configuration include:

  • Real-time demand for order creation (number of orders).
  • Number of simultaneous users.
  • Database access requirements.
  • Number of processes and associated activities.
  • High-availability requirements.

Specifically, the following metrics are collected to prepare the hardware sizing recommendation:

  • Concurrent users
  • Orders per month
  • Business days per week
  • Hours per work day
  • Transactions per order
  • Number of busy hours per day
  • Expected order life cycle duration (by type of order)
  • Transactions per hour within a busy hour
  • Transactions per second within a busy hour

There are different hardware requirements for a system depending on whether the system is in a development environment, testing environment, or production environment. A development environment has lower hardware requirements than the testing or production environments.

These recommendations provide certain guidelines about the required hardware, such as the number of servers required, the memory required, and so on, it does not recommend a specific hardware. The decisions about the specific hardware to be used has to be made by you, based upon many additional factors, such as the availability of existing servers, pricing, and standard operating platforms. Based on a few particular metrics, the hardware sizing guidelines are provided for the following types of solutions, for three kinds of environments.

  • Low Volume Solution
  • Medium Volume Solution
  • High Volume Solution
