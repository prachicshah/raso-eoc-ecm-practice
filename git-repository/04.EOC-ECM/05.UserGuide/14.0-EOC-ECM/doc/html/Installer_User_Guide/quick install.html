Quick Install (for Experienced Users)

Note: This chapter is a quick checklist of all installation steps. It is not intended for first-time users. For first-time users we recommend reading in detail every chapter of this document to attain a full understanding of the installation process.

Pre-Requisites

  1. Install J2SDK
  2. Install Oracle Database
  3. Install Revision Control System (optional)
  4. Install Application Server (optional)

Product Install Wizard

  1. Run the Product Installation Wizard (for example, \Disk1\InstData\Windows\NoVM\install.exe).
  2. Take extra care to select the proper JDK (and not a JRE). Use Choose Another if you need to.

Create Database User and run (DDL) Initialization Script

  1. Create a database user (for example, CW).

    2. Note: See Database Initialization for roles and privileges).

  2. Open <installation_folder>\DDL\CW.sql in a text editor (for example, WordPad).
  3. Edit the DEFINE statement to point to your schema user.
  4. Log into your schema and run the file CW.sql.

Set Bootstrap Database Connection in Velocity Studio

  1. Open the Velocity Studio from your start menu.
  2. Click File > New > New Project from the menu bar to create a project.
  3. Enter the internal name for the project.
  4. Click Database > Connect from the menu bar.
  5. Click the New button to create a new connection.
    • Enter the name of the connection in the Name field. You can enter the name with the following convention: CW_SCHEMA@HOST_SID (for example, CW@CW0016_LOCALDB).
    • Enter the database schema username in the User field (for example, CW).
  6. From the Connection Properties dialog, click the New button to create a new database driver.
    • Indicate the database Host, Port and SID and click the OK button.
  7. Click the Test button to test the connection.
  8. Click the OK button to save the connection properties and to close the New Connection dialog box.
  9. Type your password to login to that schema.
  10. Select Runtime > Run from the menu bar to run the framework in Config mode.
  11. Select the application name, version, and node from the Select Application dialog.

Configure Your Project in the System Configuration Application

  1. Login to System Configuration application by accessing the http://<host_name>:<port>/cwf/config (for example, http://localhost:8080/cwf/config ) URL through your Web browser.
  2. Login using upadmin/upadmin.
  3. Click the Database tab, and then click the Physical Connections tab.
  4. The New Physical Connection dialog appears. Enter the database credentials; when you have finished, click the Apply button. A message appears, indicating that you have successfully updated your Oracle thin connection.

    Note: The database attributes must be same as the database attributes set in the bootstrap database connection of Velocity Studio.

  5. Click the Logical Connection tab to associate your logical connections to the physical connection you have just created, double-click each of the following logical connections and select your newly created physical connection from the drop-down list:
    • ARCHIVE
    • ORDER
    • PE_ORDER
  6. Click the Save button to save the configuration settings.

Note: At this point your installation is complete. The following section is optional and is useful if you want to verify your installation.

Log into the System Administration Application

  1. In Velocity Studio, select Runtime > Run from the menu bar.
    • If you have just completed configuration with the system configuration application, a system restart is necessary to reload the configuration data. Stop the run before restarting by clicking Runtime > Stop.
  2. Login to Administration Application
    • Open a web browser and load URL http://localhost:8080/cwf/login.
    • Login to the application as upadmin/upadmin.
  3. Double-click the System Administration application option.
