Definitions

The following are definitions for terms used in this guide:

  • Global Scripts: The scripts defined in Script metadata object. They may have user defined parameters (including no parameters at all).
  • Object Methods or Object Scripts: The scripts defined at metadata object level such as Documents, Orders, Data Structures, etc.
    • System Methods or System Scripts: The Object Scripts that perform predefined methods such as "document load script", and are called automatically by the AVM at predefined points of the application execution. They have predefined fixed number of parameters.
    • User Methods or User Scripts: The Object Scripts that are not predefined system scripts. They may have user defined parameters (including no parameters at all).
  • Component Methods or Component Scripts: The scripts defined on the components of the metadata objects such as Document Variables of Order Items. Their names and parameters can be seen in Velocity Studio.
  • Derivation Chain: Chain of derived or extended metadata objects in project. For example, md2 extends md1, md3 extends md2, etc. Going "up" the derivation chain means going from an extended element to its base element.
