Using JConsole

Starting JConsole

Launch JConsole by executing jconsole.exe from your J2SDK\bin folder. Upon Launch you will be presented with a list of Agents or given the option to connect to a remote host.

All process engines broadcast on a designated port which may be specified at runtime via a switch on the Process Engine (see Process Engine Configuration Guide for more details).

Statistics Monitoring in JConsole

Once in the JConsole application, go to the MBeans tab, Operations tab and proceed to unfold the com.conceptwave.AVM class which reveals the Caches, Queues, and Performance monitoring properties.

The values of these variables can be adjusted and refreshed on the fly. In the case of SM Monitoring, a new Monitor can be created by simply copy/pasting the SM Spec string and clicking the launch button. Please refer to Configuration User Guide, Performance Monitoring for more information on creating an SM Spec string.

Cache Configurations in JConsole

The cache management domain is com.conceptwave.AVM and the type is DataCaches. Each cache is a separate object with a corresponding name. The management interface includes the following attributes:

  1. CacheName - provides the cache name (read-only)
  2. CacheElementsCount - the number of elements in the cache (read-only)
  3. CacheSizeInBytes - the size of the memory currently used by all cache elements (read-only)
  4. CacheHitPercentage - the percentage of the direct hits in the cache, averaged from the beginning of its use (read-only)
  5. CacheRequestsPerMinute - the number of the requests per minute, averaged from the beginning of its use (read-only)
  6. CacheCapacityInMB - the cache capacity in megabytes (read-write)
  7. CacheRefreshSchedule - the refresh schedule string (read-write)

The cache objects expose one operation: cleanTheCache which immediately refreshes the cache (independent of the schedule). Note: Use this operation carefully and only when needed because it can reduce performance significantly for short period of time.
