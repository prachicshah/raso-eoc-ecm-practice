Introduction to JMX

Java™ Management Extensions (JMX) is a Java technology that supplies tools for managing and monitoring applications, system objects, devices, and more. Those resources are represented by objects called MBeans (Managed Beans), are objects that represent these resources. In the API, classes can be dynamically loaded and instantiated.

This section contains the following information:

Profiling with JConsole
JConsole is a JMX-compliant monitoring tool, which is provided as part of Sun's Java SDK. With JConsole, you can monitor JVMs extensively, such as the runtime deployed in any J2EE container with JMX arguments enabled.

Shut down the PE
Process engines can be gracefully shutdown by simply stopping their service at their hosted J2EE application servers. As an alternative approach, you can gracefully shut down a process engine through JConsole.

Monitor and Manage MBeans in JConsole
The MBeans tab allows you to view, monitor, and manage information on all registered MBeans on your server. Specifically, this tab allows you to view, change, and monitor the com.conceptwave.AVM class, which reveals caching, queuing, and performance monitoring properties.

You can access the following domains using the left tree navigation in JConsole. Additionally, a reference for JConsole MBeans is available.

  • AVM caches
  • AVM
  • Process engine queues
  • Process engine working threads
  • Verbose
  • Statistics Monitoring
  • Diagnose problematic scripts using Set Max Execution Time
  • JConsole MBeans reference
