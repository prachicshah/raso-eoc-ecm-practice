Introduction to Deployment

This deployment guide details the planning and procedure of deploying Application metadata into a deployment environment with J2EE Application Servers.

Because each deployment environment is different ranging from system-load expectations to corporate IT policies, you are expected to use this guide to understand the product's deployment mechanics and capabilities, and strategize your own deployment procedure that is tailored for your deployment environment.

Note: During metadata development and functional testing, there is no need to deploy metadata. Simply run the metadata within Velocity Studio.

Deployment Overview

In an enterprise setting, metadata deployment would span across multiple environments. For the convenience of our discussion, we denote the following:

  • Engineering Environment: the environment where engineering/R&D team develops and tests their application metadata.
  • Packaging Environment: the environment where metadata is retrieved from source format and packaged into WAR (Web application ARchive) file. It may also be the environment to prepare for the deployment environment before deployment takes place.
  • Deployment Environment: the environment where the application is deployed

At a high level, there are several items that require your attention when deploying an application:

  • Metadata
  • Configuration
  • Database schema
  • Processes

Each of these aspects are addressed in this deployment guide.
