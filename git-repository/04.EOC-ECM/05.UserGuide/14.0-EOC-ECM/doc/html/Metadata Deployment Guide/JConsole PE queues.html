Process Engine Queues Domain

The PEQueues domain allows you to view, manage, and change a number of attributes, including Queue size and Waiting intervals. It also provides you with operations that you can use to manage your process engine queues, such as the Clean the participant cache and Show Waiting Activities operations.

Attributes

Expanding a process engine queue's Attributes node in the MBeans tree provides you with attributes that you can view, manage, and change. As an example, clicking the Waiting intervals attribute within Participant Queue displays pertinent information about it, including the attribute's description, whether it is readable or writeable, and its data type.

When you hover your mouse pointer over the Refresh button, a tooltip indicates the action that occurs after you click the button. In this example, clicking the Refresh button refreshes the attribute Value field. The Value field is blue, which allows you to edit this field and then refresh the attribute.

Note: When the cache size is too small for global requests and participant queues, an error is now logged, including the number of items in memory and time of the last overflow.

Operations

Expanding a process engine queue's Operations node in the MBeans tree provides you with operations that you can use to manage your queues. As an example, clicking the Show Waiting Activities operation within Participant Queue displays pertinent information about the operation, including its description, return type, data type, and any parameters that the operation has.

To invoke an operation, click the button in the right pane that is associated with the command you want to use. In this example, click the Show Waiting Activities button to invoke the command.

Restart a New User Process

You can restart new user processes from that JMX console that may have been skipped for some reason when starting up the process engine or a standby PE.

To restart a new user process, do the following:

  1. Start the JMX console.
  2. Connect to your destination PE.
  3. Click the MBeans tab, and then click com.conceptwave.AVM > PEQueues > NewProcesses Queue > Operations.
  4. Click the Restart button.

Analyze Global Requests Queue Content

To analyze the Global Requests queue content, a Dump button is available in JConsole.

Clicking this button creates a globalRequests.txt file in your Velocity Studio's <installation_folder>\designer\env folder, which saves all queue content and allows you to analyze it when troubleshooting.
