Running multiple Velocity Studio instances on a single machine

It is possible to run multiple Velocity Studio instances on a single machine. However, the implementation is dependent on whether the Process Engine is also implemented. Each implementation requires changes to be made to the startDesigner.cmd file located in the following directory <installation directory>\designer\env. The startDesigner.cmd file contains the following default JMX settings: 

com.sun.management.jmxremote.port=10008
com.sun.management.jmxremote.authenticate=false
com.sun.management.jmxremote.ssl=false

To launch multiple Velocity Studio instances, you must manually change the JMX_PORT setting in the startDesigner.cmd file each time. Otherwise, an error displays, indicating that there is a port conflict.

With Process Engine

If the Process Engine is being implemented within Velocity Studio, then copies of the starDesigner.cmd file is required for each instance of Velocity Studio to be run. For each instance of Velocity Studio that is being run, change the value of the com.sun.management.jmxremote.port parameter such that each instance has a unique port number.

Without Process Engine

If the Process Engine is not being implemented within Velocity Studio, then simply remove the JMX_OPTS from the startDesigner.cmd file.
