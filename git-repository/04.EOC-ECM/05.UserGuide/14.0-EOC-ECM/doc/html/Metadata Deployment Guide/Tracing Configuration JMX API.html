Tracing configuration JMX API

The JMX API supports the following commands:

JMX Method Description Parameters Returns
addTraceInterface Adds or activates an interface for public activation.
  • interface
    The interface name defined by the application.

Returns void.

addTraceObject Adds a trace object to an existing trace job. When matching a trace request, trace objects are added using the AND operation.
  • traceId
    The trace identifier of the job.

  • traceType
    The trace field type that requires matching.

  • traceValue
    The trace value to match.

  • matchType
    The field matching type.

Returns true if the trace object is successfully added. Otherwise, false is returned.
createTraceJob Creates a new trace job.
  • interface
    The trace activation interface.

  • start
    The trace period start representing when the trace job is valid.

  • end
    Trace period end representing when the trace job is valid.

  • coverage
    Trace coverage, which takes either the LOCAL or GLOBAL value.
 Trace job ID that the AVM generates and returns.
deleteTraceJob Deletes an existing trace job.
  • traceId
    The trace identifier of the job.
 Returns true if the trace job is successfully deleted. Otherwise, false is returned if the trace job does not exist.
listInterfaces Returns a list of public activation interfaces. No parameters.

Returns a string array of interface names that have been activated locally.

listTraceJob Displays or returns a list of existing trace jobs with their respective parameters. No parameters.

Returns a string array containing trace job details. For a given trace job with index n, its details are found in the n to (n + 4) elements of the array.

For each job, the following values are returned:

  • Trace job ID
  • Interface
  • Period start
  • Period end
  • Coverage
listTraceObjects Displays a list of trace objects for an existing trace job.
  • traceId
    The trace identifier of the job.

Returns a string array containing trace job details. For a given trace job with index n, its details are found in the n to (n + 2) elements of the array.
For each object, the following values are returned:

  • Trace type
  • Trace value
  • Match type
removeTraceObject Removes a trace object from a job.
  • traceId
    The trace identifier of the job.

  • traceType
    The trace field type.
 Returns true if the trace object is successfully removed. Otherwise, false is returned.
UpdateTraceJob Updates a list of trace job objects for an existing trace job.
  • traceId
    The trace identifier of the job.

  • interface
    The trace activation interface.

  • start
    The trace period start representing when the trace job is valid.

  • end
    Trace period end representing when the trace job is valid.

  • coverage
    Trace coverage, which takes either the LOCAL or GLOBAL value.
If the trace job cannot be found by its trace ID, this operation does not perform any action. If the trace job is found, the corresponding trace objects that belong to this trace jobs are removed.

For more information, see the JavaScript documentation.
