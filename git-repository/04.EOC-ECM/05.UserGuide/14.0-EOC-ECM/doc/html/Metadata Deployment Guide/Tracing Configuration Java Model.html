Tracing Configuration Java Model

The tracing configuration model is a part of the AVM configuration model, but is only available through the command-line interface. The tracing configuration model appears as a field to the CwfSystemConfigModel class.

The following table shows the model classes, their fields, and their descriptions:

Model Class Field Description
CwfTraceModel traceFileLocation A String that represents the physical path for trace files.
traceJobs List of CwfTraceJobModel
activatedInterfaces
  • List of String that represents the names of activated interfaces.
  • The interface name is not equal to metadata interfaces since applications can define trace points anywhere.
  • For an interface where trace ID activation is allowed, if it is not included in this list, it is assumed that trace ID activation is disallowed for the given interface. Public-facing interfaces are likely to be configured to not allow tracing. Internal interfaces are likely configured to allow trace ID activation.
CwfTraceJobModel ida

System-generated ID.
interface A String that is domain-specific.
coverage This field takes either LOCAL or GLOBAL as values.
start DateTime
end DateTime
traceObjects List of CwfTraceObjectModel
CwfTraceObjectModel traceType A String, which represents the trace type and is domain-specific.
matchType This field takes one of the following values:
  • BEGINS_WITH
  • ENDS_WITH
  • CONTAINS
  • EQUAL

For more information, see the JavaScript documentation.
