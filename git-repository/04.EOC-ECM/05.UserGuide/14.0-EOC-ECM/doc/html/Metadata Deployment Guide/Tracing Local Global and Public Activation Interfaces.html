Local versus Global Tracing, and Public Activation Interfaces

Local trace jobs are jobs that only impact tracing in the local AVM instance. If a message is received that matches a trace job’s criteria, tracing occurs within the application component. However, the trace ID is not expected to be propagated when the message is forwarded. As a result, tracing is not triggered in other application components.

With global tracing, it is expected that the trace ID is propagated with messages to other application components and possibly trigger tracing in them. The management server defines global trace jobs.

Public activation interfaces (PAI) are interfaces where receipt of a message with a trace ID triggers both the message being recorded and trace ID propagation. These interfaces are intended to be public-facing interfaces.

Trace Log File

Each trace job has one log file generated. The log filename is based on both the trace interface and the trace job ID.

When trace jobs are either deleted or created by the command-line interface, the AVM logback configuration must be reloaded.
