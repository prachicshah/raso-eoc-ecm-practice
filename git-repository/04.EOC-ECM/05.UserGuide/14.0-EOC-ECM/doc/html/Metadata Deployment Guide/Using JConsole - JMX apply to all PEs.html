Apply JMX Commands to all Running PEs

To apply JMX commands to all running process engines (PEs) in a single request, the Apply to all PE attribute is available from JConsole by clicking com.conceptWave.AVM > Verbose > Attributes > Apply to all PE in the left pane.

By default, the Apply to all PE attribute is set to false, meaning that attribute changes apply to attributes on the current PE only and you work with Verbose attributes as you did before.

However, when the Apply to all PE attribute is set to true, the next command to change another attribute's value applies to this attribute on all running PEs. Once this attribute is updated on all running PEs, the Apply to all PE attribute is set to false, ensuring that you specify this flag each time it is required and does not accidentally change other attributes on all PEs.

The following are examples on how the Apply to all PE attribute value affects each PE's verbose mask value.

Example 1:

  1. Two PEs, PE1 and PE2, are running.
  2. PE1 has the verbose mask attribute set to 70. PE2 has verbose mask set to 70.
  3. You are working with the connection to PE1 and the Apply to all PE attribute is set to false.
  4. You change the value of verbose mask to 80.
  5. The results are that PE1 has verbose mask set to 80 and PE2 has verbose attribute set to 70.
  6. The Apply to all PE attribute is set to false.

Example 2:

  1. Two PEs, PE1 and PE2, are running.
  2. PE1 has the verbose mask attribute set to 80. PE2 has verbose mask set to 70.
  3. You are working with the connection to PE1 and the Apply to all PE attribute is set to true.
  4. You change the value of verbose mask to 50.
  5. The results are that PE1 has verbose mask set to 50 and PE2 has the same attribute set to 50.
  6. The Apply to all PE attribute is set to false.
