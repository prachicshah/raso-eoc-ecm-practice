Process General Properties

Clicking a process from the metadata tree displays its general process properties, which are defined on the General tab.

The fields visible on the tab change, depending on the Type field selection (that is, Global, Subflow, or User).

The General tab has the following fields, depending on the Type field selected:

Field Description Type
Name Mandatory. Name of the Process within the namespace. All
Private If checked, the metadata object is hidden from display when the metadata is packaged as Product Metadata (that is, library). All
Restricted If checked, all JavaScripts contained within the object are blocked from being displayed when the metadata is packaged as Product Metadata (that is, library). All
Deprecated If checked, the metadata object is designated for archival purposes only. The object becomes hidden from all metadata object selection lists. All
Label Mandatory. Label to identify the Process. All
Description Description of the Process for documentation purposes. All
Type Mandatory; not changeable upon metadata object creation. The Process type. The values include Global and User. All
Document The  Process Document. If specified, an instance of the Document is created and assigned to each process instance. The process instance can store instance-specific data in this Document. All
Store in a separate table

If checked, the Process Document is stored in a separate database table. Otherwise, an unmapped process document is stored with the process.

User, Global
Store as a separate document Available only when Process Document is assigned. If checked, the Process Document is stored in a separate database table, instead of as NCLOB in XML format in process table. Storing as a separate document enables readable access of the document to application metadata (for example, you may create report over this data); in addition, enabling this flag may have slight, but visible performance improvements. All
In error restart from last running activities By default, this property's checkbox is unchecked, meaning that the process engine restarts the process that is in an error state from the first activity. When this property is checked, the process engine restarts this global process in an error state from the last running activity when the error occurred. Global
Priority The user can assign the Process priority. The default values include High, Normal, and Low if no customization is performed on cwf.allPriorities. All
Interface This field associates the global process with an interface so that messages can be sent to the global process. Set the Interface field before adding operation activities to the workflow for the operation to treat the specified interface as a default global participant. Operation activities are not automatically updated after changing an interface. Global
Revision Read-only. This is the revision number of the Process. The latest Process Revision of this Process is at this revision number.
  • Revision number is set to 1 for a newly created process.
  • Revision number is incremented by 1 every time a new Process Revision is created, except when this is a newly created Process that is without Process Revisions.
 All
Stale Alert At the process level for a user process, you can specify a metadata-defined alert to be issued in the event of a stale dated process. Such an alert can be configured to go to e-mail, interface, or a worklist. User
Stale Exception At the process level, a checkbox on the General tab for a user process that indicates whether a system-defined exception is thrown into a stale-dated process. User
Memory Only Select this checkbox to indicate that it is a memory-based process. Memory-based processes do not store their state. If they get an error or the process engine is restarted, these processes start from the beginning. User, Global
