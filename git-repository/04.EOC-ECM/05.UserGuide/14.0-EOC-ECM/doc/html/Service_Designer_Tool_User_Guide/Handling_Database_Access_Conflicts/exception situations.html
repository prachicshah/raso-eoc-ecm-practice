Exception Situations

Read-only States

If the Order is in a read-only state it rejects all operations that are done on its Documents. The read-only state can be changed using the script order._state = notation.

Revoked Tasks

Revoked tasks are processed in UI mode only and only if the Order parent is a work list item. The current user ID is compared to the assigned user field in the worklist to decide if the operation is valid. If the operation is revoked the Order Operation is not performed.

Cross-document Validation

The cross-document validation should be performed only in the Order validation rules. They may be started by user action or script call.

The following actions are performed:

  1. All changed Documents are stored in database. That may give conflict errors.
  2. All Order Documents are checked and if they are not up-to date are correspondingly updated.
  3. The runtime Order synchronization flag is set to Order.
  4. All Document validation rules are run.
  5. All Order validation rules are run.
  6. If the validation is successful the Order is stored and the runtime synchronization type is set to the metadata value.
