Handling Simultaneous User Access Conflicts

Conflict detection is done in UI mode only. The Document stamp field (part of the Document) is used specifically for conflict detection. The stamp field is a string with number values from 1 to 999,999,999, which makes it practically impossible to miss a conflict.

A separate Order stamp field handles the conflict detection at the Order level. It is not the stamp field of the Order Document.

The responsibility of adding these columns is left entirely to the metadata developer who has to include the corresponding fields in the Document. The metadata developer can control (if needed) the names of the columns by deriving user-specific data types from the system data types in the cwf namespace or by changing the column names in the database mapping. Refer to section on Document Leaves for the list of supported system data types.
