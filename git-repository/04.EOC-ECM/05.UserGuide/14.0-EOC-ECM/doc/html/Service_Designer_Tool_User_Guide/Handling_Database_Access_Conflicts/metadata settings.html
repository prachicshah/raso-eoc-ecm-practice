Metadata Settings

The following metadata settings can be made to optimize the conflicts handling:

  • In the Document metadata a new flag, Timestamp sync (default value false), can be set for Documents that are not part of an Order. If the flag is set and the Document does not contain the System doc stamp field, the System last updated timestamp will be used for conflict detection. This is done mainly to accommodate foreign Documents (from the existing databases) in the conflict detection mechanism.
  • The Sync mode field in the Order may have one of the following values:
    • Document: The Document stamp field is used for conflict detection. No conflict detection is done if the Document does not have a Document stamp field. Since no conflict detection is done if no conflict detection field is specified, this situation should be handled with care.
    • Order: The Order stamp field is always used for synchronization even if the Document has a Document stamp field.
    • Default: The Order stamp field is used for synchronization if the Document has no stamp field.
  • If the Document is part of an Order it MUST have a System Order ID and a System Parent ID that are mapped to database. It is highly recommended to have Document stamp field.
  • If the Document is mapped to database its Document stamp field (if any) should be mapped to the database.
  • The Lifecycle Diagram States will have the Read-only state. By default only the END state has this flag set.
