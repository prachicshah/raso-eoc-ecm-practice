Finder User Interface and Forms

The Finder User Interface behaves similarly to other User Interface metadata objects such as Documents and Orders. When configuring Finders, there are three system-defined User Interfaces to choose from and each of these User Interfaces has a set of default Forms.

User Interface Description Forms
com.conceptwave.system.FinderUserInterface The generic Finder presentation, which has a List View that presents the Input Document as search border on top (if defined) and Output Document as search results in a Table. See Finder Variables and Finder Methods for more information. Chart, Default, Menu, ReferenceForm, Result, Search
com.conceptwave.system.OrderFinderUserInterface

This User Interface is similar to the com.conceptwave.system.FinderUserInterface, but the Output Document displayed is an Order Document. Using the Order Document ID, all associated orders (with the same Document ID) are displayed in the Detail View.

For example, by configuring the showDetailAction method you can determine which Order Documents are called and sets it to display in the Detail View of the User Interface.

var order = Order.getOrderById(this.result.selected[0].cwDocId)
this.detail = new order.metadata.UserInterface(order, this);

Chart, Default, Menu, ReferenceForm, Result, Search
com.conceptwave.system.SplitFinderUserInterface Similar to com.conceptwave.system.FinderUserInterface, but in the Detail View, the Output Document Form is displayed at the bottom in a separate pane, with the Finder's Search Border and result Table still displaying on top.
This User Interface does not have the following Forms: Chart, ReferenceForm, Search.		 Default, Menu, Result

Clicking the finder's UserInterface in the left tree shows the following information in the General tab:

Form Description
Name Name of the finder's UserInterface.
Extends The user interface that the finder extends.
Overrides The default value is NONE. Click the drop-down menu and select the user interface to override.
Help Enter help text for the finder, if required.
Timer duration (minutes) Specify the number of minutes that this finder times out in this field.
Validate on update

By default, this checkbox is selected, meaning that the finder functions as usual. When this property is unchecked, the product only calls the save method on the metadata document without validating.

Note: The finder result is only the data document and does not have controller associated with it.

Extend a Finder User Interface and Invoke Its Methods

You can extend a finder User Interface and invoke its methods. The following is an example:

  1. Create a User Interface that extends com.conceptwave.system.FinderUserInterface and add a permission method in this User Interface.
  2. Create a finder and extend its User Interface to the User Interfacet that you created in the previous step.
  3. Set the Visible property of one of the User Interface elements to the permission method in step 1.
  4. The permission method is invoked when displaying the finder User Interface.

Finder User Interface Forms

By default there are several Forms that display in the Finder User Interface:

Form Description
Chart Defines different chart types that can be used for graphical presentations of the Finder data.
Default This Form is the main form that renders the Finder and includes the Search, Result, Detail and Chart sections that are shown or hidden appropriately based on user's navigation at the Finder. It also displays the preference menu items.
Menu Defines the object menu options such as Search , Select, Add, Copy, Update, Delete.
Search Defines the appearance of the Search at runtime. It includes a Form Frame that displays the content of the search.
Result Defines the appearance of the Results at runtime. It includes a table form element that displays the content of the result.

You can define the layout of the Finder, using the Forms and their extend and override functions. For example, you can add additional menu items or alter existing functionality (show results in other Form Elements other than Table).

When associating a Finder into your application User Interface, create the User Action Method at the User Interface of the application Page or Form, and associate that method using a Form Element such as Menu or Button.
