Order General Properties

The general Order properties are defined on the General tab. The following image shows the General tab of the top-level Order.

The following table describes the fields for Order; fields available to top-level Order metadata, Order Collection and Order Document are slightly different as indicated.

Fields Root-Level Order Order Collection Order Document Description
Name Yes Yes Yes Name of the metadata object (used in scripts; must conform to JavaScript naming conventions). Read-only after creation. To change it, use the Rename command in pop-up menu by right-clicking the metadata object.
Private Yes No No If checked, the metadata object is hidden from display when the metadata is packaged as Product Metadata (that is, library).
Restricted Yes No No If checked, all JavaScripts contained within the object are blocked from being displayed when the metadata is packaged as Product Metadata (that is, library).
Deprecated Yes No No If checked, the metadata object is designated for archival purposes only. The object becomes hidden from all metadata object selection lists.
Virtual Yes No No If checked, it provides the ability to create methods in template metadata that does not have implementation, but can be used in template script and be bound to UI elements. The implementation is done in user metadata (see Virtual Field for more information).
Final Yes No No Metadata elements can be marked as final, which includes any Boolean final property that is available on any metadata element. Additionally, any metadata element that has the Final property set to true can neither be extended, nor overridden.
Label Yes Yes Yes Metadata object display label.
Description Yes Yes Yes Description of the metadata object for documentation.
States Yes No No If the Order needs to be associated with a state during processing, a Life Cycle must be selected. The Life Cycle defines the Order states and state transitions. It must be defined in advanced to be seen in this field.
Instance key Yes Yes No The selected child node to show as a root node of the Order or Collection instance. If selected, the child visual key will be displayed at runtime as the visual key of the selection instead of the root/Collection visual key. The visual key of this item should uniquely identify the Order or Collection instance.
Order item No No Yes Mandatory. The Document that is referenced by this node.
Minimum, Maximum No Yes No The Minimum and Maximum values represent the range of Collection instances that can be added under a Collection at runtime. Note: there is no limit to the Maximum number. However, if the Minimum number is 0, the Collectionfolder will not be seen in the OE client UI when the Order is activated. To add an Order Document instance into such a Collection, highlight its parent element and click the Add button in the OE client UI.
Mandatory No Yes Yes Indicates that this Collection/Document is mandatory at runtime.
If checked for Collection, at least one instance will appear when the Order is being created, even if the Minimum instance is set to zero if item is a Collection. Otherwise, if unchecked, the Collection instance folder's appearance will depend on the Minimum instance number. For example, if the Minimum instance is zero, the folder will not be seen in the UI when the Order is activated.
Attachments Yes No No If checked, the attach icon is enabled on the OE client UI, allowing users to attach a file to the Order at runtime.
XSL:FO file Yes No No Specifies the url for the XSL:FO file to be used for Order Printing, as defined in the Configuration application under Resources.
Note: See Configuration User Guide, Web Resources for more information on defining resources.
Note: See Velocity Studio User Guide, Order Printing 18 for more information on PDF printing.
Sync mode Yes No No Defines a synchronization mode that is used for handling the simultaneous user access conflicts. See Metadata Settings and Handling Simultaneous User Access Conflicts for more information.
XML Settings Yes Yes Yes Specifies how to export/import orders to/from XML format. By default, all Order elements are exported/imported.
XML Name XML tag name for this element. Default value is the element name.
Instance name Visible for Collection items only. This is the XML tag name for the instance of this Collection. The default value is the Collection name. This field allows different XML tags for the Collection as a whole and its instances. If not specified, the Collection and the instances will have the same XML tag name.
Include / Exclude When Exclude is selected for any Order Item, it excludes the element XML tag from the processing (that is, it does not generate the tag when writing XML and does not look for it when reading XML). However, all children of the Order Item with XML tags should exist.
Include / Exclude / Exclude children Exclude children option is available at Collection items only. When checked, this item, and all of its children, will be excluded when generating the XML or not processed when reading the Order from XML.
Include namespace Visible for an Order root only. When checked, the xmlns attribute will be generated with the namespace value of the namespace this Order belongs to.
