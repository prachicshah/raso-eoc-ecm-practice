Permission Rule Set

A permission rule defines accessibility of data object nodes in user interface (UI). Permission of element nodes or containers is determined by rule sets defined on the referenced element. Each node in a data object can have exactly one rule defined of each type of application. Permission rules are automatically evaluated by the UI when rendering the data object's UI. The order of permission rules is not relevant.

Permission Rule Types


Type Description
Visible Indicates a given node can be visible in the UI. This rule type is applicable to all nodes.
Editable Indicates a given node can be edited in the UI. This rule type is applicable to all nodes.
Optional Indicates a value for a given node does not need to be defined. This rule type is applicable to data type.
Add Indicates a given node can be added to its parent. This rule type is applicable to container nodes only.
Delete  Indicates a given node can be removed from its parent. This rule type is applicable to container nodes only.
