Rule Sets Rules Properties

The rule properties of rule sets are on the Rules tab.

The following table describes the fields:

Field Description
Description Description of the Rule Set for documentation.
Enabled This field is mandatory and its default value is true. Editable only if the Rule Set reduces another and this Rule is defined in the base.
Type The value of this field depends on Rule Set Type.
  • For Initialization Rules, this field only takes Script.
  • For Validation Rules, this field can be either Script or Mandatory. If Mandatory, the Script property cannot be defined.
  • For Permission Rules, type can be one of the followings depending on node type:
    • Visible
    • Editable
    • Optional
    • Add
    • Delete
Severity This field is Mandatory and Editable for Validation Rules only. select from one of the following values:
  • Information
  • Warning
  • Error
Code This field is Mandatory and Editable for Validation Rules only.
Script This field is Mandatory and Editable if Rule Type is Script.Â The script is evaluated in the context of the data object, allowing the script to access the data object instance with this.
  • For initialization rules, the return value from this script is assigned to the Rule's node.
  • For validation rules, if this script returns true, the Rule is Triggered.
  • For permission rules, if this script returns true, it indicates that the user has the given permission type on this node.
