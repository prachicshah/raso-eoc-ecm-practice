Rules Config Instances Tab

The Config Instances tab allows you to maintain a list of database or deployment instances. These instances are stored in a BLOB within a CLUSTER record in the CWVMINFO table. The Config Instances tab is available for metatdata and templates (that is, from the Library tab), and is enabled when a database connection is available. This tab contains Rule Instance field.

The order of instances in the list defines the order in which the instances will be evaluated at runtime. This order can be changed using the Up and Down arrow buttons on the right side of the table and the Sort button.

Note: The Rule properties are read-only. When connected to the database, on the Config Instances pane, you can add and change rule instances for a rule either from the library or in your metadata.

The Rules Instances tab contains the following fields:

Column Description
Label The instance label.
Exclude When checked, the instance will be excluded from the evaluation sequence, but can be invoked from other instances.
Has Condition When checked, the instance has a condition.

Notes:

  • The system must be restarted once runtime instances are changed.
  • When you have a rule that contains both instance rules and configuration instances, the configuration instances are processed first.

The Rule Instance tab contains the following buttons:

Button Description
Up Moves the selected instance rule up in the instance list.
Down Moves the selected instance rule down in the instance list.
Add Opens the Rule Instance Properties dialog so that a new instance can be added for the rule selected in the Rule combo box. The new instance will be placed below the selected row. New instances can be added to any rule, including rules from libraries.
Remove Removes the selected instances from the list.
Properties Opens a Rule Instance Properties dialog allowing the user to view or change the selected instance.
Sort Sorts the instances list table in ascending order.
Create rule instance from template Click this button to either enter the object name that you want or select the rule instance available from the dialog.
Copy and Paste Create a copy of the selected instance and pastes it below the selected row.
Save All Saves all changes that you have made.

When an instance has the same name as a metadata instance, the instance overrides the metadata instance (that is, a database rule replaces a metadata one). To create a config instance from a template, click the Add button. When a database instance name is unique, it is added at runtime to the set of metadata rule instances.
