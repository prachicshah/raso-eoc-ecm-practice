Generate Metadata Documentation

Full HTML design documentation for the current project can be generated by selecting the File/Export/Documentation menu command. Once the command is selected, the Generate Documentation dialog will open where you can specify the namespaces and metadata objects to be included in the generated documentation. By default, all namespaces and objects are included. The table below describes the fields in the dialog.

Field Description
Namespaces Allows the user to select the namespaces in the current metadata that will be documented.
Options Allows the user to select the object types in the current metadata that will be documented. When the Documentation check box is selected all the object types in the Options list will be selected. When the Documentation check box is deselected all the object types in the Options list will be deselected.
All When clicked, all the namespaces in the Namespaces list will be selected.
None When clicked, all the namespaces in the Namespaces list will be deselected.
Include all objects When clicked, all metadata objects of the types listed in the Options pane will be included in the documentation
Generate When clicked, opens the Save Documentation dialog where the directory location to export the file too and the file name is specified.
Close When clicked, closes the dialog without any action.

The documentation that is generated consists of tables names, columns, etc. The example, shows documentation that was generated for a user interface application that contained one variable.
