Preferences

The Designer Preferences page allows you to define a number of Velocity Studio settings. Click any of the following sections in the left menu to change your Velocity Studio settings:

  • Runtime
  • Appearance > View
  • Project
  • Export
  • Setting
  • Test Case

Runtime

The Runtime preferences specify how Velocity Studio shall execute at runtime:

Field Description
HTTP Port The Port number to use to serve Web application when the project is run within Velocity Studio. Default is 8080.
Help Port The Port number to use to serve Help pages in Velocity Studio. Default is 8081.
Preview Port The Port number to use to display the preview pane when managing Forms in Velocity Studio. Default is 8082.

For any Port settings, if the port number is already in use by the machine, the runtime attempts to bind to the next available incremental port number. In this case, this information appears in the console.

Appearance > View

This section allows you to defines the view preferences within Velocity Studio when navigating metadata:

Field Description
Tree node labels Specifies how the text of the nodes in the metadata tree of Navigation Pane is displayed. It can be one of the following:
  • Name Only - only name of metadata object is displayed
  • Label Only - only label of metadata object is displayed
  • Name (Label) - name of metadata object is displayed, appended with its label in brackets if label is different from name. This is the default.
  • Label (Name) - label of metadata object is displayed, appended with its name in brackets if name is different from label.
Element node labels

Allows you to specify how the text of element nodes is displayed in the element tree. It can be one of the following options:

  • Name Only - only the element's name is displayed
  • Label Only - only the element's label is displayed
Form Preview - Enable Preview By default, the preview area is enabled. Deselecting this setting disables all settings in the Form Preview section and disables the preview area when you are developing your metadata.
Form Preview - Auto Refresh Browser Specifies whether to automatically refresh the preview pane when you click a different form element in the Form design in Velocity Studio.
Form Preview - Minimize Preview Allows you to minimize the preview area when you are developing your metadata in the design area, providing you with a full design area.
Form Preview - Auto Expand Element Tree Allows you to view multiple levels of children (for example, submenus) on a form. By default, this setting is checked.

Project

This section allows you to define your project preferences:

Field Description
Filtered resources When Velocity Studio manages the Project Directory (that is, source files of the project), any folders and files with the specified name in this field are ignored and left alone in the directory. This setting defaults to .svn to allow version control on Project Directory using subversion.
Filtered in war This field allows you to specify file names that are separated with a comma (,). When you create a WAR file using metadata, the files specified in this field are ignored. This setting defaults to .bak to filter out all the backup files. The Build Deployment WAR function uses this field.
Webapp library folder Specify the folder that contains extra JAR files, which will be packaged to WEB-INF/lib folder upon packaging.
Compress SmartClient Resources

By default, this setting is checked. When this setting is unchecked, upon packaging, the generated WAR file contains servlet.compress: false.

Note: See the Disabling SmartClient compression section on how this parameter in your startup script works with the Compress SmartClient Resources checkbox.
SmartClient Resources Expiry Time (hours)

By default, this setting defaults to one hour. This setting allows you to set the expiration time for SmartClient resources.
Folders to look in for template files

This property allows you to specify either a relative or absolute directory where Velocity Studio will search for template JAR files. This search is performed in addition to searching the templates folder of your metadata. To specify more than one relative or absolute directory, separate them with a semicolon (;).

Notes:

  • If a folder has the same file as another folder, the template JAR file is read from the templates folder first, and then from left to right of the specified folders.
  • When you add a new template library file to the metadata header, a prompt appears, asking whether you want to copy the file into the templates folder of your metadata. Clicking the Yes button copies the files to the template folder. Otherwise, clicking the No button adds the path to the project settings property, if it does not already exist.
Runtime Test Features

This field's drop-down menu shows features that are currently defined and activated for testing using the test handler to open any protected template. This field allows you to turn these features on and off.

Export

This section allows you to define preferences in export commands:

Field Description
Export Schema inside WSDL If checked, any Data Types or Data Structures that are referenced by the SOAP elements are included in <schema> element at the top of the WSDL file when Exporting WSDL, akin to the content of an XSD file.
Export Referenced Types Only

For WSDLs, if you select this setting, only types referenced by the services and any types that they depend on are exported.

For XSDs, the following occurs if you select this setting:

  • For the selected namespace, all types are exported.
  • For referenced namespaces, only types required by the selected namespace are exported instead of the complete set of types for all referenced namespaces.
Resolve Override Types This setting allows you to indicate whether overrides are used in the WSDL (that is, references to a base object are replaced with references to an overriding object). If you want to use overrides, ensure that you make the WSDL or XSD consistent with your business needs, as overriding is not a supported feature in XSD. This setting is off by default, meaning that overrides are not resolved.
Export Template JAR Folder

This field allows you to set the default path for building the template JAR file by clicking the Browse button and selecting the folder that you want. If the path is either empty or does not exist, a popup appears, prompting you to select a file editor. Otherwise, the default path is used to build the template JAR file.

This setting is a project preference, and not a Velocity Studio preference, as indicated in the following example:

  1. Open a project (project A) and define the Export Template JAR Folder as Y.
  2. Proceed to export the file by clicking Runtime > Build Library JAR from the menu bar. Ensure that the JAR has been exported to folder Y.
  3. Open another project (project B) and define the Export Template JAR Folder as folder X.
  4. Export the file by clicking Runtime > Build Library JAR from the menu bar. Ensure that the JAR has been exported to folder X.
  5. Reopen project A. When you click Runtime > Build Library JAR from the menu bar, the JAR is still exported to folder Y.

Setting

This section allows you to define the following setting preferences:

Field Description
Auto Save Metadata By default, this field is set to Never. You can specify how often Velocity Studio automatically saves your metadata by clicking the drop-down menu and selecting the number of minutes from the list.
Prompt for Confirmation on Method Deletion By default, when you delete a method, you do not see a confirmation message. Selecting this field allows you to see this confirmation dialog before you delete any method.
Show Finder Type Folder By default, this field shows the finder type folder in the Metadata tree.
Show Library Warning By default, this field shows all library warnings received in Velocity Studio.
Show Metadata Label in recent project list Select this checkbox to display the metadata label in the recent project list (that is, clicking File from the menu bar) and its file path.
Generate upgrade SQL in one output file By default, this setting generates one upgrade.sql file when you select Database > Upgrade System from the menu bar. When you uncheck this setting, Velocity Studio generates separate SQL script files for each logical schema.
Generate verbose message in console

When selected, this setting generates verbose messages in console of Velocity Studio. Such as, information about the calls on the system, like notices, warnings, and errors etc.
Prompt to propagate subflow revision

By default, this field is selected, meaning that each time a subflow generates a new revision, a dialog with a list of processing using this subflow appears. You can then select processes that need to have their subflows re-adjusted to use the latest subflow revision. When you click the OK button, the selected process references are automatically re-adjusted.

When this field is unchecked, generating a new subflow revision no longer prompts to re-adjust subflow references.

Test Case

The Test Case preference provides the ability to configure predefined scripts for test cases:

Field Description
Show TestCase Template By default, this field is unchecked and all three input fields are display only. Check this field to enable the three input fields and to indicate that the predefined script will be used when creating a new test case or test function.
Setup Script, Teardown Script, Script Template

By default, these three input fields are display only unless the Show TestCase Template checkbox field is checked. These fields are text areas that can be used to define a script for a test case and a test function.
Default Script If modifications have been made to the predefined script via the three input fields, this button provides a means to view, in a dialog, the default predefined script (that is, before changes).
