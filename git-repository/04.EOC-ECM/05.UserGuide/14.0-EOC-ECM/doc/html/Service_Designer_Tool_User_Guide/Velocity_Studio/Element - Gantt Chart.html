Gantt Chart

A Gantt chart is a type of bar chart that shows a project schedule, including the start and finish dates of each task belonging to a project. The Gantt Chart element enables you to configure the overall look and feel of the chart display at runtime. The Gantt Chart exists under a finder user interface, relying on its result variable of type document array. Each document in the array represents a new task in the Gantt chart.

Velocity Studio contains the GanttDocument system variable, which includes seven variables that form a template structure for your Gantt chart. The following table shows the required columns that comprise your document:

Name Description Nullable? Unique?
processName The name of the process displayed in the Gantt chart. No Yes
taskName The name or label of each task in the chart. No Yes
startDate The start date of the task No No
endDate The end date of the task No No
percentComplete The completion percentage of the task Yes No
connectToTask Another task to draw a line starting from this task Yes No
taskColorFill The colour to fill the completed part of the task Yes No

Properties

The following are the properties for the Gantt chart element:

Property Mandatory/Optional Comment
Name Mandatory The Gantt chart name.
Variable Mandatory Finder result form for the Gantt chart. Use the property's lookup button to view property details.
Caption Optional The title (label) of the Gantt chart.
Click Method Optional

When you specify this property, chart elements are clickable. Clicking these elements invokes the method. The method receives two parameters:

  • The data object that the chart node represents
  • The variable name, if applicable, that the chart node represents
Connector Thickness (max 10) Optional

The line thickness that connects two tasks in the Gantt chart. The higher the number, the thicker the connection line is between tasks. This property's default value is 2.

Note: If you enter a number that exceeds 10, the Velocity Studio automatically overrides this value and sets this field to 10.

See the Set Up Connectors section for details.
Date Format Optional The date format used in the Gantt chart. The default format is mm/dd/yyyy. Click the drop-down menu in the Value field to change the date format.
Dynamic Height Optional

This property allows you to dynamically assign a variable or a script to the Dynamic Height property.  For a variable, the data type must be either an integer or string. For the script, only the script action can be used and its return type must be defined as either integer or string.

At runtime, by default, the regular width and height are used. However, if Dynamic Width and Dynamic Height are defined, they replace the regular width and height, and the values are calculated based on either the variable or script.
Dynamic Width Optional

This property allows you to dynamically assign a variable or a script to the Dynamic Width property.  For a variable, the data type must be either an integer or string. For the script, only the script action can be used and its return type must be defined as either integer or string.

At runtime, by default, the regular width and height are used. However, if Dynamic Width and Dynamic Height are defined, they replace the regular width and height, and the values are calculated based on either the variable or script.
Dynamic Pane Duration Unit Optional

This dynamic property can be set to a method returning either a String or a String type variable, which allows you to set this property from runtime. This property gets translated into the ganttPaneDurationUnit property, which takes the following values:

Code Description
y Year
m Month
d Day
h Hour
mn Minute
s Second

These values must be used as the code table.

The following is an example of using this property:

  1. Create a data type that extends String. As the Enumeration, create the aforementioned code table.
  2. In the UserInterface, create a variable with this data type.
  3. Override the Menu form.
  4. Add a Select Field element and assign the newly created variable.
  5. Set the Run Trigger property to showChart.
  6. Go to the Gantt chart Form and set the Dynamic Pane Duration Unit property to the same variable.
Dynamic Suppress Dependencies Optional

You can set this dynamic property to either a Boolean or a method returning a Boolean type. This property can be set during runtime. It suppresses displaying dependencies connectors in the Gantt chart.

The following is an example of using this property:

  1. Create a Boolean type variable in the UserInterface.
  2. Override the Menu form.
  3. Add a Checkbox element and assign the newly created variable.
  4. Set the Run Trigger property to showChart.
  5. Go to the Gantt chart Form and set the Dynamic Suppress Dependencies property to the same variable.
Gantt Width Percent Optional

This static property gets translated as ganttWidthPercent, which can be set as a percentage between 0 and 100. The Gantt chart consists of two parts:

  • The Gantt chart
  • The data table, including the process name

This attribute allows you to set the width of the Gantt part, in percentage, with respect to the entire chart. If the Gantt takes less space, the grid automatically gets more space for the process part. As a result, it is inversely allowing you to control the area width for process names.
Height Optional

The height of the Gantt chart. Click the Value field, and then select whether you want the height represented in pixels or a percentage using the drop-down menu.

Note: Although the Height property does not have a maximum limit, specifying a large pixel or percentage in this field may show a black screen during runtime. As a result, it is recommended that you select an appropriate height value for your Gantt chart.
Palette Color Optional The base colour for the entire Gantt chart. Based on the colour specified for this property, the Gantt chart's border colours, font colours, connector colours, and more are automatically changed. Click the Ellipsis button in the Value field and select a colour from the palette provided. To remove an existing colour selection, click the Delete button. See the Notes section for additional details about palettes.
Pane Duration Optional This field specifies how many of the duration units specified in the Pane Duration Unit field to show in the Gantt chart's main screen without having to scroll. For example, if you specify a duration of one month, the Gantt chart only shows the first month and scrolls the rest. By default, this field contains the value of 1 year.
Pane Duration Unit Optional

Click this field's drop-down menu allows you to specify the following duration units:

  • Year
  • Month
  • Day
  • Hour
  • Minute
  • Second
Process Font Style Optional

The font style used for process names (labels). Click the Ellipsis button and select from the following font styles:

  • Font family (for example, Arial, Verdana, and so on)
  • Colour (click the Ellipsis button to select a colour from the palette)
  • Size in pixels
  • Bold, Italic, and Underline styles (click the drop-down menu for each option to select Yes; the default value for each option is No).

Click the Save button to save your font settings.
Process Header Label Optional The process header title (label) that appears above the process names.
Show Percent Label Optional Select this checkbox to display the percentComplete value for each task in the Gantt chart.
Show Slack As Fill Optional Select this checkbox to show the percentage of work that has not yet been completed for each task in the Gantt chart.
Show Task End Date Optional Select this checkbox to display the end date of each task in the Gantt chart.
Show Task Name Optional Select this checkbox to show the name of each task in the Gantt chart.
Show Task Start Date Optional Select this checkbox to display the start date of each task in the Gantt chart.
Show Validation Optional This boolean property has values of either True or False, which determines whether a validation warning message displays at runtime. By default, this property is unchecked, meaning that validation errors do not appear under the chart at runtime and supports backward compatibility. Otherwise, selecting this property allows validation errors to appear under the chart at runtime.
Slack Fill Color Optional If provided, the percentage provided as the percentCompleted value is displayed with the specified fill colour.
Visible Optional This field defines the user permissions assigned to this layout. Pre-defined system Permissions, such as addPerm, attachPerm, deletePerm, editablePerm, and visiblePerm can be assigned as well as Object Permissions. This field defaults to NONE, which means that no permissions are assigned.
Width Optional The width of the chart. Click the Value field, and then select whether you want the width represented in pixels or a percentage using the drop-down menu.

Set Up Connectors

Gantt charts can be set up to have connections between tasks. These connections are data-driven and can be set by providing a value for the connectToTask variable. You must ensure that a task with the exact name exists in the chart; otherwise, the connector will not show. If the proper data is provided, the connector will be displayed from the end of the current task to the beginning of the specified task.
