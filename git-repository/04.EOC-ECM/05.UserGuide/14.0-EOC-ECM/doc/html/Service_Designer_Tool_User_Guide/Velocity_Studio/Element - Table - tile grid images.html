Tile Grid Images

You can add images at the bottom of a tile in a tile grid (table element).

The following is an example of using this feature:

  1. In Velocity Studio, add a table element to your form.
  2. Select the table element's Is Tile Grid property checkbox.
  3. Set the table element's Variable property.
  4. Add data-specific columns (that is, input fields).
  5. Proceed to add a horizontal layout and set the following properties:

    Property Description
    Height Set the horizontal layout's height to be shown at the tile's button. Ensure that the tile size is large enough to accommodate for this size.
    Members Margin Set the amount of space to use between each image under the layout.
    Style Select the style to use for the layout.
    Visible Select the permission to be called for each tile in the tile grid. It is recommended that you add this permission’s second parameter to represent the data object being passed in.
    Width

    By default, the width has a value of 100%.

  6. Under the horizontal layout, you can add images. These images have a limited set of properties that can be used:

    Property Description

    Click Method

    		 This property denotes the click method for this image. To get the current tile, use this.result.selected[0].
    Disabled Set this permission method to be called for each tile in the tile grid. It is recommended that you add this permission’s second parameter to represent the data object being passed in.
    Height This property contains the image's height.
    Image Type Specify the image's type in case the height or width in the properties does not match the actual size.
    Image URL This property represents the static image to use for this tile.
    Visible Select the permission to be called for each tile in the tile grid. It is recommended that you add this permission’s second parameter to represent the data object being passed in.
    Width

    This property denotes the image's width.

  7. Save your metadata.

To use this functionality in UI Common, do the following:

  1. Create a finder user interface and set the Extends field to ui_common.baseFinder.
  2. Right-click ResultTileGrid and select Override.
  3. Right-click the table and select Extend. Proceed to add the data needed to be shown in the tile.
  4. To add new images in the horizontal layout, right-click the layout and select Add.

    By default, the ResultTileGrid form includes the following icons:

    Icon Name Visibility Disable Functionality
    tileActionIcons (layout with the icons) isShowTileActions – returns isTileActionButtonsVisible None None
    editIcon isShowTileEdit – returns isTileEditVisible isDisableTileEdit, which returns isTileEditDisabled Defaults to call onShowDetail method
    deleteIcon isShowTileDelete – returns isTileDeleteVisible isDisableTileDelete, which returns isTileDeleteDisabled Defaults to call onDelete
    copyIcon isShowTileCopy – return isTileCopyVisible and is set to false by default isDisableTileCopy, which returns isTileCopyDisabled Defaults to call onCopyAction – this method is empty. Override it to provide the functionality.
    folderIcon isShowTileFolder -  returns isTileFolderVisible and is set to false by default isDisableTileFolder, which returns isTileFolderDisabled Defaults to call onFolderAction – this method is empty. Override it to provide the functionality
    faveIcon isShowTileFavourite – returns isTileFavouriteVisible and is set to false by default onFavouriteAction, which returns isTileFavouriteDisabled Defaults to call onFavouriteAction – this method is empty. Override to provide the functionality

Note: Image clicks do not function if the tile grid contains a Selection Change method. Instead, the double-click method has been added to accommodate this restriction.
