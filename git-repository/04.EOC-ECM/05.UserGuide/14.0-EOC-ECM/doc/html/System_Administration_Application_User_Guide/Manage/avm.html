Application Virtual Machines (AVMs)

Clicking Manage > AVMs from the menu bar allows you to view information about processes in the runtime environment. This page displays a list of currently running user interface servers.

The AVMs page contains the following fields:

Field Description
AVM The ID of the node, assigned in the process engine start script.
Application This field indicates the internal name of application.
Node Name This field Indicates the node name, specified in node menu of the Configuration application, and selected in the Node field of Select Application menu in Velocity Studio.
AVM Mode The node type (PE or UI, or both).
Process Engine ID The ID of the process engine, assigned in the process engine start script.
Host The name of the host address.
IP Address The host IP address.
PE Mode The status of the process engine.
JMX Port The JMX Port for this PE, passed to the PE in its start script.
Started Date and time the AVM has been started.
Updated On Last time when the AVM heartbeat thread has updated the database, indicating that the AVM is operational. The AVM heartbeat is every 5 minutes.
Ended Time The date and time when the AVM stopped.
Memory Usage (KB) Currently used memory in KB.
Total Memory (KB) Maximum available memory in KB.
Engine Status Process Engine status. Can be Running, Stopped, Saturated (that is, there is no more memory to handle new processes) or Dead (the last heart beat is not received in the expected time).
Number of Processes The number of processes run by this Process Engine.
Bucket Number The bucket number in the span this PE is a part of (clusters only).
Activated On Displays the build version (buildVersion and buildInternalVersion) of the packaged WAR file (see also Build Deployment WAR).

Double-clicking any AVM in the search results launches the AVM Details page, which provides details about the AVM on a separate screen.

The AVM Details page shows all fields displayed on the AVM page as well as the following fields:

Field Description
System Application Name The name of the system application.
Engine Parameters Other parameters passed to the process engine in it is a start script (for example, configuration file, cdef file, sync or async, and so on.)
Running Version The GUID generated from the build version key. If deployment is successful, Running Version is updated to the value of Activated Version to indicate the current running version.
Running Iversion The GUID generated from the build version key. If deployment is successful, Running Iversion is updated to the value of Activated Iversion to indicate the current running version.
Activated Version Displays the current build version used to successfully deploy the WAR file.
Activated IVersion The GUID randomly generated during activation.

Commands

The following commands are available from the AVM page:

Menu Item Function
Reload Config

Select an AVM from the search results, and then click this button to reload the system configuration information from the database. Parameters and settings from all Configuration applicaiton modules are reloaded, except for those under Databases and Passwords.

Note: It is recommended to restart JBoss if you make reload configuration variables.
PE Info Select an AVM from the search results, and then click this button to launch a drill-down summary for a selected process engine.
Memory Processes Launches the Process Manager, allowing you to review process activities in the system.
