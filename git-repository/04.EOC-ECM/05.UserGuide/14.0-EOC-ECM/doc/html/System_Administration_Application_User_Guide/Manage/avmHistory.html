AVMs History

Clicking Manage > AVMs History from the menu bar allows you to view currently running and archived AVM records.

The AVMs History page contains the following fields:

Field Description
AVM The ID of the node, assigned in the process engine start script.
Application This field indicates the internal name of application.
Node Name This field indicates the node name, specified in node menu of the Configuration application, and selected in the Node field of Select Application menu in Velocity Studio.
AVM Mode The node type (PE or UI, or both).
Host The name of the host address.
IP Address The host IP address.
PE Mode The status of the process engine.
JMX Port The JMX Port for this PE, passed to the PE in its start script.
Started Date and time the AVM has been started.
Last Updated Last time when the AVM heartbeat thread has updated the database, indicating that the AVM is operational. The AVM heartbeat is every 5 minutes.
Ended The date and time when the AVM stopped.
Memory Usage (KB) Currently used memory in KB.
Total Memory (KB) Maximum available memory in KB.
Engine Status Process Engine status. Can be Running, Stopped, Saturated (that is, there is no more memory to handle new processes) or Dead (the last heart beat is not received in the expected time).
Number of Processes The number of processes runs by this Process Engine.

Search AVMs

You can search AVM records based on the AVM ID, host name, and IP address. The following fields are available for the search:

Field Description
AVM This field indicates the ID of AVM node.
Host This field denotes the name of the host where an AVM is either currently running or was running previously.
IP Address This field indicates the IP address of the host.

To search, enter the information in one of these fields and click the Search button.
