Memory Processes

To access the Memory Process page, which displays a list of all processes in the system, do the following:

  1. In the AVM search results, select the checkbox next to the AVM that you want to view.
  2. Click the Memory Processes button to view a list of all processes in the system.

The Memory Processes page contains the following fields:

Field Description
Process ID The Process identifier.
Bucket Number Refers to the a single process instances with a value of 0, or a multiple process instances (cluster) with a value based the number of process instances and how they are configured.
Process Type The metadata process label.
Started Date and time when the process was started.
Status Process status.
Priority Process priority.
Order ID ID of the Order the process belongs to.
Order Type Type of Order the process belongs to.
Status Changed Date and time when the process status was changed.
Parent ID ID of the parent process (if the listed process is a child process).
Metadata Version Metadata version number.
Metadata Revision

The process migration stops runtime process instances created with process revisions or convert them to the latest revision number (the process). The migration actions are entirely user defined.
Order Item ID ID of the Order Item that the process belongs to.

Selecting the checkbox next to a process from the Memory Processes page and then clicking the Activities button displays a dialog that shows all activities that are being processed.

To view memory process details, click the Details button to the left of the Process ID field.
