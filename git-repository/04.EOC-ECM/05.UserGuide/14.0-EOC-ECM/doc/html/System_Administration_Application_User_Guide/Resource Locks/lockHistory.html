Lock History

Clicking Resource Locks > Lock History displays the history of locks including the pending and active ones. From this page, you can remove a lock, search for a specific lock, or view all active locks in a read-only view.

The following is a description of the fields and columns for the search and results sections on the Lock History page:

Field Description
Resource ID This field indicates the resource identifier. The value for this field must be unique for type and system.
Resource Type This field indicates the type of the resource.
Requester ID This field indicates the requester ID (for example, process ID).
Requester Type This field provides the information about the requester type (for example, name of the process).
System This field indicates the resource system. For example, Ericsson for orders stored in the Ericsson system.
Release Action This field denotes information how the lock was released (for example, through an API call).
Acquired By This field provides the information by which user a lock has been acquired..
Released By This field indicates by which system the lock is released.
Acquired After This field indicates a date after a lock was acquired.
Acquired Before This field indicates a date before a lock was acquired.
Released After This field indicates a date after a lock was released.
Released Before This field indicates a date before a lock was released.
Acquired On This field indicates a date an order was acquired by a resource.
Released On This field indicates the lock release date.
Acquired By This field provides the information by which user a lock has been acquired.
