Pending Locks

Clicking Resource Locks > Pending Locks displays all the locks in the pending status. From this page, you can remove a lock, search for a specific lock, or view all active locks in a read-only view.

The following is a description of the fields and columns for the search and results sections on the Pending Locks page.

Field Description
Resource ID This field indicates the resource identifier. The value for this field must be unique for type and system.
Resource Type This field indicates the type of the resource.
Requester ID This field indicates the requester ID (for example, process ID).
Requester Type This field provides the information about the requester type (for example, name of the process).
System This field indicates the resource system. For example, Ericsson for orders stored in the Ericsson system.
Created By This field allows provides the information about a user who created an order.
Created After This field indicates a date after an order was created.
Created Before This field indicates a date before an order was created.
Acquired On This field indicates a date an order was acquired by a resource.
Expire On This field indicates lock expiry date.
Acquired By This field provides the information by which user a lock has been acquired.
Priority This field indicates the priority of an order in the queue. The default value for this field is eight (8).
Group ID This field represents a unique ID that is used to mark a group of resources from the lock request.
