Interface Status

Clicking Services > Interface Status from the menu bar allows you to test external services for connectivity, availability, and status:

  • Connectivity is the ability for the framework to make a network connection to the remote service
  • Availability is the existence of of a remote service or operation, or both
  • Status is the ability to get an expected or OK response from that service or operation.

The interface status has a mechanism to supply an XML test message for an interface operation. You can use the ping functionality for all operations, except for SOLICIT-RESPONSE, which is not a supported operation type for client external service calls. The result data or error details are displayed.

Notes:

  • This feature does not test the connectivity of listener interfaces for remote clients
  • The availability and status of listeners can be checked through Java™ Message Service (JMS)
  • Changes to either the metadata or the external service that renders the test message invalid are not detected by the interface status feature. Test messages need to be reconfigured.

To test the status of an interface, do the following:

  1. Specify the following search criteria:

    Field Description
    Interface

    Click the drop-down menu and select the interface that you want to test.

    Note: This list contains interfaces that have configured client ports. The ports do not have to be active.
    Interface Operation Click the drop-down menu and select the operation name from all possible interfaces from the Interface field or operations from the selected interface.
    AVM Specify the node name from configuration.

  2. Click the Ping button to apply the specified interface or operation on the specified node.

  3. The Ping Time field contains the date and time that you clicked the Ping button. Similarly, the Port field automatically contains the configured port for the specified interface.

  4. In the Response/Error field, the results appear as a list. Possible results are as follows:
    • XML data from the response
    • An OK message indicating that the operation completed, but no response was received (likely a one-way operation)
    • An error message and stack trace, if an error occurred

    Notes:
    • The results list appears when the ping function has completed.
    • The results are cleared every time the ping function is performed.

JMX

To enable either the System Administration application or the Configuration UI to be able to ping from any node, and for it to be synchronous and instant, the node has to be contacted through JMX. To facilitate this process, the port objects, which are already available in JConsole, have the Ping operation added to them.
