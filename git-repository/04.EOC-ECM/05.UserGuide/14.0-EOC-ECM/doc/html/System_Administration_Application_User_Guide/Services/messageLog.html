Message Log

Clicking Services > Message Log from the menu bar displays a report containing a list of message log items. The message log contains information about all the messages and requests that are sent to and received from external systems.

To search for an item in the message log, enter your search criteria, and then click the Search button. Your search results appear in the area under the search criteria.

The following search fields are available:

Field Description
Queue Search Indicate whether you want the message log to search the queue. By default, this field is unchecked.
Start Time - From Date and time when the message was sent. Click the Calendar icon to view the calendar and select a date.
Start Time - To Date and time when the message was received. Click the Calendar icon to view the calendar and select a date.
Message ID Unique message log entry ID. You can sort by this field. Favorites are enabled for this finder, allowing you to save your preferred format.
AVM The AVM ID that created this message log entry.
Interface Type Interface entry of the logged message.
Process ID The process identifier of the logged message.
Order ID The Order ID of the logged message.
User ID

The user identifier that performed the interface operation.

The following table describes the user identifier that appears in the search results for this field for each type of server listener:

Listener User ID
HTTP This field contains either httpApi or the authorized username from the configuration.
SOAP This field shows either soapApi or the authorized username from the configuration.
JMS

This field displays jmsApi.

Note: When a response is made and the protocol is JMS, the User ID field shows both the protocol and the queue name with Protocol-QueueName as its syntax (for example, JMS-Queue).
MQ

This field shows mqApi.

Note: When a response is made and the protocol is MQ, the User ID field shows both the protocol and the queue name with Protocol-QueueName as its syntax (for example, MQ-Queue).

For client invocations, the following table describes the user identifer that appears in the search results:

Listener User ID
From the process engine This field contains Process: followed by the process ID (for example, Process: 5001).
From a user interface call (that is, non-process) This field shows the name of the user who is processing the operation.

 
Application The application used to send the message.
Operation Interface operation of the logged message. The content for this field is dependent on the Interface drop-down selection. When an interface is selected, only the operations of this interface is displayed.

Event Log

You can view a message ID in the event log by selecting it in the search results area and then clicking Detail > Event Log from the menu bar.

Activities

You can view the activities available for the message ID by selecting it in the search results area and then clicking Detail > Activities from the menu bar.

Sent Data

To view the data that was sent by a selected message, do the following:

  1. Select a message from the search results and click Sent Data > Sent Data from the result area's menu bar.
  2. A popup dialog appears, which shows the data in XML format that was sent by the selected message.

Received Data

To view the data that was received by a selected message, do the following:

  1. Select a message from the search results and click Recv Data > Recv Data from the result area's menu bar.
  2. A popup dialog appears, which shows the data in XML format that was received by the selected message.

Sent Properties

To view a message's sent properties, complete these steps:

  1. Select the message that you want and then click Sent Data > Sent Properties from the result area's menu bar.
  2. A popup dialog appears, which shows the message's sent properties.

Received Properties

To view a message's received properties, complete these steps:

  1. Select the message that you want from the search results and then click Recv Data > Recv Properties from the result area's menu bar.
  2. A popup dialog appears, which shows the message's received properties.

Re-send Data

To resend data from a message, complete the following steps:

  1. Select the message that you want from the search results and then click the Sent Data > Re-Send from the result area's menu bar.
  2. A popup dialog appears, which allows you to edit your message before you click the Re-Send option.
