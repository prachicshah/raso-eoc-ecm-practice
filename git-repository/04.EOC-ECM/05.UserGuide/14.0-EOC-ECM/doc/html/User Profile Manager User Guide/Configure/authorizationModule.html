Authorization Module

The Authorization Module provides the ability to specify a third party authorization provider that will return a set of privileges held for a given user. The authorization provider is responsible for providing and maintaining privileges in the system, and granting privileges to groups. You can do a custom implementation of authorization provider. It is noted that custom authorization provider must provide implementation to the following APIs at minimum:

  • security.getGroupPrivilegesByKey
  • security.getPrivilegeByKey
  • security.getPrivilegeGroupsByKey
  • security.queryPrivilege


The Authorization Module page allows you to select an authorization module that can be specified by the Implementation of security APIs, and add its authorization data. To access this page, click Configure > Authorization Module from the menu bar.

To add authorization data to an authorization module, do the following:

  1. From the Authorization Module page, click the Authorization Module field and select an authorization module from the list.
  2. Specify your data in the Authorization Data field, and then click the Save button to save your changes.
  3. A confirmation message appears, indicating that your changes have been saved.
