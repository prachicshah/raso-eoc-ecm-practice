Use LDAP with a Windows NT ID

You can user an LDAP provider with a Windows NT ID to log into an application by following these steps:

  1. Run any application from Velocity Studio.
  2. Log in to the application using upadmin as both the Username and Password. The security provider is currently set as ConceptWave, which is Oracle.
  3. Open the User Profile Management application, and then click Configure > Security Providers from the menu bar and create an LDAP provider.
  4. For the Security Type field, select LDAP.
  5. For the Security Data field, enter your LDAP data. The following is an example: 
    ldap://carta:389;username@domain.com
    Click the Save button.
  6. Create a user with a SAM account name (sAMAccountName) defined in the LDAP server domain controller and give the account upadmin privileges (for example, firstName.lastName).

    Note: The SAM account name is the same as your NT ID.
  7. Log back in to the application using your SAM account name username (for example, firstName.lastName) and your NT ID password. You are logged in to the application.

    Notes:
    • When you log in using upadmin as both your username and password, it will not work.
    • When you attempt to log in without a password, an error occurs, indicating that the password is missing.
