Create a Privilege

User profile privileges define the business task that particular users, through their group associations, are allowed to perform. Privileges can be assigned to groups only. A user privilege is a combination of all privileges for which the user is active. A user may be active or inactive in more than one group. As a result, if a user belongs to more than one group, that user has combined privileges of all the groups.

The User Profile Manager contains predefined system privileges. You can also create additional privileges.

Note: You can set up multiple profile sources (for example, one profile for each individual partner). A user in the profile provider instance with the Local Profile Administrator (LPAdmin) privilege can only create, change, delete, and update users within the profile provider. A user with this privilege can also only grant privileges to groups within the profile provider and may grant equal privileges to other groups within the profile provider

To create a new privilege, perform these steps:

  1. From the menu bar, click Create > Privilege to launch the New Privilege page.



  2. Complete the following fields:

    Field Description
    Privilege

    Enter the privilege name. This field is mandatory. Once you have created this privilege, it cannot be renamed. Field restrictions are dependent on the security profile provider being used.
    Description Enter a description for this privilege. This field is mandatory.

  3. Click the Save button to create your new privilege. A confirmation message appears under the menu bar, indicating that your privilege has been successfully saved.
  4. To create another privilege, click the New button. Otherwise, you can configure your new privilege using the Groups tab, or at a later time by clicking Manage > Privileges from the menu bar.

Groups Tab

The Groups tab shows a list of groups to which the user belongs.

You can perform the following actions from the Users tab:

  • Add a group to a privilege
  • Remove a group from a privilege

Add a Group to a Privilege

To add a group to a privilege, complete these steps:

  1. Click the Groups tab, and then click the Add button to launch the Search Group dialog.



  2. Select the checkbox next to the Group Name that you want to add to the user, and then click the Select button.

    Note: Groups must be previously created to be shown in the Search Group dialog. 



  3. The Groups page appears. Your selected group has been added to your user.

Remove a Group from a Privilege

To remove a group from a privilege, complete these steps:

  1. From the Groups tab, select the checkbox next to the Profile Provider that you want to remove from the user, and then click the Remove button.



  2. The Groups tab refreshes and a confirmation message appears, indicating that the group has been successfully removed from the privilege.

User Profile Privileges

The following table contains predefined system privileges that are available in the User Profile Management application:

Privilege Description
Add Shared Favorites Add shared favourites.
Administration App Administration application access (required for all administration users).
ConceptWave API Access Access to product APIs (used by external systems)
Delegate Task Delegate assigned tasks to other users.
Delete Shared Favorites Delete shared favourites.
Everyone Standard privilege that is automatically assigned to each user.
Get Available Get next shared task. Without this privilege the user will not be able to get shared tasks from the unassigned task queue.
Group Manager Delegate unassigned tasks.
Process Manager Administrator Process management functions (start, stop, resume, suspend, and so on.)
Return Task Return task to shared queue (set task to unassigned).
Runtime Administration Runtime framework administration (view event log, status, etc.). Not applicable to System Administration Application.
Set Process Priority Modify process priority
Show Error Details Shows the actual detailed error message instead of the generic error message, "An error occurred and has been logged with reference number nnn. Please report this to your system administrator."
Take on Tasks Take tasks from other users.
User Profile Administrator User profile administrator access.
Workgroup Select Take shared task from the list of unassigned tasks. Without this privilege only the next assigned task may be taken.
Worklist Administrator Perform worklist management functions (assign, reassign and modify tasks).
