Open Task

To access the Task Details user interface, select the checkbox next to a single task, and then click the Open button.

The Task Details page appears, which provides a single-task view made up of three parts:

  • The task summary, which is the top pane
  • The task menu, which consists of actions available on the current task
  • The task user interface, which is the bottom pane

Task Summary

The task summary is a short version of a Task Detail form (see Task Details). The task summary section displays four noteworthy task fields:  

  • Task
  • Priority
  • Start Date
  • Due Date

Task Menu

The task menu provides set of actions and functionalities available on the current tasks. This set of functionalities and actions is similar to the User Worklist menu with some minor changes. The main difference is that local task actions apply to the current task only, while User Worklist actions work on all selected tasks.

The task menu contains the following set of actions and functionalities:

Action Description
Start Work

This option is visible if the current task has not been started. See Start Work for details.

Once task is started, the Start Work option disappears and the Start Date field is updated to display the new date and time.
Acknowledge

This option is visible if the current task is either a taen task or an alert. See Acknowledge for details.

Once acknowledged, the task is removed from the worklist, and the task view is refreshed to display the task summary only. Neither the task menu, nor the task user interface is applicable to the acknowledged task. Once the user navigates back to the User Worklist or to another task using the Previous or Next navigation buttons, the acknowledged task is no longer accessible.
Delegate

See Delegate for details. Once delegated, the task is removed from the worklist and and the current task view is refreshed to display the task summary only. Once the user navigates away from the removed task’s view, it is no longer accessible.
Return

The Return option is visible to users having the Return Task privilege in User Profile Management. This option allows the user to unassign his tasks (that is, return them to the available task queue).By default, a confirmation dialog appears, prompting the user to accept unassigning the selected worklist tasks.

See Return for details. Once the task is returned, it is removed from the worklist and current task view is refreshed to display the task summary only. Once the user navigates away from the removed task’s view, it is no longer accessible.
Actions

Task actions display if the task is a regular started task. The action displays only if it applies to the current task. See Task Actions for details.

Once an action is completed, it can either close the task or leave it open, based on the action’s Keep Open flag. If the task is closed, it is removed from the worklist and the current task view is refreshed to display the task summary only. Once the user navigates away from the removed task’s view, it is no longer accessible. Otherwise, task remains open and task view remains unchanged. User continues working on this task.
Task User Interface

The worklist task’s user interface comes from the task’s Participant user interface. As a result, all tasks belonging to the same participant share same user interface.

By default, regular tasks and alerts display the order associated with the task. If the task is also associated with an order item, the order selection is set on this item. When a task does not have an order, it displays the process document. If the task’s process does not have a document, the user interface is empty.

Taken Task, by default, displays a form with two fields:

  • Taken By
  • Reason
