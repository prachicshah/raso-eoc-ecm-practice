Participant Task

A task can be created in one of the following ways:

  • By a business process
  • Calling the Process.createWorklistAction() method
  • Publishing the WORKLIST_TASK_CREATE  event

Regardless of how the task is created, each task is based on the specified a One-Way participant operation responsible for task generation. As a result, each task is associated with a participant and a participant operation, and these metadata objects define the task’s core parameters.

The task participant defines privileges for the task. A task can be acquired and worked on only by users having privileges specified by participant.

Participant configuration also defines the distribution of new tasks. If the cwOnPartDistrib() participant method is implemented, the newly created task is automatically assigned to the user returned by this script. Moreover, task assignment can be defined by the Participant Distribution type and tasks can be set to be automatically distributed by the system, manually assigned by administrator, or allowed to be selected from a shared queue by the user. See the Participant documentation for further details.

The Participant’s Duration setting is used to calculate the Due Date for tasks.

Participant interface operations of type Notification represent a list of actions that can be performed on the task. See the Task Actions section for further details.

The participant’s User Interface represents the task’s User Interface.

Note: Each participant that is used to create a worklist task must create a User Interface in your metadata. Otherwise, opening and displaying a task results in an exception.
