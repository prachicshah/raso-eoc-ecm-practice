Worklist Refresh

The User Worklist is a dynamic finder and can be changed often for the following reasons:

  • Tasks are created and assigned
  • Some tasks are taken by other users resulting in new taken tasks
  • New alerts are generated

As a result, the User Worklist must be systematically refreshed, either automatically or manually.

Automatic Refresh

By default, the User Worklist is set up to be refreshed automatically every five minutes. You can configure this value in the Configuration application. From the System > System Parameters page, under the Framework Parameters section, change the value for the Worklist Poll Time parameter.

The worklist refresh time can also be set up in the metadata by going to your Worklist finder User Interface and setting the Timer Duration. This way, if you have multiple worklist finders, they can all be set up to refresh at different time intervals. Once worklist is refreshed, any status message displaying from previous worklist actions is removed. Task selection is preserved if and only if selection consists of a single task. Selection of multiple tasks is lost.

Manual Refresh

To manually refresh worklist, click the menu Task (Number) from the top right menu bar. The User Worklist reloads and displays the up-to-date list of tasks.

Some user actions, such as delegating the task and returning to queue, are set up to fully refresh the current worklist. When user changes his own worklist, most changes are instantly reflected.
