Migration through Velocity Studio

When you are upgrading from an older version 5.x release to the latest version 14.x release, your metadata requires migration. Such a migration is required due to product changes.

Metadata Migration

To migrate your metadata, do the following:

  1. Open Velocity Studio.
  2. Open your metadata by clicking File > Open Project from the menu bar.
  3. A popup appears, indicating that your metadata requires migration.



  4. Click the OK button to begin the migration of your metadata.
  5. A progress bar appears, indicating the status of your metadata migration.
  6. A message displays, indicating that your metadata has been successfully migrated. Click the OK button.
  7. Save your metadata project.

Database Migration

To migrate your database, do the following:
  1. After migrating the application metadata successfully, click the Database > Upgrade System from the menu bar.
  2. A database connection appears if you database is not connected; set up your bootstrap database connection.

    3. Note: This first time you are not prompted for an application to read information from database schema.

  3. The Upgrade SQL dialog appears, specify the directory and the name of the generated file.
  4. Click the Save button to create the file.
  5. Run the generated SQL file to your database schema.

Migrate Configuration

This next part in migration is to migrate the configuration settings as described in the following steps:

  1. After successful migration metadata and database, click runtime > Run from the menu bar
  2. Create a new application and select the newly created application.
  3. At runtime, login to the System Configuration application and click Configure > System option from the menu bar.
  4. Your 5.X configuration settings are fully migrated to 14.x upon successful processing of the System Configuration application's user interface.

    5. Note: If you have not defined the database schemas, you must define them at this point.

Sometime, the user interface of System Configuration application may not appear at all when you select Configure > System menu. You must refresh the browser page to resolve the issue.

Notes:

  • Your metadata requires migration before it can be packaged and deployed.
  • It is recommended that you make a copy of your metadata before proceeding with migrating it. It is also a best practice to make regular backups of your metadata.
  • If your metadata has FormFrame, Section, or Tab elements with the Form property set to the Form.Default, the migration process will reset the value to Default.
