Welcome to Velocity Studio

The Velocity Studio is a J2EE-compliant software development platform and services for telecommunication providers to build, deploy and manage OSS/BSS enterprise solutions.

The following figure shows an overview of software development and deployment.

Applications are metadata-based, which is developed in Velocity Studio, a graphical software development environment. Velocity Studio can run the metadata, which is capable of serving the Web interface of the application without an external application server.

An Oracle database is required to enable Velocity Studio to run the application. The metadata, which is stored in a local directory in file system, is expected to be version controlled using an external version control system.

For production environment, the metadata is packaged and deployed to J2EE container. There are two types of runtime that may be deployed:

  • Application Server -- runtime that serves users at web interface and execute related business logic, such as order entry, managing worklist and creating user processes.
  • Process Engine -- specialized runtime that handles activities of business processes as defined by metadata. These processes may be user processes such as managing orders, global processes such as scheduled tasks, and interface processes such as handling requests/responses of an external interface.

The Application Virtual Machine (AVM) -- the framework technology beneath any runtime built with Velocity Studio -- includes a Web-based configuration application and a system administration application that helps manage the application such as external interface parameters, caching and logging settings, process management, and user profile management.

Leveraging J2EE technology, the application (both Application Server and Process Engine) can be clustered across multiple nodes for load distribution. Runtimes require an Oracle database for information storage. See certification matrix for the list of certified application servers, browsers, and database for Velocity Studio.

Getting Started

If you are new to Velocity Studio, start by referring to the Velocity Studio User Guide. You may want to consider signing up for Velocity Studio training courses by contacting us for details.

If you are working with metadata that is built by a product release prior to version 14.x (for example, version 5.x), you must migrate your metadata before starting any development with the current product release.
