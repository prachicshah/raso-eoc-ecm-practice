Installation Procedure

The different types of installations and upgrades are defined by the top level playlists described in the Playlist Details section. To install or upgrade an application you need to create a job and run a top level playlist. For information about the SUF configuration variables, refer to the Prerequisites section.

Access the menu Job > Current. Click Create New. Follow these steps:

  1. Playlist Selection (Job - Add - Step 1 / 8):

    1. Enter the Job Name for the type of installation or upgrade.
    2. Node Type: Select AVM.
    3. Package Name: Select PSCM.
    4. Select the playlist appropriate for the type of installation or upgrade. For example, for a brand new installation choose the _PSCM_NewInstallation.xml playlist to execute.
    5. Click Next >.

  2. File Selection (Job - Add - Step 2/ 8):

    3. Based on the playlist selected, SUF asks to Select an AVM installation package or Select application package or both.
    1. In the Select an AVM installation package field, select swtools/AVM_Platform.rpm.
    2. In the Select application package field, select swtools/AVM_BSSF_PSCM.rpm.
    3. In the Select host node mapping file field, select swtools/host_node_mapping.properties.
    4. In the Select the application configuration file field, select swtools/pscm_config.xml.

      5. Note: This selection is optional. It is only required to override the default application configuration. The file pscm_config.xml is used to override the application default file config.xml that is found in AVM_BSSF_PSCM.rpm file.
    5. Click Next >.

  3. Node Selection (Job - Add - Step 3/ 8):

    1. Select JBoss node: Select PSCM_NETWORK>AVM>PSCM.
    2. Select Oracle node: Select PSCM_NETWORK>AVM>Oracle.
    3. Click Next >.

  4. Variable Selection (Job - Add - Step 4/ 8):

    Enter the values for the following fields. For more information refer to Prerequisites.
    1. Schema Name
      Note: The DB information is used to create a new database user, if it does not already exist. The user with the same name is dropped. The same credentials are used to upgrade the system.
    2. Schema Password
    3. Oracle System User
    4. Oracle System Password
    5. Oracle Port
    6. Oracle SID
    7. Application Internal Name: BSSF_PSCM
    8. Application Version: The application version must be set to 1.
    9. Web Application Name: The default value is cwf. When multiple applications are to be used on the same environment then provide an appropriate name. For example, for the ECM application, use cwf_PSCM.
    10. JNDI Datasource Name: The default value isEricssonConfig. When multiple applications are to be used on the same JNDI Data Source, there is no need to change this value. Otherwise, provide a new name.
    11. JMX Domain Name: Use default value com.Ericsson.AVM.
    12. JMX Application Name: BSSF_PSCM
    13. License Type: Select option FILE.
      14. Two types of licenses are supported:

      • File license: The folder for this type of license contains all license files for different JBoss servers and the files that need to be named as {HOST.name}_avm_license.lic.
      • RMS license: The folder for this type of license contains .ini and .cfg files which are needed for an RMS license.

    14. License Folder: For a new installation, specify the location in the field. The location of JBoss license folder on SUF server: /root/lics. This folder contains all licenses files to be distributed to the JBoss servers. The naming convention is important.
    15. Click Next >.

  5. Configuration Selection (Job - Add - Step 5/ 8):
    There are no node selections available for this step. Click Next >.

  6. Step and Task Selection (Job - Add - Step 6/ 8):

    7. SUF lists all the invoked playlists for the top level playlist selected. When running a top level playlist, each invoked playlist is processed as a step.
    By default, all the invoked task-specific playlist steps are selected. It is recommended to execute all these steps.

    Add a User Defined Checkpoint.

    A checkpoint can be added on any option displayed on this page. To add a checkpoint, do the following:
    1. Select a playlist, for example DB_Upgrade and click the Add Checkpoint button.
    2. The User Defined Checkpoint dialogue appears. Enter the Checkpoint ID and Checkpoint Description fields.
    3. Click Save.
    4. Click Next >.

  7. Checkpoint Definition (Job - Add - Step 7/ 8):
    Any user defined checkpoints configured in step 6 Step and Task Selection appear here.
    1. Choose option User to Confirm or Wait Until.
    2. Click Next >.

  8. Creation Summary (Job - Add - Step 8/ 8):

    9. SUF displays the summary details of the job. It is possible to go back to the previous steps to make corrections.
    1. Click Apply to create the new job.
    2. Click Done to return to the Job List page.

Note: Verify the job and execute.
