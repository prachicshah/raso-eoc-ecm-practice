SUF Server Configuration

Login to the SUF server as an administrator using the ‘nmtroot’ user account and do the following steps:

  • Create a Node Type
  • Create a User Class
  • Create a User
  • Import Network Configuration
  • Import Playlists
  • Verify Prerequisites Job

Create a Node Type

  1. Access the menu Administration > Node Type.
  2. Create a new Node Type ‘AVM’ (case sensitive) with a Description.
  3. Click the Apply button

Create a User Class

  1. Access the menu Administration > General > User Classes
  2. Enter the value in the User Class Name field.
  3. Enter a Description.
  4. In the Network Permissions section, select all permissions (‘Admin’, ‘Editor’ and ‘Viewer’).
  5. In the Other Permissions section, for node type ‘AVM’, select all permissions (‘Admin’, ‘Editor’ and ‘Viewer’) under the columns Repository and Job Permissions.
  6. Click the Apply button

Create a User

  1. Access the menu Administration > General > Users
  2. Enter a value in the User Name, User ID and Password fields.
  3. Enter the e-mail information and from the list of available classes, select the user class created in the SUF Server Configuration > Create User Class section.
  4. Click the Apply button.
  5. Logout from the SUF server and then log back in with the user name created in step 2.

Import Network Configuration

  1. Login to the SUF Server with the user name created in the SUF Server Configuration > Create a User section.
  2. From the Network menu, click the Import button.
  3. From the Network Configuration File field, click the Browse button and select the file network.xml.
  4. Edit the network connections
    Note: It is possible to modify an existing JBoss host or add a new JBoss host. It is only possible to modify an existing Oracle host. The Oracle node can have only one Oracle instance on a SUF client.
  5. Click the Apply button.
  6. After importing the network file, the network nodes can be edited and verified by clicking a node and pressing the Check button.

Import Playlists

  • Import the AVM playlist library
    1. From the Repository > Library menu, click Import.
    2. The Library - Import page appears.
    3. From the Package File field, browse and select the AVM_SUF.tar.gz file.
    4. Click Apply.
  • Import the PSCM playlist package
    1. From the Repository > Node Package menu, click Import. The Node Package - Import page appears.
    2. In the Package Name field, enter a user defined package name, for example, PSCM.
    3. In the Node Type field, select AVM.
    4. From the Package File field, browse and select the PSCM_SUF.tar.gz file.
    5. Click Apply.

Verify Prerequisites Job

For the first time deployment, a job can be created to verify the prerequisites for executing any other playlists. This job checks the Java Development Kit (JDK) version on the SUF server and clients, and the JBoss installations on the SUF clients.

To verify the prerequisites, do the following:

  1. Access the menu Job > Current
  2. Click the Create New button.
  3. Do the following for the Playlist Selection step:
    • Enter the Job Name and select Node Type AVM.
    • In the Package Name field, enter AVM_SUF.
    • In the Playlist Name field, select the verify_prereq.xml file.

    Note: If you want to run the playlist on multiple nodes, select the Run this playlist on multiple nodes check box. Selection of this step enables an option to add multiple node sets in step 3.

  4. Click the Next button for the subsequent steps: File Selection, Node Selection, Variable Selection, Configuration Setup, Step and Task Selection and Checkpoint Definition.

    5. Note: A checkpoint can be added on the Step and Task Selection step.

  5. The Creation Summary step displays the summary details of the job.
  6. Click the Apply button to create the new job.
  7. Click the Done button to return to the Job List page.

If the job does not run successfully, check if there are any fundamental environment problems, such as the wrong JDK version.
