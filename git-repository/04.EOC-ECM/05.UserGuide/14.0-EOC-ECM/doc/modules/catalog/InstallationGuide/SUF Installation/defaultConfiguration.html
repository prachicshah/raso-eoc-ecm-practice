Default Configuration

Business Configuration

There are default configurations for business entities such as Product Type, Offer Type, etc. that are deployed with ECM. Any additional configuration can be done by the user via the Codes Table.

The following predefined tariff time specification is also published to CIL during the installation of ECM.

  • TariffTime_Spec

The product, service and pricing entities and relationships of the following sample plans are loaded with the deployment of ECM:

  • Silver Line
  • Gold Nugget

Recurrence specification with default recurrent event configurations:

  • Daily
  • Weekly
  • Monthly
  • Yearly

Resource specifications with characteristics:

  • MSISDN-PCS
  • MSISDN-DCS
  • ISMI
  • Numbering Scheme
  • Serial Number
  • IPv4 Address
  • IPv6 Address

System Configuration

  • Node: PEUI node is defined and active
  • Configuration Variables:
    • PSCM_CILSERVER_HOST: Host from CIL
    • PSCM_CILSERVER_PORT: 8184 (to be replaced with port from CIL)
    • PSCM_CILSERVER_ROOT: /rest
  • Default Calendar. USA
  • Privileges: Default privileges for catalog
  • User: All privileges as defined by upadmin user
  • Catalog Mode: Server
  • Web Services: The following services are active:
    • services.ICFSSpecification (HTTP)
    • services.ICharacteristics (HTTP)
    • services.IProductOffering(HTTP)
    • services.IProductSpecification(HTTP)
    • services.IRFSSpecification(HTTP)
    • services.IResourceSpecification (HTTP)
    • worklist.service.worklist(SOAP).
  • Charging Capabilities
    The application configuration file contains the following default configuration for the charging capabilities:

      • Capability Group Service: From the bmarp.wsInterface.capabilityGroupService node, create a new port (sub-node). Configure as follows:

      • .Port Name: portHTTP
      • Binding: bmarp.wsInterface.capabilityGroupBindingHTTP
      • Provider Type: HTTP
      • Active: Set as checked.
      • Provider Settings > Location: http://localhost:xxxx/charging/capabilities/{groupName}

      Function Definition Service: .From the bmarp.wsInterface.funcitonDefinitionService node, create a new port (sub-node). Configure as follows:

      • Port Name: serviceDeterminationPortHTTP
      • Binding: bmarp.wsInterface.functionDefinitionsBindingHTTP
      • Provider Type: HTTP
      • Active: Set as checked.
      • Provider Settings > Location: http://localhost:xxxx/charging/capabilities/FunctionDefinitions

      Service Determination Service: From the bmarp.wsInterface.serviceDetermination node, create a new port (sub-node). Configure as follows:

      • Port Name: serviceDeterminationPortHTTP
      • Binding: bmarp.wsInterface.serviceDeterminationBindingHTTP
      • Provider Type: HTTP
      • Active: Set as checked.
      • Provider Settings > Location:http://localhost:xxxx/charging/capabilities/ServiceIdentification

    After the ECM application is deployed, the localhost and the port number (xxxx) of the Provider Settings > Location must be configured.
    For more information on how to configure the location of webservices for these charging capabilities, refer to the Operations and Maintenance > System Configuration > Configure Charging Capabilities section.
