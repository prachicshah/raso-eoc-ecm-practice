SUF Installation - Prerequisites

The following are installation prerequisites:

  1. The following software package files are available:

    • AVM_Platform.rpm
    • AVM_SUF.tar.gz
    • AVM_BSSF_PSCM.rpm
    • network.xml
    • host_node_mapping.properties
    • pscm_config.xml

    Note: The pscm_config.xml file is optional and is used to override the application default file config.xml that is found in AVM_BSSF_PSCM.rpm file.

  2. AVM licences are available: One for the SUF Server and one for each ECM application server being installed.
  3. All steps listed in the Installation Procedure section of the Execution Environment Installation have been performed.
  4. The xsltproc command is required on the SUF server for a SUF installation. Execute “yum install libxslt”.
  5. The Oracle DB node required by the ECM application must be installed and available.
  6. The SUF server, AVM SUF package and Java versions must be compatible with each other.
  7. The following SUF configuration variables are available:

    SUF Configuration Variables Description
    Oracle System User

    User name of the Oracle system user. Default value is ‘sys’.
    Oracle System Password Password of the Oracle system user. Default value is 'sysadm'.
    Schema Name Database user name. During a new installation a user name can be specified, but during an upgrade the same name must be used as was initially specified.
    Example: PSCM
    Schema Password Database password. During a new installation a password can be specified, but during an upgrade the same password must be used as was initially specified.
    Example: PSCM
    Oracle SID Oracle Service ID. Example: PSCMDB
    Oracle Port Oracle Port has a default value of 1521.
