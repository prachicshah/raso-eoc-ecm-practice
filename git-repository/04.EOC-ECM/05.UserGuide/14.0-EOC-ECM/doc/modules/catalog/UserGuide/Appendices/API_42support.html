Supported Version 4.2 function APIs in Catalog 5.x

The following table shows Catalog version 4.2 functions, what their corresponding Catalog version 5.x functions are, and descriptions of the version 5.x functions.

Version 4.2 API Name Corresponding Version 5.x API Description
Catalog.setGlobalContext() Catalog.getGlobalContext() You can create a family of variables using the Context object. Once the context is set as opened, the variables can be set or accessed using theContext.
Catalog.getCurrentCatalogDate() getCatalogTime() Use the getCatalogTime() method from the Catalog class.
Catalog.getImageURL() getImageURL() Use the getImageURL() method from the Catalog class.
Catalog._getAttachmentURL() getAttachmentURL()
Use the getAttachmentURL() method from the Catalog class.
Catalog.__getCatalogImageSize() getImageHeight(), getImageWidth() Use the getImageHeight() and getImageWidth() methods from the Catalog class.
Catalog.extendsDocument() isBasedOn() Use the isBasedOn() method from the CwObject class.
Catalog.getCTCodes() getCTCodesAndDescriptions()
Use the getCTCodesAndDescriptions() method from the Global class.

The following is the current list of version 4.2 APIs that are supported in this release:

  • Catalog.getCurrentComponent()
  • Catalog.getCurrentOffer()
  • Catalog.getCurrentOrder()
  • Catalog.getCurrentPriceItem()
  • Catalog.getCurrentProduct()
  • Catalog.setCurrentComponent()
  • Catalog.setCurrentOffer()
  • Catalog.setCurrentOrder()
  • Catalog.setCurrentPriceItem()
  • Catalog.setCurrentProduct()
