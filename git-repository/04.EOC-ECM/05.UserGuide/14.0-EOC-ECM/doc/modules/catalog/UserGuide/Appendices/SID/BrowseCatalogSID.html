BrowseCatalog SID Interface

The BrowseCatalog interface browses the Catalog for all available items for a given search criteria and it returns a list of items. The BrowseCatalog interface is available in the Library files of Velocity Studio, and contains input, output, and fault data structures defined as follows:

Input

The input file contains the following elements and data types defined in Velocity Studio's data dictionary's cwt_pcsid.browseCatalogRequest data structure:

Input DataType Element Notes
hierarchyCode String

    
depth Integer    
date DateTime   Refer to Velocity Studio's documentation for more details on the DateTime format.
language String    
searchCriteria[] array cwt_pcsid.searchCriteria Contains the following elements:
  • key (string)
  • operation (string)
  • value (string)
context[] array cwt_pcsid.context

Contains the following elements:

  • code (string)
  • attributes(array)
    • nameValuePair
      • name (string)
      • value (string)
  • label (string)
inventory   cwt_pcsid.inventory

Contains the following element:

  • id (string)

Output

The output file contains the following elements and data types defined in the cwt_psid data structure (queryResult).

Output DataType Element Notes
resultTime Date

    
queryType Integer    
fromNode String    
hasPartialResults Boolean    
hasErrors Boolean    
errors

Array / String

    
hierarchyNodes Array pcsid.hierarchyNode

Contains the following elements:

  • code (string)
  • items
    • browseResponseItems
      • browseResponseItem
        • code (string)
        • label (string)
  • hierarchyNodes (array)
    • code (string)
    • items (array)
    • hierarchyNodes (array)
  • label (string)

Input and Output File Examples

These input and output files are examples using a SoapUI tool.

InputXML: 
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Header/>
   <soapenv:Body>
      <browseCatalogRequest>
         <!--Optional:-->
         <!--Optional:-->
         <hierarchyCode>virtualdesktop</hierarchyCode>
         <!--Optional:-->
         <depth>1</depth><date>2011-04-11T00:00:00.000-04:00</date>
         <!--Optional:-->
         <language>?</language>
         <!--Zero or more repetitions:-->
         <searchCriteria>
            <!--Optional:-->
            <!--Optional:-->
            <key>cpu</key>
            <!--Optional:-->
            <operation>==</operation><value>Single Core</value>
         </searchCriteria>
         <!--Optional:-->
         <context>
            <!--Optional:-->
            <code>?</code>
            <!--Zero or more repetitions:-->
            <attributes>
               <!--Optional:-->
               <name>?</name>
               <!--Optional:-->
               <value>?</value>
            </attributes>
            <!--Optional:-->
            <label>?</label>
         </context>
         <!--Optional:-->
         <inventory>
            <!--Optional:-->
            <id>?</id>
         </inventory>
      </browseCatalogRequest>
   </soapenv:Body>
</soapenv:Envelope>
Output XML: 
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/">
   <soapenv:Body>
      <queryResult>
         <resultTime>2011-04-18T13:52:40-04:00</resultTime>
         <queryType>3</queryType>
         <fromNode>virtualdesktop</fromNode>
         <hasPartialResults>0</hasPartialResults>
         <hierarchyNode>
            <code>virtualdesktop</code>
            <browseResponseItems>
               <browseResponseItem>
                  <code>basicdesktop</code>
                  <label>Basic Desktop</label>
               </browseResponseItem>
            </browseResponseItems>
         </hierarchyNode>
      </queryResult>
   </soapenv:Body>
</soapenv:Envelope>
