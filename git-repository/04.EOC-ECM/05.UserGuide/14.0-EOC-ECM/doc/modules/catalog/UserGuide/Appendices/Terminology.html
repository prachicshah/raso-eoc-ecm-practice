Terminology

The following table describes the terminology:
Term Description
Catalog The functionality and information provided by the Catalog products.
Catalog Solution pack (template) The two Catalog solution packs: Catalog server and Catalog client solution pack.
Catalog maintenance server An AVM node that runs the Catalog Server application and allows maintaining the Catalog definitions. There may be Catalog Server nodes that do not allow maintenance. Such nodes are used to increase the scale and reduce the Catalog response time.
Catalog item A definition of product, service, etc. Items refer to the things that a company sells to its customers.
Catalog definitions The information describing the catalog items and other objects and associations that is stored in the Catalog database. The Catalog definitions are de facto metadata – they describe what information is to be collected at order entry time and what rules have to be applied during creation, modification, deletion and pricing of the user shopping baskets.
Catalog elements The atomic components of the Catalog definitions.
Catalog objects All Catalog elements which may exist and be changed independently – for example items, attribute types, charge types, etc. The applications using the Catalog perform operations on the Catalog items. The implementation of the operations uses the other objects to fulfill its goals.
Catalog associations The links between Catalog objects and other objects and or data.
Shopping basket A set of Catalog basket items and their associations that specify the products and services the user wants to get and/or already have. The basket consists of context, items, item associations and prices.
Basket item The data representing a runtime instance of a Catalog item definition. It is part of a basket and is implemented as dynamic document.
Order context A set of named data elements that are used in the Catalog decision making process. The Catalog order context gives information about the location and consumer of the products and services defined in the Catalog.
Item association A link between an item and a property of the item. The property can be another item, attribute, charge formula, rule, image, specification, label, etc.
Basket association A runtime instance of an association between two basket items. There is only one association between two items in the basket.
Catalog rules User specified units of logic stored in the Catalog database. The rules can be linked to one item only (item rules) or reused in many items (Catalog rules). Rules can be changed dynamically and do not require Catalog shutdown and restart.
Info models Definitions of tabular data used by the Catalog for decision making or calculations.
Info tables Tabular data structures that are defined by the Info models and are used in the Catalog rules.
Static Info tables Info tables that have static content and may exist in many versions.
Dynamic Info tables Info tables that do not have predefined content and which rows are dynamically created (computed) at runtime when requested. These tables provide mechanism for using metadata scripts defined in the Catalog Server application by the Catalog Client rules that do not have access to the server metadata (see the document “Info Models�?).
Catalog projects Mechanism for secure change of the Catalog definitions, controlling the access of the Catalog maintenance teams to the Catalog data.
Code A string of characters used to identify uniquely a Catalog object. If used in JavaScript the code is a string variable or constant.
Name An identifier that identifies uniquely an item association and can be used in the JavaScript dot expression.
