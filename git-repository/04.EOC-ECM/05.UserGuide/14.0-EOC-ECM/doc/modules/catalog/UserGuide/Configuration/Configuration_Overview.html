Configuration Overview

The Catalog Designer allows you to configure various aspects of your catalog. This section contains the following topics:

Code Tables
The Code Tables feature allows the maintenance of all enumeration lists used in an application and the maintenance of code tables and codes.

Catalog Configuration
This page allows you to set default currency, markup, date, and privileges for the Catalog.

Configure PLD
This topic allows you to configure templates, stages, participants, and microflows to work with Orchestration Framework application.

Catalog Domains
Catalog Domains features access of a catalog object instance (or record) to the users associated with that record. Users can be associated to a position in the Organizational Chart and a position can be associated with a Domain or Owner on an instance or record.

Catalog Privileges
Assign catalog privileges to a catalog object which dictates the user access to that object. Catalog privileges are created and associated to groups of users in the System Administration application.

Exporting Catalog
The Exporting Catalog page describes how you can export Catalog data and different levels of exports that are available.

Importing Catalog
The Importing Catalog page details how you can import data, including images, attachments, and files to your Catalog.

Import and Export Catalog through SQL
Catalog allows you to export catalog data directly from an Oracle server to external tables placed on the file system. You can also import external table files from the file system to your Oracle database.

Import and Export Catalog in XML Format
The import and export functions allow you to transfer catalog data between different environments. You can export catalog data to an XML file, which you can open in Excel and view it as a spreadsheet.

Validate Project
The Validate Project function allows you to validate a project.

Validate Catalog
The Validate Catalog function allows you to validate your catalog data.

Configuration Variables
Configuration variables are specific system variables that you can define to adjust system settings in the Configuration application.

Version Code Tables
A version a code table can be added to a project, which allows the user to define both the start and end dates for each code within a code table. These version code tables can be used when defining an attribute in a catalog.
