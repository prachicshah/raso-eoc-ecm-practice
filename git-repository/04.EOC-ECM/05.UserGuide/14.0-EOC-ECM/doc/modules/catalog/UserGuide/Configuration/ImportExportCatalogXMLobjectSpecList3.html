XML Object Specifications: Restriction to Tax Model Item

When you use the XML file to import and export your catalog data, all objects have a specific format. In addition, each object`s fields have defined meaning and a required format.

The following is a list of objects, which includes a short description of the object itself and the fields that define it. The Mandatory column indicates whether the field is mandatory when importing.

  • Restriction
  • Relation type
  • Relation type attribute
  • Project
  • Project assignment
  • System identifier
  • Tax
  • Tax model
  • Tax model item

Restriction

This object defines restrictions for various objects.

The following table describes the object's field names, their descriptions, and whether these fields are mandatory:

Field Name Description Mandatory?
restrictionName This field denotes the unique code used to identify this restriction. Yes
itemCode This field references the item that this restriction is applied to. Yes
catalogObjectType This field indicates the type of object involved in the association. It can be either an item attribute or an item relation. Yes
catalogObjectCode

This field allows you to specify the exact object that this restriction is applied to, which can be either one of the following:

  • Item attribute code
  • Item relation code
 Yes
startDate

This field denotes the start date for the object in YYYY-MM-DD format.

 Yes
endDate

This field represents the optional end date for the object in YYYY-MM-DD format.

 No
Status

This field indicates the state of the object, which can be one of the following:

  • definition (editable)
  • active (not editable)
 Yes
Immutable This parameter indicates the the object is immutable when set to true. No
isNull This parameter takes a Boolean value, indicating whether the object has a value of null when this field is set to true. No
selected This field denotes the Selected parameter of this object. No
Sequence This field specifies an integer value, which indicates the order of the object. No

defaultValue

 The field contains the object's default value. No
attrValueType

This field denotes the type of attribute value in this restriction as one of the following:

  • I (Intersection)
  • D (Difference)
  • C (Code Table)
 No
Cancel This field accepts a Boolean value. If this field is set to true, there is a restriction if it is inherited from a parent. No
createdBy This field assigns an owner to the object. By default, the owner is set to upadmin. No
createDate This field denotes the date that the object was created. By default, the create date is set as the current date and takes YYYY-MM-DDThh:mm:ss as its format. No
updatedBy This field specifies the user that last changed the object. By default, the user is set to upadmin. No
updateDate This field denotes the date that the object was last updated. By default, the update date is set to the current date and takes YYYY-MM-DDThh:mm:ss as its format. No
