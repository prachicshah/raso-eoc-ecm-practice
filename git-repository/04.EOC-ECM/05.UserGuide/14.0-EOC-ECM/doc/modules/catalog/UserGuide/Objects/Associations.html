Association Types

Associations define all characteristics of the item (for example, content, structure, business rules, business name, image, specification, and so on). Association Types can be created within the Catalog to define items with any level of detail and any rule complexity.

In general, an association links a Catalog object to another Catalog object or set of data items. The majority of associations are item associations. Some associations have a static, predefined set of properties. Other associations allow user-defined content, such as a list of attributes. The values of these attributes are defined within the Catalog and are immutable at runtime (that is, they are association properties).

When you create a new item, the associations that are created are grouped by association type and are listed as a drop-down list for the Type field.
