Create an Association

To create a new association, complete these steps:

  1. In Catalog Designer, open a project.
  2. From the menu bar, click Designer > Association Types to launch the Association Type Finder page.
  3. Click the Add button to launch the Association Type Detail page.



    Complete the following fields:

    Field Description Rules
    Code This code uniquely identifies the association's instance name. This code must follow the convention for script identifiers (that is, start with a letter, and contain letters and digits). The Catalog is case-sensitive. The Catalog API and Catalog Rule Language use this code. As a result, it is important that the case in this field matches that used by them. Mandatory, Unique
    Category This field denotes the association category. Each association that you create describes either the structural or behavioural aspects of an item. Click the field's drop-down menu and select a catalog from the list.:

    Category Description Association
    Attachment This category allows an attachment file to be uploaded to the system and associated with a catalog item. Associated to an item
    Description This category allows you to group description categories for a catalog item. The Item Description Association allows for long descriptions to be entered. Associated to an item
    Group Groups allows for restrictions to be placed on the quantity of an item in the Catalog (for example, items added to the shopping basket). Associated to an item
    Image This category allows image files or icons to be uploaded and displayed on the client system. Associated to an item
    Item This category creates relationships between other items. You use this category when creating the item's Relation association. Associated to an item
    Rule This category defines associations between items and rules (for example, validation and condition). Associated to an item
    Item Attribute This category allows for various groupings of item properties to be defined. Associated to an item
    Relation Attribute This category allows for various groupings of relational properties to be defined. Associated to an item
    Price This category is an association type for pricing an item. Associated to an item
    Restriction This category enables restrictions to be created for attribute values. Associated to an item attribute
    Charge Type Description This category allows for grouping characteristics of a chargeable item, such as pricing and discounts. Associated to an item


    Mandatory
    Status Click the drop-down menu and select a status for this Association (for example, Definition, Active, and so on). Mandatory
    Name This field denotes a unique name for the association. Mandatory
    Start Date This field represents the start (effective) date that the association is available. Refer to Start and End dates for details. Optional
    End Date This field indicates the date when the association is no longer effective. Refer to Start and End dates for details. Optional
    Label The visual label used for this association, which is used in the application user interface. This field is a translational string that supports language translations (refer to Velocity Studio's Translation Element for more details). Optional
    Description This field allows you to provide details about the association. Optional
    Project Each object is assigned to a Project. This defaults to the active or opened project when the association is first created. Mandatory

  4. Click the Save button to create your new attribute.
  5. Both the Attributes and Domains tabs appear. To create an attribute for the association, click the Attributes tab, and then click the Add button.
  6. The following information is available for the Association Type Attribute section:

    Field Description
    Code The Code ID for this Context Attribute.
    Attribute Code Search for the attribute type code that you want to associate with this Context.
    Name The name of the attribute for internal use when designing your catalog model. This field is not supported by language translation. This name becomes the context variable name.
    Label The visual Label used for this Attribute, which is used in the application user interface.
    Project Search for the name of the project that holds the Attribute object.

  7. When you have completed entering your information, click the Save button to save your changes to the Association Type Attribute section.



  8. Similarly, for domains, click the Domains tab, and then click the Add button.
  9. Click the Org Chart field's Finder button and select an organizational chart from the list. Click the Save button to save your changes.



  10. Click the association's Save button to save all changes.

Default System Association Types

The Catalog Designer contains the following default system association types with built-in logic.

Association Type Code Category Label Functionality
eligibility Rule Eligibility The AddItemToBasket API invokes this association type to check if a catalog item is eligible to be ordered. Eligibility rules are checked when adding an item to a basket.
contains Item Contains Contains is an association type of the item-item association. It forces the association target to be added to the basket when the association source is added.
groups Group Groups Group is an association type for an item group association and is used for grouping.
validation Rule Validation The validateBasket API invokes this association type to check the validity of a catalog basket.
price Price Price Price is an association type for an item price association and is used for pricing.
availability Rule Availability The Browse Catalog API invokes this association type to check if a catalog time is available for the given user contexts.
