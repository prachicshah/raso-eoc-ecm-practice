Delete Active Items from Catalog Hierarchies

Deleting an active item on-the-fly from a Catalog Hierarchy requrest that you set an end date for the active item or hierachy link.

Suppose that you have a project, P1, that contains the Catalog Hierarchy and is active. The following steps show how to delete active items from the hierarchy:

  1. Create a new project called P2.
  2. Go to project P1.
  3. Click either the Item Hierarchy or the Catalog Hierarchy link. The Item Hierarchy document appears in the right pane.
  4. Click the Change End Date button. In the Project field, select the new project, P2 and set the End Date field.
  5. Click the Save button. At this stage, the Item Hierarchy's end date has not taken effect. However, this change is stored as part of project P2.
  6. To verify what end date has been selected, go to project P2. Notice that the same Item Hierachy appears in this project.
  7. Select the Item Hierarchy under the P2 project.
  8. At the top of the screen, the project Assignment Details section appears, which has an end date that was set in step 4. However, the end date of the Item Hierarchy document remains unchanged.
  9. Activate project P2. The Item Hierarchy's end date is set to the value that you have chosen.

Note: For older projects, to set an end date for an existing Item Hierarchy may not be removed on-the-fly. Instead, you must restart your server for the change to take effect.
