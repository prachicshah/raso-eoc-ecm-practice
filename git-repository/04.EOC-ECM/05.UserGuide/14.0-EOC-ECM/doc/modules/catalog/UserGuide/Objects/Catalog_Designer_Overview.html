Catalog Designer Overview

The Catalog Designer menu allows you to define, control, and maintain Catalog objects, which have properties and attributes associated with them. The menu contains the following options:

  • Attribute Types – definition of the data types used as attributes of different objects
  • Catalog Rules - definition of object and catalog business rules used
  • Taxes – definition of tax types
  • Tax Models – ordered sets of tax types and discounts
  • Charge Types – definition of prices and discounts
  • Info Models – definition of tabular data structures
  • Version Code Tables - definition of codes within code tables
  • Contexts – definition of the sets of attribute types that hold request specific data at runtime
  • Association Types – definition of the associations
  • Catalog Hierarchies - definition of the directory structure used to organize products and offers
  • Projects - mechanism to control changes to the Catalog definition, including releasing changes to a live production environment in a controlled manner
  • Items – definitions of products and services
  • Domains - ability to limit data access
  • Add Project - ability to use the Product Lifecycle Designer (PLD) feature to enable change requests for catalog items to be processed into the live production environment

With the exception of Projects, the Catalog Designer® maintains the aforementioned Catalog objects. Additionally, Catalog Designer® maintains the Info Tables object, which contains the content of tabular data structures defined by the Info Models.

The Catalog items (or simply items) occupy central place among the Catalog objects. Items and their associations usually account for 90% of the Catalog elements. The items represent the products and services that the company offers. The other Catalog objects serve to define some aspects of the items' structure, price, and business rules.

Catalog Objects properties consist of data that is defined, entered, and maintained by the Catalog maintenance application or the API on the Catalog Server. The properties are immutable data for the Catalog client applications. Attributes consist of data that is defined by the Catalog maintenance application or the API on the Catalog server, but is entered and maintained by the Catalog client applications. The values of the object attributes are stored in the shopping basket and service inventory are used in the Catalog rules and Catalog client applications.

Catalog Elements

Conceptually, the Catalog can be presented in an entity-relationship model in which the entities represent the Catalog objects and relationships represent the Catalog associations. The entities and relationships of the Catalog are called Catalog elements.

Catalog elements are versioned or non-versioned. Non-versioned elements exist in one instance only while the versioned elements may have many instances.

All elements have a life span (for example, they have start and end dates). The elements in a production state have mandatory start date, but may not have end date. In this case, their life span is not limited in the future.
