Charge Type Conditions

You can add conditions to a charge type. Conditions are business rules which decide, at runtime, which association (version) apply for particular operation, user, shopping basket and service inventory.

To add a condition to a charge type, from the Conditions tab, click the Add button to launch the Condition Detail section.

The Condition tab has the following buttons:

Action Description
Add Click this button to a new condition.
Delete Deletes the selected condition.
Copy Create a new version with the same values of the selected condition. This button also copies the Charge Type in the selected condition.
Detail View the properties of the selected condition.

The Condition Detail section has the following fields:
Field Description Rules
Code An identifying condition code for this Charge Type. Mandatory
Rule Name Enter a name for the condition. Mandatory
Name Enter a descriptive name for the condition. Optional
Label Enter the label for this condition, which is used in the application user interface. This field can be translated at runtime. Optional
Business Type Select from the options: biweekly, monthly, one time. Options are defined in Associations. Optional
Status Select the status for the Charge Type. Choose between Definition and Active. Mandatory
Start Date The start date of the Charge Type. Mandatory
End Date The period of the Charge Type that will cease to apply. Optional
Message Code Display a pop-up error message if eligibility fails. Mandatory
Project Indicates which project in which the Charge Type is included. Mandatory
Language Specify whether the condition language used in the Script field is either the Catalog rule language or JavaScript. Mandatory
Script Specifies the price recurrence. Values come from the Recurring Charge code table.  Mandatory
