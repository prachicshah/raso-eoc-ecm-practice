Charge Type Details

Charge Types are used to define a pricing model for items and components added to the shopping basket. Using Charge Types, the user is able to define a pricing structure that includes discounts, costs and charges. Conditional charges can be created using the Formula field that uses the Catalog Rules. For example, conditional charges can be created based the components selected. One component is offered for free while a second component is considered an upgrade and contains an additional charge. There are three main Charge Types available for selections: Charge, Discount, and Cost.

The Charge Type details contains various configuration fields that enable flexible pricing model definitions. The price of or discounts on the item can be calculated based on the charges defined in this table. The currency, frequency of the charge applied and pricing type can be configured for various charges. Further conditional charges can be defined using the Catalog Rules (Formula field) and Info Model. The charge can also be associated to a tax model such that the tax model defined is applied to the charge whenever the item added to the shopping basket. Privileges to modify the charge are applied by the Owner field such that only the users with the owner's group have access to modify the record. Below are the three different charge type definitions available Charge, Cost, and Discount.

Create a Charge Type

To create a new Charge Type, do the following:
  1. From the Home page, click Pricing & Tax > Charge Type to launch the Charge Types page.
  2. Click the Add button at the bottom of the finder to launch the Charge Type Detail page.



  3. From the Charge Type Details page, click the Type field and select from one of following charge type options:
    • Charge
    • Cost
    • Discount

Charge

You can configure price information for a component or item using the Charge Type details and by setting the enumerated Type field to Charge.

Notes:

  • You can add an attribute association into a charge type and item charge association.
  • Charge type inheritance allows inheriting an attribute assoication from the parent charge type.
  • Price amounts are also added at the item charge association level to allow value-overridding from the charge type during pricing.

Cost

Item costs can be entered using the enumeration Type field set to Cost. The costs created can be added to the price of an item or shown on an invoice for display purposes only.

Discount

A Discount charge type is used to calculate a reduced price adjustment or an item. A discount can be applied as a flat amount, percentage, coupon, another charge type and conditional formula. The coupon option provides the ability to apply a coupon string name that can be referenced by a third-party application to apply conditions and special pricing discounts to the coupon selected.

External

When the pricing of a basket item is done in an external system, creating a charge type and giving a fixed price beforehand is not possible. As a result, the basket charge needs to be created on the basket item.

When selected, this property indicates that the price information comes from an external system. When you have External selected, the Amount and Formula fields can be left empty, and this charge type can be used to create a basket charge manually.

Notes:

  • No Item Charge is required to be created between the item and the external charge type.
  • The Price() API does not price the external charge.
  • A tax model can be added to the charge, if required. The tax is added during basket charge creation.
