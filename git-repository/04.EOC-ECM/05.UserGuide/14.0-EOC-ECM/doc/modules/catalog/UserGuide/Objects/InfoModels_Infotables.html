Info Table

The info table enables you to enter data for the associated Info Model Attributes. The Info Model Attributes are the table columns of a Standard Info Model Table. The Start and End Dates version the data in this table and the data is stored as a single row in the Data field of the CWPC_INFOTABLE Catalog table. For more information, refer to Info Models.

It is possible to define Info Tables that are Static or Dynamic. The main difference between the Static and Dynamic tables is the way the data is loaded into cache memory by the AVM. The Static tables are loaded in whole automatically whereas the Dynamic tables are loaded one row at a time on demand. There are three types of Static and Dynamic Info Tables that can be referenced in Catalog Management:

Table Static or Dynamic Comments
Standard Static Standard Table is maintained and defined within Catalog. Data values are stored in a standard Catalog table that supports version control.
Database Static Database table is an external table that is referenced by Catalog.
Dynamic Dynamic The Dynamic table is external table that has the same name as the Info Model that defines it.

The info tables are used in the catalog for several major purposes:

  • As data tables used in rules to by the rule decision logic.
  • As price tables to provide data for calculating the catalog items prices.
  • As dynamic table rows uniquely identified by set of keys.

You can access info tables from the Info Model Details. Click Designer > Info Models from the menu bar, and then click the Info Tables tab. The system then displays the Info Table details.

When the info model is active, after adding active attributes to it, the following actions can be performed with the info table:

  • Add an info table
  • Delete an info table
  • Copy an info table
  • View info table details
  • Export an info table
  • Import an info table
