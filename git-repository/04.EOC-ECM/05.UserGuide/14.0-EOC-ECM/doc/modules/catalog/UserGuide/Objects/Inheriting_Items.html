Inherit Items

A Catalog item may inherit (derive) from another catalog item called the base item. Changes in the base item become immediately visible in the derived items. This method of maintaining Catalog information is a powerful mechanism which reduces the cost of maintaining data.

The item inheritance is implemented on the association level, which means that the derived item can add new associations, replace some associations, and remove others. The matching of the associations is performing by association category, type, and name.

The inheritance mechanism works in the following ways:

  • The derived item copies all associations (versions) of the base item and treats them as if they are its own associations
  • If the two association versions has the same start date, the base association version is not copied

The item inheritance mechanism uses the special flag Cancelled (CAN) to indicate that a given association is not needed for the specified period.
