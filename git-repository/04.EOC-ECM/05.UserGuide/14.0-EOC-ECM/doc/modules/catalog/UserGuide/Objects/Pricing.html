Pricing and Discounts

The Charge Type object is the most significant entity for defining pricing in the Product Catalog, providing flexibility on implementing complex pricing requirements. The Charge Type object enables you to configure pricing or discount information and associating items to this information. The Charge Type describes the characteristics of a chargeable item that is displayed in an invoice, such as charge recurrence (for example, one-time, monthly) and method of pricing calculation. A Charge Type can have several different version, each defining a facet of the overall pricing such as the applicable date range.

The Charge Type object consists of several attributes that when configured create pricing for items in a project:

  • Pricing Type
  • Info Model (for customized pricing only)
  • Tax Model

Note: For information regarding Discounts, see Charge Types.

Pricing Type

The Pricing Type field in the Charge Type object describes the characteristics of a chargeable item that is displayed in an invoice, such as charge recurrence (that is, one-time, monthly) and method of pricing calculation. In determining the price of a particular Item in the Product Catalog, the calculation rule to employ is based on the item’s associated Pricing Type:

Type Description Example
Standard Pricing A static fixed price Subscription Fee Installation Fee

Standard Pricing

Standard Pricing is the most common pricing method.

For example, the Product Digital Cable TV service has a subscription fee of $50/month and a $25 Installation fee. 

  • Digital Cable TV Subscription Fee
  • Monthly charge
  • Standard pricing
  • Cable TV Installation Fee
  • One-time charge
  • Standard pricing

The Product is associated with both of these Charge Types.

Charge Type Amount Markup (%)
Digital Cable TV Subscription Fee 50 0
Cable TV Installation Fee 25 0

The Amount is the monetary value of Charge Type, and the Markup is the percentage of desired price increase to the Amount.

Because both Charge Types are Standard Pricing without formula, the resulting price of Charge Type is calculated by the default formula of Charge X (1 + markup/100), resulting in $50 and $25 respectively.  Each of the two Charge Types appears on the Order Invoice as a separate item.

For the recurrence interval of the Charge Types, the Product Catalog does not perform business logic, other than to categorize them in the Order Invoice; a Billing system can integrate and use this information in the Product Catalog for charging and invoicing. The Standard Pricing can also be accompanied with a tailored formula, which uses Catalog Rules Language to compute and return the resulting price.  Formulas provide flexibility to satisfy complex pricing requirements such as variable-based pricing, or looking up for information in other systems (that is, business credit rating). 

Defining Discounts

A Discount defines a promotion to be applied on top of a set of catalog items’ Charge Type associations.

For example, you can create a discount by applying a Christmas promotion of $20 off any Product Smart Phones .  If a Family Plan offer contains multiple Smart Phones, the discount is applied once in each of the applicable phones at the offer’s pricing.

Discount Type Description
Amount An absolute amount to be deducted off the charge (that is, $10 off)
Percentage A percentage amount to be deducted off the charge (that is, 20% off)
Formula A formula script to compute the amount to be deducted off the charge (for example, variable amount based on elapsed years of customer's account)
Charge Type A variable amount to be deducted off the charge, allowing the use of the same pricing mechanism available in a pricing

Some discounts are only applied for a limited period of time. For example, for signing a 1-year cable TV contract you would receive the first 3 months free. The Discount object stores the Effective Period Start and Duration (that is, 1 and 3, respectively, in this case) fields, but does not perform billing business-logic on it. Gimmicks and perks can also be conveyed as discounts by enabling the Display Only option, which does not deduct anything off applicable charges. For example, Per-second billing, or 24/7 Technical Support.

Creating Pricing and Discounts Using Associations

There are several fields available (Info Models, Tax Models and Rules) in the Charge Type object that allow you to associate Catalog information to a Charge Type creating specific pricing.

Creating Pricing Using Info Models

Items can be priced differently based on geographic regions, customer loyalty, and/or class of service, etc, which can create a Charge Type with multiple input variables.  Info Model enables the creation of Pricing with additional input attribute(s). Info Models provide supplementary information tables required to build the Catalog including pricing. Examples may include geographic or demographic categorization data, complex pricing structures, or other generic mapping tables for business logic.

For example, the Component Small Business Internet Service has an Attribute CoS (Class of Service), which can be either Gold, Silver or Bronze.  Internet Gold costs $99/month, Internet Silver costs $79/month, Internet Bronze costs $59/month. An Info Model can be created with Attribute CoS associated. 

With a Charge Type setup as:

  1. Small Business Internet Subscription Fee
  2. Monthly charge
  3. Standard pricing
  4. Tailored formula that
    • Assigns component CoS to pricing
    • Retrieves the price from Price List
    • Returns the result price
Charge Type CoS Charge
Small Business Internet Subscription Fee Gold 99
Small Business Internet Subscription Fee Silver 79
Small Business Internet Subscription Fee Bronze 59

The Custom Price Model now performs CoS-based pricing as required.

Associating Tax Models

Tax Models allows the user to set up various tax situations within a Catalog. It allows users to define a Target Tax (defined in the Taxes object) which allows taxes to be applied, using a percentage value, to a sold item (goods or services). In addition, Target Tax Model option within the Tax Model Items allows taxes to be applied to on top of taxed items. Configuring Tax Model Items and assigning a Sequence to the items allows user to apply tax not only to Item but to each tax sequence cumulatively.

By configuring the Taxes object, you can configure taxes on all charges during pricing or discounts by defining a percentage. For example, Canada’s Goods and Services Tax (HST) is 13%. Next, using the Tax Model you can define the Target Tax (this is the 13% previously defined in the Taxes object), define the Target Tax Model (any additional tax percentage in addition to the 13%) and the sequence.

The Sequence field allows tax to applying on top of another tax. For example, (charges * tax1) * tax2.  For example, Quebec’s Sales Tax (QST) is 8% applied on top of 7.5% GST.

Associating Catalog Rules

Pricing of Catalog items are rarely simple; many rules apply (or do not apply) to charges based on different business variables.  For this reason, various objects in the Product Catalog may be assigned with Catalog Rules to implement these requirements.

The Conditions tab of the Charge Type Details page refers to any rules configured for a Price Type or Discount.

The following table summarizes the pricing objects that are capable of having eligibility rules, and their common uses and examples are highlighted.

Pricing Eligibility Purpose Examples
Charge Type

Conditional Charging based on:

  • choice of attributes / components / products
  • existing customer contract
  1. Waive cable-box rental fee with purchase of PVR
  2. Cable subscription includes the first outlet, but monthly charges apply to extra outlets
  3. Contract renewal fee for premature transferring to another price plan
Discount Type

Eligibility rules of the Discount by:

  • demographics
  • customer loyalty
  • geography
  1. Student Discounts, Senior Discounts
  2. Discount to new customers only
  3. Discounts only applicable to certain states / provinces
Tax Model

Tax eligibility required by law:

  • Regional taxes
  • Tax exemption on individuals
  1. State tax
  2. Tax exemption to First Nations

Similar to non-pricing Eligibility Rule assignments, if multiple eligibility rules are assigned to the same pricing object, all eligibility criteria must be satisfied for the object to be considered eligible.

Associating Items to a Charge Type

Charge Types can be associated at different levels in the Product Catalog, by associating items and groups to the Charge Type.

Object Type Association to Price Model Charge Type Notes
Item Yes  
Item Item Association Yes Has option to ignore pricing at the Item level (and all subsequent levels beneath)

Has option to ignore pricing at the child Item level

If associated, this overrides child Items's Charge Type
Group Yes Takes effect only when none of the child Item have any associated Charge Type at the Item or child Item level

When pricing an ordered Product Catalog item, the charges are tallied top-down in the item’s composition.  For example, if an Item is priced, not only is the Item’s Charge Type (if any) subject to pricing, but the Item's associations are also subject to pricing. All the prices are added to become the total price of the Item. At the Item level, an option can be selected to ignore pricing at the Item association level respectively, thus the applicable items are bypassed in the composition’s pricing.

Logging Charge Type Information

You can use custom logging to troubleshoot Catalog pricing rules when there is a discrepancy processing them in the process engine and the user interface. In the Configuration application, you can add the com.conceptwave.catalognew flag in System > Logging. In Catalog Designer, when you add an item and create a charge type with a pricing rule, and then turn on Test Mode and pricing, information about the charge type is logged.
