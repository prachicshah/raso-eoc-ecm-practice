Activate a Project

When your project is ready to be used in a production environment, change your project's Status field to Active, and then click the Save button.

Alternatively, you can run the following SQL script, which activates all objects in your project:

UPDATE CWPC_ATTRIBUTE SET STATUS = 'ACT';
UPDATE CWPC_ITEM SET STATUS = 'ACT';
UPDATE CWPC_ITEMATTRIBUTE_V SET STATUS = 'ACT';
UPDATE CWPC_CHARGETYPE_V SET STATUS = 'ACT';
UPDATE CWPC_RESTRICTION_V SET STATUS = 'ACT';
UPDATE CWPC_PROJECT SET STATUS = 'ACT';
UPDATE CWPC_TAX_V SET STATUS = 'ACT';
UPDATE CWPC_ITEMCHARGE_V SET STATUS = 'ACT';
UPDATE CWPC_ITEMRULE_V SET STATUS = 'ACT';
UPDATE CWPC_ITEMIMAGE_V SET STATUS = 'ACT';
UPDATE CWPC_ITEMATTACHMENT_V SET STATUS = 'ACT';
UPDATE CWPC_CATALOGHIERARCHY SET STATUS = 'ACT';
UPDATE CWPC_ITEMHIERARCHY SET STATUS = 'ACT';
UPDATE CWPC_ASSOCIATIONTYPE SET STATUS = 'ACT';
UPDATE CWPC_ITEMRELATION_V SET STATUS = 'ACT';
UPDATE CWPC_ITEMLABEL_V SET STATUS = 'ACT';
UPDATE CWPC_ITEMGROUP_V SET STATUS = 'ACT';
UPDATE CWPC_INFOMODEL SET STATUS = 'ACT';
UPDATE CWPC_CATALOGRULE_V SET STATUS = 'ACT';
UPDATE CWPC_TAXMODEL_V SET STATUS = 'ACT';
UPDATE CWPC_INFOTABLE SET STATUS = 'ACT';
UPDATE CWPC_CONTEXT SET STATUS = 'ACT';
commit;

Notes:

  • The project needs to be activated at least 1 hour before the project effective date.
  • You change the status in the script to suit your needs. As an example, if you want to set your project's Status field to Definition, replace ACT with DEF in the sample script.
