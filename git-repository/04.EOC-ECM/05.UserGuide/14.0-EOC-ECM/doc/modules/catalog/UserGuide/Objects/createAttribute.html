Create an Attribute Type

To create a new attribute type, complete these steps:

Note: A project must be opened to create a new attribute type.

  1. From the menu bar, click Designer > Attribute Types.
  2. Click the Add button on the Attribute Types page.

  3. The Attribute Type Detail form displays; enter the following information in the fields:

    4. Field Description Rules
    Code This field denotes the code that is used to identify the attribute type. This code is not restricted to alphanumeric characters; it can also contain a combination of spaces, hyphens, etc. Strings can be up to 32 characters long. Mandatory, Unique
    Status This field indicates a list of available status for the attribute (for example, Definition and Active). Mandatory
    Name This field denotes a unique name for the attribute. Mandatory, Unique
    Start Date This field indicates the effective date of the attribute. Refer to Start and End dates for details. Optional
    End Date

    This field indicates the end date of the attribute that represents the date when this attribute is no longer effective. Refer to Start and End dates for details.

    		 Optional
    Type Some types may have a mandatory restriction. Click the drop-down menu and select one of the following:

    Type Restriction Valid Values
    Number TotalDigits Whole digits (.) decimal digits
    String MaxLength
    Boolean No restriction 0 or 1
    Date No restriction yyyy-mm-dd
    Code Table Select from available code tables

    		Mandatory
    Length

    This field represents the decimal value indicating the maximum length or expected digits. The system defaults to a value of zero (0). For Strings, the length can be a value from 1 to 4000. For Numbers, total number of digits can be 1 to 15. This field is visible when the type is string or number.

    Optional
    Label This field denotes the visual label used for the attribute. The value can be a non-internationalized string up to 128 characters long. This field is a translational string that supports language translations (refer to Velocity Studio's Translation Element for more details). Optional
    Description This field denotes the description of the attribute. Optional
    Project Each object is assigned to a project. The value of this filed defaults to the active or opened project when the attribute is first created. Default
    Owner This field represents the owner of the attribute. The value defaults to the the opened project's owner. Default
    Precision This field indicates the number of digits after the decimal. The value of this field must be equal to or less than the value of length field. This field is visible when type is number.

    Optional
    Code Table This field denotes a list of code cables available to the application used for enumeration. This field is visible when type is code table.

    Optional

  4. Click the Save button.
