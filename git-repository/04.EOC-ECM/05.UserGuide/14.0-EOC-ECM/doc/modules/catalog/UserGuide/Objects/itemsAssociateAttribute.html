Associate an Attribute

To create a new Item Attribute, follow these steps:

Note: A project must be opened to create an item attribute.

  1. From the Overview page, click Commercial Modeling > Items in the Quick start section.
  2. The Item page appears; perform a search and then double-click one item from the results list (for example, Phone Offer).
  3. Click the Attributes tab and then click the New button.
  4. The Item Attribute page appears, click the Finder button next to the Attribute Type field.
  5. The Attribute Finder dialog appears with the list of attributes.
  6. Click one attribute (for example, Bandwidth) from the list and click the Select button.
  7. The Attribute Detail dialog appears with information filled in some fields inherited from the base attribute .

  8. Enter the information in the following fields:

    9. Field Description
    AttributeType This field defines the basic property of an item's attribute and defines the data type of the Item Attribute. This field cannot be updated; the information is inherited from the base Attribute Type.
    Code This is the code used to identify the item attribute. This code is not restricted to alphanumeric characters, it can also contain a combination of spaces, hyphens, etc. Strings can be up to 32 characters long.
    Name The name of the attribute can be used as a visual label for internal use when designing your catalog model (for example, Internet Offer Bandwidth). This field is not supported by language translation.
    Status A list of available Status for this attribute (for example, Definition and Active).
    Type This is a list of association types of category Item attribute.
    Label The visual label used for this attribute, which is used in the application user interface. This field is a translational string that supports language translations (refer to Velocity Studio's Translation Element for more details).
    Start Date The effective date of this attribute. Refer to Start and End dates for details.
    End Date The end date of this attribute represents the date when this attribute is no longer effective. Refer to Start and End dates for details.
    Cancel Objects that are extended have this Cancel flag checked. If the flag is checked, then the extended item does not inherit that association.
    Is Property If checked, the Default Value cannot be modified in the client application.
    Sequence If checked, the Default Value cannot be modified in the client application.
    Project Each object is assigned to a Project. This defaults to the active or opened project when the attribute is first created.
    Default Value The value that appears in the client application during runtime for this item attribute.
    Default Value Rule The default value can be derived from a condition. This launches the Catalog Rule Finder. User is able to select a Catalog Rule which will determine the runtime Default Value for this item attribute.
    Classification Types Select different classification types to identify item under different environments.

  9. Click the Save button.
  10. The item attribute appears under the list of attribute versions.

Alternatively, you can also use the item palette to create an item association.

Default Value Rule

A default value rule can be used to assign attribute values during runtime. For example, during a promotional period the BasicSpeed and HighSpeed Internet offers have been upgraded. The default values for the Upload Speed, Download Speed, and Bandwidth are assigned during the promotional period. The rule would define that during the promotional period the values for attributes Upload Speed is 250K, Download speed is 2MB, and Bandwidth is 30GB/month for the BasicSpeed offer.
