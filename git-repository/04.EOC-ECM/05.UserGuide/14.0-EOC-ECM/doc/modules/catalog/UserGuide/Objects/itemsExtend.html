Extend an Item

The Catalog application provides the functionality to extend the items. The extened item is a new instance of item that inherits the properties of the base item. To extend an item, follow these steps:

  1. From the Overview page, click Commercial Modeling > Items.
  2. The Item page appears; click the Search button to search the available items.
  3. Select one item from the results list and then click the Extend button.
  4. The Item page appears; the fields for this page are prepopulated with the item details with the exception of the Code field. This field remains empty as it has to be unique.
  5. The following fields are available; change the information according to your preferences.

    6. Field Description
    Code This field denotes item code that uniquely identifies the item.
    Item Type This field indicates the item type. It takes on the value of the extended item.
    Status This field denotes the status of the extended item.
    Label The label for the item, which is used in the application user interface. This field is a translational string that supports language translations (refer to Velocity Studio's Translation Element for more details).
    Start Date This field indicates the start date of the item. Refer to Start and End dates for details.
    End Date This field denotes the end date of the item which represents the date when this item is no longer effective. Refer to Start and End dates for details.
    Base Item This field denotes the selected item for extension as the base item.
    Orderable If checked, the item can be ordered.
    Project Each object is assigned to a Project. The value for this field defaults to the project opened in the Catalog application.
    Enforce Actions

    Select this check box to apply action for the items set through the Item Actions. This field takes a Boolean value. The default value is false. You can use this flag during runtime. For example, an exception occurs when you set enforce Action flag to true while item creation but there is no action for add type. The enforce Actions() in Item object is used to get the value.
    Classification Types Select different classification types to identify item under different environments.

  6. Click the Save button to save your changes.
