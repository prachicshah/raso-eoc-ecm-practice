Catalog Menu Options

The Catalog application contains the following menu options:

  • Overview
  • Designer
  • Configuration
  • Test
  • View

Overview

The Overview page provides controls for adding, editing, and removing objects from your catalog. You can also view tasks assigned to the user who is currently logged in to Catalog and available tasks that can be assigned to users.

Designer Menu

The Designer menu displays options to define, control, and maintain Catalog objects, which have properties and attributes associated with them.

The following table describes each Designer menu option:

Menu Item Function
Attribute Types This feature allows you to define, edit, and remove a characteristic of any component or product.
Catalog Rules You can create, define, and edit catalog rules by selecting this option.
Taxes This option allows for defining all your taxation rules.
Tax Models Tax models promote setting up various tax situations within a catalog, such as different tax models for different countries.
Charge Types You can define, edit, and remove charge types, which are a collection of formulas and price tables, which a product or category drives.
Info Models This feature allows you to define, change, and remove references to data tables.
Contexts This option provides a means to configure variables that define user-specific environments, such as the location or market segment.
Association Types Association types define item characteristics, such as content, structure, business rules, specification, and so on).
Catalog Hierarchies A catalog hierarchy is a directory structure that you can define and change for organizing products and offers, and is the framework by which your products nest into expandable groups for selection when integrated with a metadata application.
Projects You can define and change your project, which serves to control changes to the Catalog definition and release the changes to a live production environment in a controlled manner.
Items Items occupy a central place among Catalog Objects and all item properties are expressed through associations.
Change Requests This feature permits creating a new
Business Date Models The business dates model is a concept that represents a set of dates.
Business Date Definitions Business date definitions allow you to handle a number of scenarios that are not handled by the Start Date and End Date fields.
Market Segment Dimensions This feature provides a way of building rules based on market segment.
PSCM Products This option pertains to Ericsson Catalog Manager products, such as specifications and product offerings. Refer to the ECM documentation for details.
Tariff Elements This option allows you to access tariff elements. Refer to the ECM documentation for details.
Product Account Specification This option provides access to product account specifications. Refer to the ECM documentation for details.

Configuration Menu

The Configuration menu allows you to configure various aspects of your catalog.

The following table describes each Configuration menu option:

Menu Item Function
Code Tables This feature allows you to maintain all enumeration lists used in and application, as well as maintain code tables and codes.
Catalog Configuration This page is available to set default currency, markup, date, and privileges for your catalog.
Configure PLD

This menu option allows you to configure your Product Lifecycle Designer (PLD) settings, including the following:

  • Configure templates
  • Configure Stages
  • Configure participants
  • Configure microflows
  • Request type template configuration
Validation This feature allows you to validate your project and catalog.
Rules This option allows you to configure catalog rules.

Test Menu

The Test menu allows you to view and test changes made to your catalog.

View Menu

The View menu provides controls for managing event logs and accessing the Worklist Manager.

A description of these options appears in this table:

Menu Item Function
Event Log This option displays a list of messages logged by the Catalog application. 
Worklist Manager This option displays a list of pending worklist items that require user action.
