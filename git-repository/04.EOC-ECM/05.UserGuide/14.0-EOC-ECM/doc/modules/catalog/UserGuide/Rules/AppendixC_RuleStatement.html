Appendix C: Rule Statement

The Catalog automatically generated the Rule group. However, for internal development testing of the rule compiler, there may be a need to manually write a rule statement.

The rule statement defines the rule name, parameters, and return value, and contains the rule logic. All other statements must be within the rule group. The rule statements cannot be nested one in another.

The catalog rules can be one of two types:

  • Standard rules (condition, validation, or price) that do not have parameters and return a predefined type
  • Helper rules that may have parameters and return a user-defined type

The standard rule format is as follows:

rule[:] Identifier of type condition/validation/price [do[:] / { ]
  Statements...
  end[ rule] / }

The helper rule is as follows:

rule[:] Identifier
     [parameter[s][:] object-type Identifier [,object-type Identifier] ...]
     returning/returns object-type]  [do[:] / { ]
 Statements...
End[ rule] / }

The Rule statement specifies the rule type or the return value of the rule. If the of type clause is used, the rule returns the following values:

  • Type condition should return a Boolean value
  • Type validation should return either the constant Invalid or null
  • Type price should return a number value

If the returning clause is specified, the object type is one of the following data types:

  • Boolean
  • Date
  • Number
  • Constant list of numbers
  • String
  • Constant list of strings
  • Item
  • Row

The following is an example:

  Rule: myRule1                     
       Parameters:  String City, Number Length  
       Returning  Number
  ...
  End rule.      

  Rule myRule2 of type Condition Do 
  ...
  End rule.
