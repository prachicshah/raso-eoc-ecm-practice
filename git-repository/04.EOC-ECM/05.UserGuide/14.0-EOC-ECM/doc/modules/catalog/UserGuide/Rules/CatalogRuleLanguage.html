Catalog Rule Language

The Catalog Rule Language is a specialized language for writing catalog rules. It includes non-procedural constructs that resemble SQL, and reduce the rule size and complexity.

The Rule Language is case sensitive language that has many synonyms, which allows writing the logic in a format that can be easily read and understood by non-programmers. The language also caters to programmers. The language can be written in a format that, from syntax view point, resembles a simpler version of JavaScript.

Rules can be of any standard type (that is, condition, validation, or price) in which case they do not have user-defined parameters and return values, as well as helper rules that take user-defined parameters and return user-defined values. The helper rules can be only global catalog rules. They may have many versions with different content, but they cannot change their signature.
