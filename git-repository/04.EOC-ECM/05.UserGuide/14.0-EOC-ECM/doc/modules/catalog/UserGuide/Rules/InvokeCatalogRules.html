Invoke Catalog rules

A single identifier that is not a keyword, parameter, or variable is considered either a catalog rule name or predefined global function. Catalog rules are global rules that may be called from within another rule. They may or may not have parameters.Â Catalog rules are called using positional parameter notation:

rule-name(expression,  expression,...)

An example of a catalog rule using positional parameter notation is as follows: 

  v1 = rule1(c.name, a + 5)

If a catalog rule does not any parameters, the call must specify the empty parameter list as shown in this example:

  Return  myRule2() + Select Row Count from InfoTable2 
  where City is "Toronto"
