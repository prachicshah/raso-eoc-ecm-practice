Object types

Objects that the rule language works with can be divided in three groups:

  • Simple data types
    A simple data type object contains a value that can be a string, number, Boolean value, or date.
  • Objects
    Objects are complex types that may contain other objects or simple data types, or both. These object types define methods (object functions) that can be performed on any object instance of the same type. Objects that are components of a complex object are called members. Some simple data types have methods, but they do not have members.
  • Collections
    Collections are object that contain zero, one, or more objects of the same type. The rule language supports a collection of basket items and info table rows, but you cannot define variables of collection type. Collections can be used only in the “in�? clause of the “For each�? operator. Constant lists are also considered collections.

Rule language expressions and identifiers have the following object types:

  • String – a simple string data type
  • [Constant] List Of String[s] / String[] – a constant list of strings
  • Number – a simple numeric data type
  • Constant List Of Number[s] / Number[] – a constant list of strings
  • Boolean – a simple Boolean data type
  • Date – a simple date data type
  • Row – a row of info table
  • Item – a basket item
  • Basket – a catalog basket (or shopping basket)
  • Inventory – the inventory for the given request
  • Context – a generic object that contains other named objects, either simple or complex

Each data type exposes methods and allows operations that are defined in their respective sections.
