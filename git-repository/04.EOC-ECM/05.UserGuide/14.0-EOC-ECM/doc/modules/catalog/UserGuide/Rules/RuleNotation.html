Rule Notations

The Catalog Rules section uses the following notations when defining the rule language syntax:

  • Bold words are keywords that have to be written exactly as defined.Â Letter case is not important only for the first letter (all remaining letters are case sensitive), as shown in this example:
    assign expression to identifier
  • Elements in square brackets are optional and can be omitted, as shown in this example:
    select [single] row from InfoTable

    Note: The exception to this rule is the short form for constant list types (see Statements), which has either the format String[] or Number[].
  • The ellipses specify repeating elements:
    [variable]object-type Identifier [is Exp][, Identifier [as Exp]]...
  • A forward slash divides a list of alternative keywords. Only one keyword can be used, as shown in this example:
    while Exp [do[:]/ { ]
