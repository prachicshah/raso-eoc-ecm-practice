Catalog Rule Types

Rules are computational units that may receive parameters and each rule always has a return value. The rules may be called from the Catalog product or from other rules.

All rules have access to the following four predefined parameters:

  • theItem
  • theBasket
  • theContext
  • theInventory
  • theBasketItemCharge

The Catalog offers two different types of rules:

  • Object rules
    • , which are strictly called by the Catalog product
  • Catalog rules, which define a rich set of business rules that can be attached to the Catalog elements

Object Rules

The rules that are called only by the Catalog product are called Object rules. The object rules are not reusable â€“ other rules cannot call them. The object rules are of type Condition, Validation, or Price. They cannot define parameters or return values. If an object rule consists only of one return statement, you may omit the return keywords and enter only the expression.

Catalog Rules

Rules are global, which means that they can be called from other rules, are called Catalog rules. For example, an Attribute element can access a rule that was configured in the Item element. They must have unique names. The parameters and the return value of such rules are user-defined. If a Catalog rule consists only of one return statement, you may omit the return keywords and enter only the expression.

There are three different types of catalog rules:

  • Condition rules that check if a condition is valid. Condition rules return Boolean value of true if the condition is valid.
  • Validation rules that check if the shopping basket or particular item of it is correctly configured. Validation rules can add one or more errors to the operation error list. The validation rule result is either invalid or null. The invalid constant indicates that the Catalog adds the error message to the operation error list. If a message (code) is defined for the rule, the Catalog puts it in the operation error list. If there is no message defined for the rule, the Catalog uses a predefined error message with code DE9179 and the text, Validation rule xxx failed for item xxx. The null constant indicates that the rule has either found no errors or added the corresponding errors to the error list using the function reportError. In this case, the Catalog does not add any message to the operation error list.
  • Pricing rules (formulas) that calculate the price of a given item or a set of items. The price rules return a number that represents the price.

The rules are either locally defined for an element and can be used only within this element (element or local rules), or are globally defined and can be used by any item or called by other rule language specifications (Catalog or global rules).
